PROJECT 9: REDUCING NETWORK TRAFFIC — SUMMARIZING SALES BEFORE SHIPPING

🔴 HYPOTHETICAL PROBLEM

SCENARIO:
→ You are the data engineer at a nationwide retail chain
→ 5,000 stores report daily transactions to HDFS
→ Data: 50GB per day, stored at /retail/raw/transactions/2024/01/
→ Each line: store_id,product_id,quantity,unit_price,timestamp
→ Your job: Sum total revenue per store per day

YOU RUN THE JOB FROM PROJECT 4/5 PATTERN:
→ Mapper: emit (store_id, revenue)   ← revenue = quantity × unit_price
→ Reducer: sum all revenues per store
→ Job completes but you check Project 7 counters and see:

  Map input records:     500,000,000    (500 million transactions)
  Map output records:    500,000,000    (one emit per transaction)
  Map output bytes:      12,000,000,000 (12 GB of mapper output)
  Reduce shuffle bytes:  12,000,000,000 (12 GB transferred over network!)
  Reduce input records:  500,000,000    (all 500M records hit reducers)

THE PROBLEM:
→ 12 GB of data transferred across the network during shuffle
→ With 100 mappers and 10 reducers:
   Each reducer pulls from ALL 100 mappers = 1000 network connections
→ Network becomes the BOTTLENECK
→ Job takes 45 minutes, 30 minutes of which is SHUFFLE

THE INSIGHT:
→ Each mapper sees MANY transactions for the SAME store
→ Mapper 1 might see 50,000 transactions for Store_001
→ It emits 50,000 separate (Store_001, revenue) pairs
→ ALL 50,000 cross the network to reach the reducer
→ But the reducer just SUMS them!
→ Why not SUM ON THE MAPPER SIDE first?
→ Then emit ONE pair: (Store_001, total_revenue_from_this_mapper)
→ 50,000 pairs reduced to 1 pair BEFORE crossing the network
→ THIS IS WHAT A COMBINER DOES

🔍 HOW TO APPROACH IT

FROM PREVIOUS PROJECTS WE KNOW:
→ Project 5: Shuffle is the most expensive phase (network I/O)
→ Project 5: SUM, MAX, MIN are ASSOCIATIVE operations
→ Project 5: AVERAGE is NOT associative (cannot combine partial averages)
→ Project 6: In-mapper aggregation via HashMap (similar concept)
→ Project 7: Reduce shuffle bytes counter tracks network cost

THE COMBINER CONCEPT:

Without Combiner:
┌─────────┐                                    ┌──────────┐
│ Mapper 1 │  (Store_001, 10.50)  ────────────→│          │
│          │  (Store_001, 25.00)  ────────────→│          │
│          │  (Store_001, 15.75)  ────────────→│          │
│          │  (Store_002, 30.00)  ────────────→│ Reducer  │
│          │  (Store_001, 8.25)   ────────────→│          │
│          │  (Store_002, 12.50)  ────────────→│          │
└─────────┘  6 pairs over network              └──────────┘

With Combiner (runs ON the mapper side, BEFORE network transfer):
┌─────────┐         ┌──────────┐               ┌──────────┐
│ Mapper 1 │ ──────→│ Combiner │               │          │
│          │ 6 pairs│ (local)  │  2 pairs      │ Reducer  │
│          │        │          │──────────────→│          │
│          │        │ Store_001│  (Store_001,  │          │
│          │        │  sum=    │   59.50)      │          │
│          │        │  59.50   │  (Store_002,  │          │
│          │        │ Store_002│   42.50)      │          │
│          │        │  sum=    │               │          │
│          │        │  42.50   │               │          │
└─────────┘        └──────────┘               └──────────┘
                   6 → 2 pairs                 67% REDUCTION

Scale this to 500 million records with 5,000 stores:
→ Without combiner: 500,000,000 pairs cross network (12 GB)
→ With combiner: ~5,000 pairs per mapper × 400 mappers = 2,000,000 pairs
→ Network: ~50 MB instead of 12 GB
→ REDUCTION: 99.6%!

HOW IT WORKS MECHANICALLY:

┌──────────────────────────────────────────────────────────────────────┐
│ MAPPER PIPELINE WITH COMBINER                                        │
│                                                                      │
│ ① map() emits pairs → written to circular buffer (100MB)            │
│                                                                      │
│ ② Buffer reaches 80% → SPILL triggered                              │
│    → Sort by (partition, key)                                        │
│    → IF COMBINER SET: Run combiner on sorted spill data             │
│    → Write to local disk (spill file)                                │
│                                                                      │
│ ③ After all map() calls complete:                                   │
│    → Merge all spill files                                           │
│    → IF COMBINER SET: Run combiner AGAIN during merge               │
│    → Final merged file ready for reducer to fetch                    │
│                                                                      │
│ IMPORTANT: Combiner may run 0, 1, or MULTIPLE times                 │
│ → 0 times: if map output fits in buffer without spilling            │
│ → 1 time: during single spill                                       │
│ → N times: once per spill + once during merge                       │
│ → YOU CANNOT DEPEND ON HOW MANY TIMES IT RUNS                      │
│                                                                      │
│ THIS MEANS: Combiner MUST be IDEMPOTENT and ASSOCIATIVE             │
│ → Running it 0 times = same final result as running it 100 times   │
│ → This restricts WHICH operations can use a combiner               │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

💡 HOW CAN WE SOLVE IT

THE COMBINER CONTRACT — CRITICAL RULES:

RULE 1: Combiner input types MUST match mapper output types
  → Mapper output:   (Text, DoubleWritable)
  → Combiner input:  (Text, DoubleWritable)  ← MUST match

RULE 2: Combiner output types MUST match mapper output types
  → Combiner output: (Text, DoubleWritable)  ← MUST match
  → NOT the reducer output types!

  WHY? Because combiner output feeds BACK into the shuffle pipeline
  → Mapper → Combiner → Shuffle → Reducer
  → Combiner sits BETWEEN mapper and shuffle
  → Its output must be compatible with what the reducer expects as INPUT
  → Since reducer input = mapper output, combiner output = mapper output

RULE 3: Combiner MUST be mathematically valid for the operation
  → The final result MUST be the same whether combiner runs 0 or N times
  → This means the operation must be ASSOCIATIVE and COMMUTATIVE

  combine(combine(a, b), c) = combine(a, combine(b, c))  ← ASSOCIATIVE
  combine(a, b) = combine(b, a)                           ← COMMUTATIVE

WHICH OPERATIONS CAN USE A COMBINER?

┌──────────────────┬───────────┬───────────────────────────────────────┐
│ OPERATION        │ COMBINER? │ REASON                                │
├──────────────────┼───────────┼───────────────────────────────────────┤
│ SUM              │ ✅ YES    │ sum(sum(a,b), sum(c,d)) = sum(a,b,c,d)│
│ COUNT            │ ✅ YES    │ Count = sum of 1s (same as SUM)       │
│ MAX              │ ✅ YES    │ max(max(a,b), max(c,d)) = max(a,b,c,d)│
│ MIN              │ ✅ YES    │ min(min(a,b), min(c,d)) = min(a,b,c,d)│
│ PRODUCT          │ ✅ YES    │ Multiplication is associative         │
│                  │           │                                        │
│ AVERAGE          │ ❌ NO     │ avg(avg(a,b), avg(c,d)) ≠ avg(a,b,c,d)│
│                  │           │ (Project 5 proved this)               │
│ MEDIAN           │ ❌ NO     │ Cannot combine partial medians        │
│ STANDARD DEV     │ ❌ NO     │ Cannot combine partial std devs       │
│ DISTINCT COUNT   │ ❌ NO     │ Cannot combine partial distinct counts│
│ TOP N            │ ❌ PARTIAL│ Can combine but may miss borderline   │
│                  │           │                                        │
│ AVERAGE (trick)  │ ✅ YES*   │ *If mapper emits (sum, count) pair    │
│                  │           │  Combiner sums sums and sums counts   │
│                  │           │  Reducer computes total_sum/total_count│
└──────────────────┴───────────┴───────────────────────────────────────┘

THE AVERAGE TRICK IN DETAIL:

  Instead of: Mapper emits (store_id, revenue)
  We emit:    Mapper emits (store_id, "sum:revenue,count:1")

  Combiner receives (store_id, ["sum:10.50,count:1", "sum:25.00,count:1", ...])
  Combiner emits:   (store_id, "sum:35.50,count:2")  ← summed both fields

  Reducer receives:  (store_id, ["sum:35.50,count:2", "sum:80.00,count:5", ...])
  Reducer computes:  total_sum / total_count = correct average

  This works because BOTH sum and count are individually associative!

✅ WHAT COULD BE DONE

THREE IMPLEMENTATION APPROACHES:

APPROACH 1: Reducer as Combiner (simplest)
→ If your reducer is ALREADY associative (sum, max, min, count)
→ Just set: job.setCombinerClass(MyReducer.class)
→ The SAME class runs as both combiner and reducer
→ Works when: reducer input types = reducer output types = mapper output types

APPROACH 2: Separate Combiner class
→ When combiner logic differs from reducer
→ Example: Reducer computes average (sum/count)
→ Combiner must emit (sum, count) pairs — different logic from final division
→ Write a dedicated CombinerClass

APPROACH 3: In-Mapper Combining (manual combiner)
→ From Project 6: accumulate in HashMap during map(), emit in cleanup()
→ ALWAYS runs exactly once (predictable)
→ More memory usage but guaranteed behavior
→ Can be combined WITH a formal combiner for double optimization

✅ WHAT SHOULD DO

1. ALWAYS set a combiner when the operation is associative
   → It costs ONE line of code: job.setCombinerClass(MyReducer.class)
   → Benefit: potentially 90%+ network reduction
   → No downside if operation is truly associative
   → If you forget: job still produces correct results, just slower

2. VERIFY combiner effectiveness via counters (Project 7)
   → Combine input records:  500,000,000  (entered combiner)
   → Combine output records: 2,000,000    (left combiner)
   → Reduction ratio: 99.6%
   → If combine output ≈ combine input: combiner not helping
     (all keys unique — no combining possible)

3. USE the "Reducer as Combiner" pattern when types match
   → Mapper output: (Text, LongWritable)
   → Reducer input: (Text, LongWritable)
   → Reducer output: (Text, LongWritable)
   → All types match → reducer can be combiner directly

4. USE a separate Combiner when types DON'T match
   → Mapper output: (Text, Text)  where Text = "sum:X,count:N"
   → Reducer output: (Text, DoubleWritable) where Double = average
   → Types differ → need separate combiner class

5. COMPARE shuffle bytes WITH and WITHOUT combiner
   → Run job once without combiner → note Reduce shuffle bytes
   → Add combiner → run again → note new Reduce shuffle bytes
   → The difference is your network savings

6. COMBINE in-mapper aggregation WITH formal combiner
   → In-mapper: aggregates across records in ONE mapper's input
   → Formal combiner: aggregates across spill files
   → Together: maximum possible reduction

🚫 WHAT SHOULD NOT DO

1. DO NOT use a combiner for non-associative operations
   → AVERAGE: combine(avg(1,2), avg(3)) = avg(1.5, 3) = 2.25
   → CORRECT: avg(1, 2, 3) = 2.0
   → DIFFERENT! Combiner gives WRONG results
   → If your operation isn't in the "YES" column above: NO COMBINER

2. DO NOT assume the combiner runs a specific number of times
   → It might run 0 times (no spills — small output fits in buffer)
   → It might run 1 time (one spill)
   → It might run 5 times (multiple spills + merge)
   → Your FINAL RESULT must be the same regardless
   → Test by running with and without combiner — results must match

3. DO NOT have combiner output types differ from mapper output types
   → Combiner output feeds BACK into shuffle
   → Shuffle expects mapper output types
   → Type mismatch → ClassCastException at runtime
   → RULE: Combiner input types = Combiner output types = Mapper output types

4. DO NOT use combiner as a replacement for proper data modeling
   → If every key is unique: combiner does NOTHING
   → 500M records with 500M unique keys → 0% reduction
   → Combiner helps when many records share the same key
   → For high-cardinality keys: in-mapper combining is better

5. DO NOT confuse Combiner with Reducer
   → Combiner runs on MAPPER side (local to each mapper)
   → Reducer runs on REDUCER side (after network shuffle)
   → Combiner may not see ALL values for a key (only this mapper's values)
   → Reducer ALWAYS sees ALL values for a key (from all mappers)
   → Combiner is an OPTIMIZATION — reducer is the COMPUTATION

6. DO NOT use combiner to change the number of output keys
   → Combiner should only aggregate values for existing keys
   → Adding new keys or filtering keys in combiner = unpredictable results
   → Because combiner may run 0 times: filtered keys would reappear

🚫 WHEN NOT TO USE

1. WHEN operation is NOT associative (already covered above)

2. WHEN every key is unique
   → Combiner groups by key — if no key repeats within a mapper: nothing to combine
   → Example: Mapper emits (unique_transaction_id, amount)
   → Every key appears once → combiner runs but produces same output
   → Wasted CPU on sorting + combining with zero benefit
   → CHECK: If combine output records ≈ combine input records → remove combiner

3. WHEN mapper output is already tiny
   → If mapper filters 99% of data and emits very little
   → Shuffle volume is already small
   → Combiner adds overhead (sorting, combining) for minimal savings
   → Only add combiner when shuffle bytes are a significant bottleneck

4. WHEN combiner CPU cost exceeds network savings
   → Combiner runs sorting + combining logic
   → If keys are very long strings: sorting is expensive
   → If combining logic is complex: CPU cost per record is high
   → Rare situation but possible with complex custom combiners

│   Reducer output:  (Text, DoubleWritable)  ← final average         │
│                                                                      │
│   Combiner I/O types = Mapper output types ✓                       │
│   Reducer output types can differ ✓ (final result format)           │

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "Results differ with and without combiner"
→ REASON: Operation is NOT associative
→ TEST: Run job with combiner, then without (remove setCombinerClass)
→ If results differ: your operation cannot use a combiner
→ Most common offender: AVERAGE without sum+count trick

PITFALL 2: "Combiner runs but Combine output ≈ Combine input (no reduction)"
→ REASON 1: All keys are unique (no duplicates to combine)
→ REASON 2: Very few records per spill (buffer barely fills)
→ REASON 3: High cardinality keys with few records per mapper
→ CHECK: If 5,000 stores and each mapper sees ~50,000 records:
   average 10 records per store per mapper → 90% reduction possible
→ CHECK: If 50,000,000 unique keys and 500,000 records per mapper:
   average 0.01 records per key per mapper → 0% reduction

PITFALL 3: "ClassCastException in combiner"
→ REASON: Combiner output types don't match mapper output types
→ EXAMPLE: Mapper outputs (Text, IntWritable)
           Combiner outputs (Text, DoubleWritable)  ← TYPE MISMATCH!
→ Shuffle expects IntWritable but receives DoubleWritable
→ FIX: Combiner I/O types must EXACTLY match mapper output types

PITFALL 4: "Combiner seems to change the number of keys"
→ REASON: Combiner has filtering logic (skipping some keys)
→ PROBLEM: Combiner might run 0 times → filtered keys reappear
→ Results become non-deterministic
→ RULE: Combiner should NEVER filter keys — only aggregate values

PITFALL 5: "Job is SLOWER with combiner than without"
→ REASON: Combiner adds sorting + combining overhead
→ If shuffle volume is already tiny: overhead > savings
→ If key comparison is expensive (long strings): sorting dominates
→ CHECK: Compare wall clock time and shuffle bytes with/without
→ FIX: Remove combiner if shuffle bytes savings < 20%

PITFALL 6: "In-mapper combining + formal combiner causes double counting"
→ REASON: In-mapper combining already summed values
→ Formal combiner sums the already-summed values AGAIN
→ This is CORRECT for SUM (sum of sums = total sum)
→ This is WRONG if in-mapper computed averages (average of averages)
→ ENSURE: Both levels combine the SAME WAY

→ Maximum reduction: N records per key per mapper → 1 record
→ After that: adding more data just adds more mappers

import java.io.IOException;

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.io.DoubleWritable;
import org.apache.hadoop.io.LongWritable;

import org.apache.hadoop.mapreduce.Job;

public class StoreRevenue {

    // ================================================================
    // MAPPER: Parse transactions, emit (store_id, revenue)
    // Same mapper used for ALL three approaches
    // ================================================================
    
    public static class RevenueMapper
            extends Mapper<LongWritable, Text, Text, DoubleWritable> {
        
        private final Text storeKey = new Text();
        private final DoubleWritable revenueValue = new DoubleWritable();
        
        @Override
        protected void map(LongWritable key, Text value, Context context)
                throws IOException, InterruptedException {
            
            String line = value.toString();
            if (line.startsWith("store_id")) return;  // Skip header (Project 4 pattern)
            
            try {
                String[] fields = line.split(",");
                if (fields.length < 5) return;
                
                String storeId = fields[0].trim();
                int quantity = Integer.parseInt(fields[2].trim());
                double unitPrice = Double.parseDouble(fields[3].trim());
                double revenue = quantity * unitPrice;
                
                storeKey.set(storeId);
                revenueValue.set(revenue);
                context.write(storeKey, revenueValue);
                
            } catch (Exception e) {
                context.getCounter("StoreRevenue", "parse_errors").increment(1);
            }
        }
    }
    
    
    // ================================================================
    // IN-MAPPER COMBINING MAPPER: Accumulate in HashMap, emit in cleanup
    // (Project 6 pattern, applied here for combiner comparison)
    // ================================================================
    
    public static class InMapperCombiningMapper
            extends Mapper<LongWritable, Text, Text, DoubleWritable> {
        
        private Map<String, Double> storeRevenues;
        
        @Override
        protected void setup(Context context) {
            storeRevenues = new HashMap<>();
        }
        
        @Override
        protected void map(LongWritable key, Text value, Context context)
                throws IOException, InterruptedException {
            
            String line = value.toString();
            if (line.startsWith("store_id")) return;
            
            try {
                String[] fields = line.split(",");
                if (fields.length < 5) return;
                
                String storeId = fields[0].trim();
                int quantity = Integer.parseInt(fields[2].trim());
                double unitPrice = Double.parseDouble(fields[3].trim());
                double revenue = quantity * unitPrice;
                
                // Accumulate in HashMap — no emit yet!
                storeRevenues.merge(storeId, revenue, Double::sum);
                
            } catch (Exception e) {
                context.getCounter("StoreRevenue", "parse_errors").increment(1);
            }
        }
        
        @Override
        protected void cleanup(Context context)
                throws IOException, InterruptedException {
            
            // Emit aggregated results — ONE pair per store per mapper
            Text storeKey = new Text();
            DoubleWritable revenueValue = new DoubleWritable();
            
            for (Map.Entry<String, Double> entry : storeRevenues.entrySet()) {
                storeKey.set(entry.getKey());
                revenueValue.set(entry.getValue());
                context.write(storeKey, revenueValue);
            }
            
            context.getCounter("StoreRevenue", "unique_stores_in_mapper")
                   .increment(storeRevenues.size());
        }
    }
    
    
    // ================================================================
    // REDUCER: Sum revenues per store (also usable as Combiner!)
    // ================================================================
    
    public static class RevenueSumReducer
            extends Reducer<Text, DoubleWritable, Text, DoubleWritable> {
        
        private final DoubleWritable totalRevenue = new DoubleWritable();
        
        @Override
        protected void reduce(Text key, Iterable<DoubleWritable> values, Context context)
                throws IOException, InterruptedException {
            
            double sum = 0.0;
            for (DoubleWritable val : values) {
                sum += val.get();
            }
            
            totalRevenue.set(Math.round(sum * 100.0) / 100.0);  // Round to cents
            context.write(key, totalRevenue);
        }
    }
    
    
    // ================================================================
    // AVERAGE COMBINER: For demonstrating the sum+count trick
    // ================================================================
    
    // Mapper for average: emits (store_id, "revenue,1")
    public static class AverageMapper
            extends Mapper<LongWritable, Text, Text, Text> {
        
        private final Text storeKey = new Text();
        private final Text sumCountValue = new Text();
        
        @Override
        protected void map(LongWritable key, Text value, Context context)
                throws IOException, InterruptedException {
            
            String line = value.toString();
            if (line.startsWith("store_id")) return;
            
            try {
                String[] fields = line.split(",");
                if (fields.length < 5) return;
                
                String storeId = fields[0].trim();
                int quantity = Integer.parseInt(fields[2].trim());
                double unitPrice = Double.parseDouble(fields[3].trim());
                double revenue = quantity * unitPrice;
                
                // Emit as "sum,count" pair
                storeKey.set(storeId);
                sumCountValue.set(revenue + ",1");
                context.write(storeKey, sumCountValue);
                
            } catch (Exception e) {
                context.getCounter("StoreRevenue", "parse_errors").increment(1);
            }
        }
    }
    
    // Combiner for average: sums sums and sums counts
    public static class AverageCombiner
            extends Reducer<Text, Text, Text, Text> {
        
        private final Text outputValue = new Text();
        
        @Override
        protected void reduce(Text key, Iterable<Text> values, Context context)
                throws IOException, InterruptedException {
            
            double totalSum = 0.0;
            long totalCount = 0;
            
            for (Text val : values) {
                String[] parts = val.toString().split(",");
                totalSum += Double.parseDouble(parts[0]);
                totalCount += Long.parseLong(parts[1]);
            }
            
            // Emit combined sum and count — NOT the average!
            outputValue.set(totalSum + "," + totalCount);
            context.write(key, outputValue);
        }
    }
    
    // Reducer for average: computes final average from sum/count
    public static class AverageReducer
            extends Reducer<Text, Text, Text, Text> {
        
        private final Text outputValue = new Text();
        
        @Override
        protected void reduce(Text key, Iterable<Text> values, Context context)
                throws IOException, InterruptedException {
            
            double totalSum = 0.0;
            long totalCount = 0;
            
            for (Text val : values) {
                String[] parts = val.toString().split(",");
                totalSum += Double.parseDouble(parts[0]);
                totalCount += Long.parseLong(parts[1]);
            }
            
            double average = totalCount > 0 ? totalSum / totalCount : 0.0;
            
            outputValue.set(String.format(
                "avg:%.2f|total:%.2f|count:%d", average, totalSum, totalCount
            ));
            context.write(key, outputValue);
        }
    }

    public static void main(String[] args) throws Exception {
        
        if (args.length < 3) {
            System.err.println("Usage: StoreRevenue <mode> <input> <output>");
            System.err.println("  mode: 'nocombiner', 'withcombiner', 'inmapper', 'average'");
            System.exit(1);
        }
        
        String mode = args[0].toLowerCase();
        String inputPath = args[1];
        String outputPath = args[2];
        
        Configuration conf = new Configuration();
        FileSystem fs = FileSystem.get(conf);
        Path outPath = new Path(outputPath);
        if (fs.exists(outPath)) fs.delete(outPath, true);
        
        Job job = Job.getInstance(conf);
        job.setJarByClass(StoreRevenue.class);
        job.setNumReduceTasks(1);
        
        FileInputFormat.addInputPath(job, new Path(inputPath));
        FileOutputFormat.setOutputPath(job, outPath);
        
        switch (mode) {
            case "nocombiner":
                job.setJobName("Store Revenue — NO Combiner");
                job.setMapperClass(RevenueMapper.class);
                job.setReducerClass(RevenueSumReducer.class);
                // NO combiner set!
                job.setOutputKeyClass(Text.class);
                job.setOutputValueClass(DoubleWritable.class);
                break;
                
            case "withcombiner":
                job.setJobName("Store Revenue — WITH Combiner");
                job.setMapperClass(RevenueMapper.class);
                job.setReducerClass(RevenueSumReducer.class);
                // ★ THE KEY LINE: Set combiner = reducer (SUM is associative)
                job.setCombinerClass(RevenueSumReducer.class);
                job.setOutputKeyClass(Text.class);
                job.setOutputValueClass(DoubleWritable.class);
                break;
                
            case "inmapper":
                job.setJobName("Store Revenue — In-Mapper Combining");
                job.setMapperClass(InMapperCombiningMapper.class);
                job.setReducerClass(RevenueSumReducer.class);
                // Also add formal combiner for spill-level optimization
                job.setCombinerClass(RevenueSumReducer.class);
                job.setOutputKeyClass(Text.class);
                job.setOutputValueClass(DoubleWritable.class);
                break;
                
            case "average":
                job.setJobName("Store Revenue — Average with Combiner Trick");
                job.setMapperClass(AverageMapper.class);
                job.setCombinerClass(AverageCombiner.class);
                job.setReducerClass(AverageReducer.class);
                // Map output types = Combiner I/O types = (Text, Text)
                job.setMapOutputKeyClass(Text.class);
                job.setMapOutputValueClass(Text.class);
                // Reducer output types = (Text, Text)
                job.setOutputKeyClass(Text.class);
                job.setOutputValueClass(Text.class);
                break;
                
            default:
                System.err.println("Unknown mode: " + mode);
                System.exit(1);
        }
        
        System.out.println("========================================");
        System.out.println("Mode: " + mode.toUpperCase());
        System.out.println("========================================");
        
        long startTime = System.currentTimeMillis();
        boolean success = job.waitForCompletion(true);
        long elapsed = System.currentTimeMillis() - startTime;
        
        // ── Print combiner analysis ──
        Counters counters = job.getCounters();
        
        long mapOutput = counters.findCounter(
            "org.apache.hadoop.mapreduce.TaskCounter", "MAP_OUTPUT_RECORDS").getValue();
        long combineInput = counters.findCounter(
            "org.apache.hadoop.mapreduce.TaskCounter", "COMBINE_INPUT_RECORDS").getValue();
        long combineOutput = counters.findCounter(
            "org.apache.hadoop.mapreduce.TaskCounter", "COMBINE_OUTPUT_RECORDS").getValue();
        long reduceInput = counters.findCounter(
            "org.apache.hadoop.mapreduce.TaskCounter", "REDUCE_INPUT_RECORDS").getValue();
        long shuffleBytes = counters.findCounter(
            "org.apache.hadoop.mapreduce.TaskCounter", "REDUCE_SHUFFLE_BYTES").getValue();
        
        System.out.println("\n========================================");
        System.out.println("COMBINER ANALYSIS");
        System.out.println("========================================");
        System.out.printf("Map output records:     %,15d%n", mapOutput);
        System.out.printf("Combine input records:  %,15d%n", combineInput);
        System.out.printf("Combine output records: %,15d%n", combineOutput);
        System.out.printf("Reduce input records:   %,15d%n", reduceInput);
        System.out.printf("Shuffle bytes:          %,15d (%s)%n", shuffleBytes,
            shuffleBytes > 1024*1024 ? String.format("%.1f MB", shuffleBytes/1024.0/1024.0) :
            String.format("%.1f KB", shuffleBytes/1024.0));
        
        if (combineInput > 0 && combineOutput > 0) {
            double reductionPct = (1.0 - (double) combineOutput / combineInput) * 100;
            System.out.printf("Combiner reduction:     %.1f%%%n", reductionPct);
        } else {
            System.out.println("Combiner: NOT USED");
        }
        
        System.out.printf("Total time:             %d seconds%n", elapsed / 1000);
        System.out.println("========================================");
        
        System.exit(success ? 0 : 1);
    }
}
