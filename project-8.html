PROJECT 8: RUNNING EVERYTHING LOCALLY BEFORE THE CLUSTER — PSEUDO-DISTRIBUTED MODE

🔴 HYPOTHETICAL PROBLEM

SCENARIO:
→ You have been developing MapReduce jobs on your laptop (Projects 1-7)
→ Your company just approved a 20-node production Hadoop cluster
→ The ops team asks you THREE critical questions:

  Q1: "What configuration files do we need and what goes in each?"
  Q2: "How do we size memory for mappers and reducers?"
  Q3: "How do we test everything before deploying to the real cluster?"

WHAT GOES WRONG WITHOUT UNDERSTANDING CONFIGURATION:

  Developer A: Deploys to production with replication=1
  → First DataNode failure = DATA LOSS

  Developer B: Sets mapper memory to 512MB on machines with 64GB RAM
  → Only uses 1% of available resources per node → jobs take forever

  Developer C: Tests only in local mode, deploys to cluster
  → Job fails because HDFS paths don't exist on cluster
  → Classpath differences between local and distributed mode
  → Serialization issues that only appear with real network shuffle

  Developer D: Copies production config to laptop for development
  → Laptop tries to contact non-existent NameNode at prod-master:9000
  → HDFS tries to create 3 replicas with 1 DataNode → constant warnings
  → YARN allocates 8GB per container → laptop runs out of memory

THE ROOT CAUSE:
→ Hadoop has THREE execution modes
→ Each mode requires DIFFERENT configuration
→ Understanding EVERY property is essential for reliable operation
→ Configuration mistakes are the #1 cause of Hadoop problems

FROM PREVIOUS PROJECTS:
→ Project 1: We set up pseudo-distributed mode (basic config)
→ Project 7: We saw how misconfiguration causes mysterious failures
→ NOW: We understand EVERY configuration property and WHY it exists

🔍 HOW TO APPROACH IT

HADOOP'S THREE EXECUTION MODES:

┌──────────────────────────────────────────────────────────────────────┐
│ MODE 1: LOCAL (STANDALONE) MODE                                      │
│                                                                      │
│ → NO daemons running (no NameNode, no DataNode, no YARN)            │
│ → Everything runs in a SINGLE JVM process                           │
│ → Files read from LOCAL filesystem (not HDFS)                       │
│ → MapReduce runs as sequential Java code (no parallelism)           │
│ → NO network, NO shuffle, NO replication                            │
│ → FASTEST for small tests and debugging                             │
│ → DEFAULT mode (no configuration needed)                            │
│                                                                      │
│ USE FOR: Unit testing, quick prototyping, debugging logic            │
│                                                                      │
│ COMMAND: hadoop jar MyJob.jar input/ output/                        │
│ (reads "input/" from LOCAL filesystem, writes "output/" locally)    │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ MODE 2: PSEUDO-DISTRIBUTED MODE                                      │
│                                                                      │
│ → ALL daemons running on ONE machine                                │
│ → NameNode, DataNode, ResourceManager, NodeManager — all local      │
│ → Files stored in HDFS (real distributed filesystem, 1 node)        │
│ → MapReduce runs through YARN (real scheduling, real containers)    │
│ → Real network shuffle (localhost to localhost)                      │
│ → Real serialization/deserialization of Writables                   │
│ → Replication factor = 1 (only one DataNode)                        │
│                                                                      │
│ USE FOR: Integration testing, development, learning                  │
│ THIS IS WHAT WE'VE BEEN USING IN PROJECTS 1-7                      │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ MODE 3: FULLY DISTRIBUTED MODE                                       │
│                                                                      │
│ → Daemons distributed across MULTIPLE machines                      │
│ → NameNode on dedicated master machine                              │
│ → DataNodes on many worker machines                                 │
│ → ResourceManager on dedicated machine (or same as NameNode)        │
│ → NodeManagers on worker machines (alongside DataNodes)             │
│ → Real network across physical/virtual machines                     │
│ → Replication factor = 3 (production standard)                      │
│ → Rack awareness for fault tolerance                                │
│                                                                      │
│ USE FOR: Production, performance testing at scale                    │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

THE FOUR CORE CONFIGURATION FILES — COMPLETE ANATOMY:

Every Hadoop installation has these files in $HADOOP_HOME/etc/hadoop/

┌──────────────────────────────────────────────────────────────────────┐
│ FILE 1: core-site.xml                                                │
│ PURPOSE: Global settings shared by ALL Hadoop components            │
│ AFFECTS: HDFS, MapReduce, YARN, and all ecosystem tools             │
│                                                                      │
│ KEY PROPERTIES:                                                      │
│                                                                      │
│ fs.defaultFS                                                         │
│   → The DEFAULT filesystem URI                                       │
│   → Local mode: file:///  (local filesystem)                        │
│   → Distributed: hdfs://namenode-host:9000                          │
│   → EVERY hdfs dfs command uses this as base                        │
│   → EVERY MapReduce job reads input from this filesystem            │
│                                                                      │
│ hadoop.tmp.dir                                                       │
│   → Base temporary directory for Hadoop                             │
│   → NameNode, DataNode, YARN store temp data here                   │
│   → DEFAULT: /tmp/hadoop-${user.name}                               │
│   → ⚠️ /tmp is cleared on reboot on many Linux systems!            │
│   → ALWAYS set to a persistent directory in production              │
│                                                                      │
│ io.file.buffer.size                                                  │
│   → Buffer size for file I/O operations                             │
│   → DEFAULT: 4096 (4KB)                                             │
│   → RECOMMENDED: 131072 (128KB) for production                     │
│   → Larger buffer = fewer system calls = better throughput          │
│                                                                      │
│ fs.trash.interval                                                    │
│   → Minutes before trash is auto-deleted                            │
│   → DEFAULT: 0 (trash disabled)                                     │
│   → RECOMMENDED: 1440 (24 hours) for safety                        │
│   → Allows recovery of accidentally deleted files                   │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ FILE 2: hdfs-site.xml                                                │
│ PURPOSE: HDFS-specific settings                                      │
│ AFFECTS: NameNode, DataNode, HDFS clients                           │
│                                                                      │
│ KEY PROPERTIES:                                                      │
│                                                                      │
│ dfs.replication                                                      │
│   → Default replication factor for new files                        │
│   → Pseudo-distributed: 1                                           │
│   → Production: 3 (survives any single node failure)                │
│   → Per-file override: hdfs dfs -setrep N /path (Project 3)        │
│                                                                      │
│ dfs.blocksize                                                        │
│   → Default block size for new files                                │
│   → DEFAULT: 134217728 (128MB)                                      │
│   → Large sequential files: 268435456 (256MB)                       │
│   → Per-file override: hdfs dfs -D dfs.blocksize=X -put (Project 2)│
│                                                                      │
│ dfs.namenode.name.dir                                                │
│   → Where NameNode stores fsimage and edits                         │
│   → CRITICAL: This is the MOST IMPORTANT data in the cluster       │
│   → Loss of this = loss of ALL file metadata                        │
│   → PRODUCTION: Set to MULTIPLE directories on different disks      │
│   → Example: /disk1/nn,/disk2/nn (mirror for redundancy)           │
│                                                                      │
│ dfs.datanode.data.dir                                                │
│   → Where DataNode stores actual blocks                             │
│   → Can be MULTIPLE directories (across multiple disks)             │
│   → DataNode stripes blocks across all directories                  │
│   → Example: /disk1/dn,/disk2/dn,/disk3/dn                        │
│   → More disks = more parallel I/O = better performance             │
│                                                                      │
│ dfs.namenode.handler.count                                           │
│   → Number of threads handling RPC requests to NameNode             │
│   → DEFAULT: 10                                                      │
│   → PRODUCTION: 20 × ln(number_of_cluster_nodes)                   │
│   → 20-node cluster: ~60 handlers                                   │
│   → Too few = RPC timeouts under load                               │
│                                                                      │
│ dfs.namenode.heartbeat.recheck-interval                              │
│   → How often NameNode rechecks for dead DataNodes                  │
│   → DEFAULT: 300000 (5 minutes)                                     │
│   → Combined with heartbeat interval (3s) for dead detection        │
│   → From Project 2: dead detection = 2×recheck + 10×heartbeat      │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────┐
│ FILE 3: mapred-site.xml                                              │
│ PURPOSE: MapReduce-specific settings                                 │
│ AFFECTS: MapReduce jobs, ApplicationMaster, task containers          │
│                                                                      │
│ KEY PROPERTIES:                                                      │
│                                                                      │
│ mapreduce.framework.name                                             │
│   → Execution framework for MapReduce                               │
│   → "local" = Local mode (single JVM, no YARN)                     │
│   → "yarn" = Run on YARN (pseudo or fully distributed)             │
│   → THIS IS THE SWITCH between local and distributed mode          │
│                                                                      │
│ mapreduce.map.memory.mb                                              │
│   → Memory allocated to each mapper CONTAINER                       │
│   → DEFAULT: 1024 (1GB)                                             │
│   → Container memory = JVM heap + overhead                          │
│   → If task exceeds this: YARN KILLS the container                  │
│                                                                      │
│ mapreduce.reduce.memory.mb                                           │
│   → Memory allocated to each reducer CONTAINER                      │
│   → DEFAULT: 1024 (1GB)                                             │
│   → Reducers often need more than mappers (merge-sort phase)        │
│                                                                      │
│ mapreduce.map.java.opts                                              │
│   → JVM options for mapper child process                            │
│   → MUST set -Xmx to ~80% of container memory                      │
│   → Container = 1024MB → -Xmx819m                                  │
│   → Remaining 20% = JVM overhead, native code, buffers             │
│   → If -Xmx > container memory: container killed by YARN           │
│                                                                      │
│ mapreduce.reduce.java.opts                                           │
│   → JVM options for reducer child process                           │
│   → Same 80% rule as mappers                                        │
│                                                                      │
│ mapreduce.task.io.sort.mb                                            │
│   → Mapper's sort buffer size (circular buffer from Project 5)      │
│   → DEFAULT: 100                                                     │
│   → Must be LESS than mapper JVM heap (-Xmx)                       │
│   → Larger = fewer spills = better performance                      │
│   → From Project 7: Spilled Records >> output = buffer too small    │
│                                                                      │
│ mapreduce.map.sort.spill.percent                                     │
│   → Buffer fullness threshold triggering spill to disk              │
│   → DEFAULT: 0.80 (80%)                                             │
│   → Higher = more buffer utilization but risk of blocking           │
│                                                                      │
│ mapreduce.job.reduces                                                │
│   → Default number of reduce tasks                                  │
│   → Can be overridden per job: job.setNumReduceTasks(N)             │
│   → Production rule: 0.95 × total_reduce_slots                     │
│                                                                      │
│ mapreduce.map.maxattempts                                            │
│   → Maximum retry attempts for failed map tasks                     │
│   → DEFAULT: 4                                                       │
│   → From Project 7: Check task attempt count for flaky tasks        │
│                                                                      │
│ mapreduce.reduce.maxattempts                                         │
│   → Maximum retry attempts for failed reduce tasks                  │
│   → DEFAULT: 4                                                       │
│                                                                      │
│ mapreduce.reduce.shuffle.parallelcopies                              │
│   → Number of parallel HTTP fetches during shuffle                  │
│   → DEFAULT: 5                                                       │
│   → More copies = faster shuffle but more network connections       │
│                                                                      │
│ mapreduce.task.timeout                                                │
│   → Milliseconds before a non-responsive task is killed             │
│   → DEFAULT: 600000 (10 minutes)                                    │
│   → If task doesn't report progress for 10 min: killed             │
│   → Increase for tasks with long processing per record              │
│                                                                      │

✅ WHAT SHOULD DO

1. ALWAYS test in LOCAL mode first
   → Catches logic bugs without cluster overhead
   → Runs in seconds, not minutes
   → Full Java debugging support (breakpoints, step-through)
   → Then promote to pseudo-distributed for integration testing

2. ALWAYS keep separate config directories per environment
   → /etc/hadoop/conf.local/    (local mode configs)
   → /etc/hadoop/conf.pseudo/   (pseudo-distributed configs)
   → /etc/hadoop/conf.prod/     (production configs)
   → Switch: export HADOOP_CONF_DIR=/etc/hadoop/conf.local/

3. SET hadoop.tmp.dir to a PERSISTENT directory
   → DEFAULT /tmp/ is cleared on reboot
   → Loss of tmp = loss of NameNode metadata if not backed up elsewhere
   → Use: /opt/hadoop/tmp or /data/hadoop/tmp

4. SIZE containers based on machine resources
   → Leave 20% RAM for OS + daemons
   → Mapper containers: 1-4GB (depends on workload)
   → Reducer containers: 2-8GB (larger for heavy shuffle)
   → JVM heap = 80% of container memory
   → NEVER set JVM heap > container memory (instant kill)

5. SET io.file.buffer.size to 128KB in production
   → Default 4KB is too small for big data workloads
   → 128KB reduces system call overhead significantly
   → Minimal memory impact (one buffer per open file)

6. ENABLE log aggregation from the start
   → yarn.log-aggregation-enable = true
   → Essential for post-mortem debugging (Project 7)
   → Logs collected to HDFS after job completion
   → Set retention: yarn.log-aggregation.retain-seconds

7. CONFIGURE environment files properly
   → hadoop-env.sh: JAVA_HOME, HADOOP heap sizes
   → yarn-env.sh: YARN daemon heap sizes
   → mapred-env.sh: MapReduce daemon heap sizes

🚫 WHAT SHOULD NOT DO

1. DO NOT use production config on development machines
   → Production config expects: real cluster, multiple nodes, 64GB+ RAM
   → Laptop with 16GB: containers won't launch (insufficient memory)
   → NameNode tries to contact non-existent DataNodes
   → ALWAYS maintain environment-specific configs

2. DO NOT set JVM heap (-Xmx) larger than container memory
   → Container = 1024MB, -Xmx = 1024MB → KILLED by YARN
   → The 20% overhead is NOT optional
   → YARN monitors physical memory and kills violators

3. DO NOT leave hadoop.tmp.dir as /tmp
   → OS may clean /tmp on reboot or periodically
   → NameNode metadata lost → HDFS becomes inaccessible
   → DataNode block references lost → blocks become orphaned
   → ALWAYS set to dedicated persistent directory

4. DO NOT skip the workers file in distributed mode
   → $HADOOP_HOME/etc/hadoop/workers lists all worker nodes
   → Without it: start-dfs.sh only starts local DataNode
   → Without it: start-yarn.sh only starts local NodeManager
   → In pseudo-distributed: "localhost" is sufficient

5. DO NOT forget to format NameNode after CHANGING namenode dir
   → If you change dfs.namenode.name.dir → old metadata not found
   → Must format NameNode (first time only for new directory)
   → NEVER format on running production cluster (erases all metadata)

6. DO NOT allocate ALL machine memory to YARN
   → OS needs ~2-4GB for filesystem cache, networking, etc.
   → Hadoop daemons need 1-4GB each (NameNode, DataNode, NodeManager)
   → Starving OS = swapping = catastrophic performance collapse
   → RULE: YARN gets at most 80% of physical RAM

7. DO NOT use the same ports across environments
   → Development and production on same network?
   → Use different ports: dev NameNode on 9000, prod on 8020
   → Prevents accidental connections to wrong cluster

🚫 WHEN NOT TO USE

1. DO NOT use LOCAL mode for testing shuffle-dependent logic
   → Local mode has NO shuffle — map output goes directly to reducer
   → Serialization/deserialization of Writables is SKIPPED
   → Custom partitioners, combiners, sort comparators NOT exercised
   → Bugs that only appear during real shuffle will be missed

2. DO NOT use PSEUDO-DISTRIBUTED for performance testing
   → One machine = all daemons compete for CPU/memory/disk
   → No real network latency (localhost)
   → No rack awareness
   → Performance numbers are MEANINGLESS for production sizing

3. DO NOT use FULLY DISTRIBUTED for quick development iterations
   → Minutes to deploy, start, run, check results
   → Local mode: seconds for the same
   → Developer productivity drops dramatically
   → Use cluster only for final integration testing

4. DO NOT use default configurations in production
   → Defaults are optimized for development/testing
   → Default memory too small (1GB containers)
   → Default replication too high for single node (3)
   → Default handler count too low for large clusters (10)
   → ALWAYS customize for your hardware and workload

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "Container killed by YARN — exceeding memory limits"
→ CAUSE: JVM heap (-Xmx) set equal to or larger than container memory
→ JVM uses heap + non-heap (class metadata, thread stacks, JNI, etc.)
→ Total JVM memory > container limit → YARN kills it
→ FIX: Set -Xmx to 80% of container memory
→ EXAMPLE:
   Container: 2048MB
   -Xmx: 1638m (80% of 2048)
   Remaining 410MB: JVM overhead

PITFALL 2: "Virtual memory exceeded — container killed"
→ CAUSE: Java's virtual memory reservation is ~2-3× physical
→ Default vmem-pmem-ratio is 2.1
→ Container: 2048MB physical → limit: 4300MB virtual
→ JVM may reserve more virtual than this threshold
→ FIX OPTION 1: Increase yarn.nodemanager.vmem-pmem-ratio to 4.0
→ FIX OPTION 2: Disable check: yarn.nodemanager.vmem-check-enabled=false
→ Most production clusters disable virtual memory checking

PITFALL 3: "NameNode fails to start after reboot"
→ CAUSE: hadoop.tmp.dir was /tmp, cleaned on reboot
→ NameNode's fsimage and edits were in /tmp/hadoop-*/dfs/name
→ After reboot: /tmp is empty → NameNode finds no metadata
→ FIX: Set hadoop.tmp.dir to persistent directory
→ BETTER: Set dfs.namenode.name.dir explicitly (not under tmp)

PITFALL 4: "Jobs stuck in ACCEPTED state, never start running"
→ CAUSE: YARN cannot allocate containers
→ Check 1: yarn.nodemanager.resource.memory-mb too small
→ Check 2: Container request exceeds maximum-allocation-mb
→ Check 3: NodeManager not running (jps shows no NodeManager)
→ Check 4: Queue capacity exhausted (multi-tenant — Project 39)
→ DEBUG: ResourceManager UI → Scheduler page → check available resources

PITFALL 5: "MapReduce works in local mode but fails in distributed"
→ CAUSE 1: Custom Writable not implementing serialization correctly
  → Local mode skips serialization (same JVM)
  → Distributed mode serializes for network → bugs exposed
→ CAUSE 2: Input paths using local filesystem paths (file:///...)
  → Distributed mode looks for paths in HDFS
→ CAUSE 3: Classes not in JAR file
  → Local mode finds classes on local classpath
  → Distributed mode only has classes in the submitted JAR

PITFALL 6: "DataNode refuses to start — Incompatible clusterID"
→ CAUSE: NameNode was reformatted but DataNode still has old clusterID
→ Format creates a new cluster ID → DataNode's ID doesn't match
→ FIX: Delete DataNode's data directory and restart
  rm -rf /opt/hadoop/data/datanode/*
  (This deletes all stored blocks — only do on development!)
→ PREVENTION: Never format NameNode unless setting up fresh cluster

def run_cmd(cmd):
    result = subprocess.run(cmd, shell=True, capture_output=True, text=True)
    return result.returncode, result.stdout.strip(), result.stderr.strip()

def parse_hadoop_xml(filepath):
    """Parse a Hadoop XML configuration file into a dictionary."""
    properties = {}
    try:
        tree = ET.parse(filepath)
        root = tree.getroot()
        for prop in root.findall('property'):
            name = prop.find('name')
            value = prop.find('value')
            desc = prop.find('description')
            if name is not None and value is not None:
                properties[name.text] = {
                    'value': value.text if value.text else '',
                    'description': desc.text if desc is not None and desc.text else ''
                }
    except Exception as e:
        print(f"  Error parsing {filepath}: {e}")
    return properties

def format_bytes(b):
    """Format bytes into human-readable string."""
    if b >= 1024*1024*1024:
        return f"{b/(1024*1024*1024):.1f} GB"
    elif b >= 1024*1024:
        return f"{b/(1024*1024):.0f} MB"
    elif b >= 1024:
        return f"{b/1024:.0f} KB"
    return f"{b} B"

def analyze_config_set(config_dir, label):
    """Analyze a complete Hadoop configuration set."""
    
    print(f"\n{'='*70}")
    print(f"  CONFIGURATION ANALYSIS: {label}")
    print(f"  Directory: {config_dir}")
    print(f"{'='*70}")
    
    config_files = [
        ('core-site.xml', 'Global Settings'),
        ('hdfs-site.xml', 'HDFS Settings'),
        ('mapred-site.xml', 'MapReduce Settings'),
        ('yarn-site.xml', 'YARN Settings')
    ]
    
    all_properties = {}
    
    for filename, section_name in config_files:
        filepath = os.path.join(config_dir, filename)
        
        if not os.path.exists(filepath):
            print(f"\n  [{section_name}] — {filename}: NOT PRESENT (using defaults)")
            continue
        
        properties = parse_hadoop_xml(filepath)
        all_properties[filename] = properties
        
        print(f"\n  ┌─── {section_name} ({filename}) ───")
        print(f"  │ Properties defined: {len(properties)}")
        print(f"  │")
        
        for name, prop in sorted(properties.items()):
            value = prop['value']
            # Truncate long values
            display_value = value[:50] + "..." if len(value) > 50 else value
            print(f"  │ {name:<45} = {display_value}")
        
        print(f"  └{'─'*65}")
    
    return all_properties

def validate_configuration(all_properties, mode_label):
    """Validate configuration for common issues."""
    
    print(f"\n  ┌─── VALIDATION CHECKS ─────────────────────────────────────────┐")
    
    issues = []
    warnings = []
    healthy = []
    
    # Flatten all properties
    flat = {}
    for filename, props in all_properties.items():
        for name, prop in props.items():
            flat[name] = prop['value']
    
    # ── Check 1: Filesystem ──
    fs_default = flat.get('fs.defaultFS', 'file:///')
    if 'hdfs://' in fs_default:
        healthy.append(f"fs.defaultFS = {fs_default} (HDFS mode)")
    elif 'file:///' in fs_default:
        healthy.append(f"fs.defaultFS = {fs_default} (Local mode)")
    else:
        warnings.append(f"fs.defaultFS = {fs_default} (unusual filesystem)")
    
    # ── Check 2: MapReduce framework ──
    mr_framework = flat.get('mapreduce.framework.name', 'local')
    if mr_framework == 'yarn' and 'file:///' in fs_default:
        issues.append("YARN framework with local filesystem — inconsistent")
    elif mr_framework == 'local' and 'hdfs://' in fs_default:
        warnings.append("Local framework with HDFS — MapReduce won't use YARN")
    else:
        healthy.append(f"mapreduce.framework.name = {mr_framework}")
    
    # ── Check 3: Temp directory ──
    tmp_dir = flat.get('hadoop.tmp.dir', '/tmp/hadoop-${user.name}')
    if '/tmp/' in tmp_dir or tmp_dir.startswith('/tmp'):
        issues.append(f"hadoop.tmp.dir = {tmp_dir} — WILL BE LOST ON REBOOT!")
    else:
        healthy.append(f"hadoop.tmp.dir = {tmp_dir} (persistent)")
    
    # ── Check 4: Memory configuration ──
    map_mem_str = flat.get('mapreduce.map.memory.mb', '1024')
    reduce_mem_str = flat.get('mapreduce.reduce.memory.mb', '1024')
    map_opts = flat.get('mapreduce.map.java.opts', '-Xmx819m')
    reduce_opts = flat.get('mapreduce.reduce.java.opts', '-Xmx819m')
    yarn_mem_str = flat.get('yarn.nodemanager.resource.memory-mb', '8192')
    
    try:
        map_mem = int(map_mem_str)
        reduce_mem = int(reduce_mem_str)
        yarn_mem = int(yarn_mem_str)
        
        # Extract -Xmx value
        import re
        map_xmx_match = re.search(r'-Xmx(\d+)m', map_opts)
        reduce_xmx_match = re.search(r'-Xmx(\d+)m', reduce_opts)
        
        if map_xmx_match:
            map_xmx = int(map_xmx_match.group(1))
            map_ratio = map_xmx / map_mem * 100
            
            if map_ratio > 85:
                issues.append(f"Mapper -Xmx={map_xmx}m is {map_ratio:.0f}% of container {map_mem}MB")
                issues.append(f"  → Risk of YARN killing container (recommend 80%)")
            elif map_ratio < 60:
                warnings.append(f"Mapper -Xmx={map_xmx}m is only {map_ratio:.0f}% of container {map_mem}MB")
                warnings.append(f"  → Wasting {map_mem - map_xmx}MB per mapper")
            else:
                healthy.append(f"Mapper memory: container={map_mem}MB, heap={map_xmx}MB ({map_ratio:.0f}%)")
        
        if reduce_xmx_match:
            reduce_xmx = int(reduce_xmx_match.group(1))
            reduce_ratio = reduce_xmx / reduce_mem * 100
            
            if reduce_ratio > 85:
                issues.append(f"Reducer -Xmx={reduce_xmx}m is {reduce_ratio:.0f}% of container {reduce_mem}MB")
            else:
                healthy.append(f"Reducer memory: container={reduce_mem}MB, heap={reduce_xmx}MB ({reduce_ratio:.0f}%)")
        
        # Check containers fit in YARN allocation
        if map_mem > yarn_mem:
            issues.append(f"Mapper container ({map_mem}MB) > YARN total ({yarn_mem}MB)")
        if reduce_mem > yarn_mem:
            issues.append(f"Reducer container ({reduce_mem}MB) > YARN total ({yarn_mem}MB)")
        
        # Estimate max containers per node
        max_map_containers = yarn_mem // map_mem
        max_reduce_containers = yarn_mem // reduce_mem
        healthy.append(f"Max mappers per node: {max_map_containers} (YARN={yarn_mem}MB / mapper={map_mem}MB)")
        healthy.append(f"Max reducers per node: {max_reduce_containers} (YARN={yarn_mem}MB / reducer={reduce_mem}MB)")
        
    except (ValueError, TypeError):
        warnings.append("Could not parse memory values for validation")
    
    # ── Check 5: Sort buffer ──
    sort_mb_str = flat.get('mapreduce.task.io.sort.mb', '100')
    try:
        sort_mb = int(sort_mb_str)
        if map_xmx_match:
            map_xmx = int(map_xmx_match.group(1))
            if sort_mb >= map_xmx:
                issues.append(f"Sort buffer ({sort_mb}MB) >= mapper heap ({map_xmx}MB) — OOM risk!")
            elif sort_mb > map_xmx * 0.5:
                warnings.append(f"Sort buffer ({sort_mb}MB) is {sort_mb/map_xmx*100:.0f}% of heap — leaves little for app")
            else:
                healthy.append(f"Sort buffer: {sort_mb}MB ({sort_mb/map_xmx*100:.0f}% of heap)")
    except (ValueError, NameError):
        pass
    
    # ── Check 6: Replication ──
    replication = flat.get('dfs.replication', '3')
    if replication == '1' and 'production' in mode_label.lower():
        issues.append(f"Replication = 1 in production — NO FAULT TOLERANCE!")
    elif replication == '3' and 'local' in mode_label.lower():
        warnings.append(f"Replication = 3 but only 1 DataNode — under-replicated blocks")
    else:
        healthy.append(f"Replication factor: {replication}")
    
    # ── Check 7: Virtual memory check ──
    vmem_check = flat.get('yarn.nodemanager.vmem-check-enabled', 'true')
    if vmem_check == 'true':
        warnings.append("Virtual memory check ENABLED — may cause false container kills")
        warnings.append("  Consider: yarn.nodemanager.vmem-check-enabled=false")
    else:
        healthy.append("Virtual memory check: disabled (avoids false kills)")
    
    # ── Check 8: Log aggregation ──
    log_agg = flat.get('yarn.log-aggregation-enable', 'false')
    if log_agg == 'false':
        warnings.append("Log aggregation DISABLED — cannot use 'yarn logs' command")
    else:
        healthy.append("Log aggregation: enabled")
    
    # ── Print results ──
    if issues:
        print(f"  │")
        print(f"  │ 🔴 ISSUES ({len(issues)}):")
        for issue in issues:
            print(f"  │    {issue}")
    
    if warnings:
        print(f"  │")
        print(f"  │ 🟡 WARNINGS ({len(warnings)}):")
        for warning in warnings:
            print(f"  │    {warning}")
    
    if healthy:
        print(f"  │")
        print(f"  │ 🟢 HEALTHY ({len(healthy)}):")
        for item in healthy:
            print(f"  │    {item}")
    
    print(f"  │")
    
    # Overall assessment
    if len(issues) == 0:
        print(f"  │ OVERALL: ✅ Configuration is valid for {mode_label}")
    elif len(issues) <= 2:
        print(f"  │ OVERALL: ⚠️  Configuration has issues — review before deploying")
    else:
        print(f"  │ OVERALL: 🔴 Configuration has critical issues — DO NOT DEPLOY")
    
    print(f"  └{'─'*65}")
    
    return issues, warnings, healthy

def compare_configurations(configs):
    """Compare properties across multiple configuration sets."""
    
    print(f"\n{'='*70}")
    print(f"  CONFIGURATION COMPARISON ACROSS ENVIRONMENTS")
    print(f"{'='*70}")
    
    # Collect all unique properties
    all_props = set()
    for label, config in configs.items():
        for filename, props in config.items():
            for name in props:
                all_props.add(name)
    
    # Key properties to compare
    key_properties = [
        'fs.defaultFS',
        'mapreduce.framework.name',
        'hadoop.tmp.dir',
        'dfs.replication',
        'dfs.blocksize',
        'mapreduce.map.memory.mb',
        'mapreduce.reduce.memory.mb',
        'mapreduce.map.java.opts',
        'mapreduce.reduce.java.opts',
        'mapreduce.task.io.sort.mb',
        'yarn.nodemanager.resource.memory-mb',
        'yarn.nodemanager.resource.cpu-vcores',
        'yarn.nodemanager.vmem-check-enabled',
        'yarn.log-aggregation-enable',
        'io.file.buffer.size',
        'fs.trash.interval',
        'dfs.namenode.handler.count',
    ]
    
    labels = list(configs.keys())
    
    # Header
    header = f"  {'Property':<40}"
    for label in labels:
        header += f" {label:<18}"
    print(f"\n{header}")
    print(f"  {'─'*40}" + f" {'─'*18}" * len(labels))
    
    for prop in key_properties:
        row = f"  {prop:<40}"
        for label in labels:
            value = "(default)"
            for filename, props in configs[label].items():
                if prop in props:
                    value = props[prop]['value']
                    # Truncate
                    if len(value) > 16:
                        value = value[:13] + "..."
                    break
            row += f" {value:<18}"
        print(row)
    
    print(f"  {'─'*40}" + f" {'─'*18}" * len(labels))

def production_sizing_calculator():
    """Interactive production cluster sizing guide."""
    
    print(f"\n{'='*70}")
    print(f"  PRODUCTION CLUSTER SIZING CALCULATOR")
    print(f"{'='*70}")
    
    # Preset scenarios
    scenarios = [
        {
            "name": "Small Cluster (5 nodes)",
            "nodes": 5, "ram_gb": 32, "cores": 8, "disks": 6, "disk_tb": 2
        },
        {
            "name": "Medium Cluster (20 nodes)",
            "nodes": 20, "ram_gb": 64, "cores": 16, "disks": 12, "disk_tb": 4
        },
        {
            "name": "Large Cluster (100 nodes)",
            "nodes": 100, "ram_gb": 128, "cores": 32, "disks": 24, "disk_tb": 8
        },
    ]
    
    for scenario in scenarios:
        name = scenario["name"]
        nodes = scenario["nodes"]
        ram_gb = scenario["ram_gb"]
        cores = scenario["cores"]
        disks = scenario["disks"]
        disk_tb = scenario["disk_tb"]
        
        print(f"\n  ┌─── {name} ─────────────────────────────────────────┐")
        print(f"  │ Hardware: {nodes} nodes × {ram_gb}GB RAM × {cores} cores × {disks}×{disk_tb}TB")
        print(f"  │")
        
        # Memory allocation
        os_reserved_gb = max(2, int(ram_gb * 0.1))
        daemon_gb = 4  # NameNode/DataNode/NodeManager
        yarn_gb = ram_gb - os_reserved_gb - daemon_gb
        yarn_mb = yarn_gb * 1024
        
        # Container sizing
        mapper_mb = min(4096, max(1024, yarn_mb // (cores * 2)))
        # Round to nearest 512
        mapper_mb = ((mapper_mb + 255) // 512) * 512
        reducer_mb = mapper_mb * 2
        
        mapper_xmx = int(mapper_mb * 0.8)
        reducer_xmx = int(reducer_mb * 0.8)
        sort_mb = min(512, int(mapper_xmx * 0.3))
        
        max_mappers_per_node = yarn_mb // mapper_mb
        max_reducers_per_node = yarn_mb // reducer_mb
        
        # Storage
        raw_storage_tb = nodes * disks * disk_tb
        usable_storage_tb = raw_storage_tb / 3  # replication factor 3
        
        # NameNode sizing
        estimated_blocks = int(usable_storage_tb * 1024 * 1024 / 256)  # 256MB blocks
        nn_heap_gb = max(2, int(estimated_blocks * 150 / (1024 * 1024 * 1024)) + 2)
        
        # Handler count
        handler_count = max(10, int(20 * math.log(nodes)))
        
        print(f"  │ MEMORY ALLOCATION PER NODE:")
        print(f"  │   Total RAM:           {ram_gb} GB")
        print(f"  │   OS reserved:         {os_reserved_gb} GB")
        print(f"  │   Daemon reserved:     {daemon_gb} GB")

        print(f"  │")
        print(f"  │ CONTAINER SIZING:")
        print(f"  │   Mapper container:    {mapper_mb} MB (heap: {mapper_xmx}m)")
        print(f"  │   Reducer container:   {reducer_mb} MB (heap: {reducer_xmx}m)")
        print(f"  │   Sort buffer:         {sort_mb} MB")
        print(f"  │   Max mappers/node:    {max_mappers_per_node}")
        print(f"  │   Max reducers/node:   {max_reducers_per_node}")
        print(f"  │   Total mapper slots:  {max_mappers_per_node * nodes}")
        print(f"  │")
