PROJECT 11: CLEANING MESSY HOSPITAL RECORDS BEFORE ANALYSIS

🔴 HYPOTHETICAL PROBLEM

SCENARIO:
→ You are a data engineer at a national healthcare analytics firm
→ 200 hospitals submit patient visit records daily via CSV uploads
→ Each hospital has DIFFERENT data quality standards
→ The raw data looks like this:

GOOD RECORD:
  PAT_001,John Smith,1985-03-15,M,Cardiology,2024-01-10,Completed,450.00

BAD RECORDS (real examples from the feed):
  PAT_002,,1990-07-22,F,Neurology,2024-01-10,Completed,300.00          ← Missing name
  PAT_003,Jane Doe,13/25/1975,F,Orthopedics,2024-01-11,Completed,275  ← Invalid date format
  PAT_004,Bob Lee,1978-11-30,X,Cardiology,2024-01-11,Pending,0.00     ← Invalid gender 'X'
  ,,,,,,,,                                                               ← Completely empty
  PAT_005,Alice Wu,1992-04-18,F,,2024-01-12,Completed,500.00           ← Missing department
  PAT_001,John Smith,1985-03-15,M,Cardiology,2024-01-10,Completed,450  ← DUPLICATE of first

→ 40% of records have at least ONE quality issue
→ Downstream analytics (aggregation, ML models) FAIL on dirty data
→ REQUIREMENT: Clean the data BEFORE any analysis

THE KEY INSIGHT:
→ This job needs NO aggregation
→ No counting, no summing, no grouping
→ Each record is evaluated INDEPENDENTLY
→ Clean records pass through, dirty records are rejected
→ This is a MAP-ONLY job — no reducer needed

🔍 HOW TO APPROACH IT

STEP 1: Understand WHY a reducer is unnecessary

In Projects 4-10, every job had this flow:
  MAP → SHUFFLE → SORT → REDUCE

But consider what happens in data cleaning:
  → Read a record
  → Check if it's valid
  → If valid → write it out
  → If invalid → discard or write to error file
  → NO interaction between records needed
  → Record #5000 doesn't affect Record #5001

When there is NO NEED to:
  → Group records by a common key
  → Aggregate values across records
  → Compare one record against another
...then the SHUFFLE-SORT-REDUCE pipeline is PURE WASTE:

STEP 2: Understand what changes technically

  job.setNumReduceTasks(0);   ← THIS SINGLE LINE

When numReduceTasks = 0:
  → Hadoop SKIPS shuffle phase entirely
  → Mapper output goes DIRECTLY to HDFS
  → Output files named: part-m-00000 (not part-r-00000)
  →                      ↑ 'm' for MAP    ↑ 'r' for REDUCE
  → Number of output files = number of mappers (one per input split)
  → No sorting of output (mapper output order preserved)
  → No network transfer between nodes

STEP 3: Understand NullWritable

In Projects 4-5, every mapper emitted (key, value).
But in a cleaning job:
  → We don't NEED a key — there's nothing to group by
  → We just want the clean record written as-is

NullWritable is a special Hadoop type:
  → Occupies ZERO bytes when serialized
  → Represents "no value" for key or value position
  → Used when one side of the key-value pair is meaningless

  context.write(NullWritable.get(), cleanRecord);
  → Output: just the clean record, no tab-separated key prefix

✅ WHAT COULD BE DONE

APPROACH A: Map-only job with validation in mapper (OUR CHOICE)
→ Zero reducers
→ All logic in mapper
→ Fastest possible execution
→ Error tracking via counters (Project 7 concept)

APPROACH B: Map-only job writing to TWO outputs
→ Clean records → /output/clean/
→ Rejected records → /output/rejected/
→ Uses MultipleOutputs (covered in Project 20)
→ More sophisticated but adds complexity

APPROACH C: Mapper + Reducer for deduplication
→ Mapper emits key=PATIENT_ID+VISIT_DATE
→ Reducer sees duplicates grouped together → keeps first
→ Handles cross-mapper duplicates
→ But adds shuffle overhead for ONE validation rule
→ Overkill if duplicates are rare

WE CHOOSE APPROACH A because:
→ Simplest demonstration of map-only pattern
→ Covers 90% of cleaning scenarios
→ Deduplication within same mapper's split is handled
→ Cross-split deduplication is rare in practice
   (same record unlikely to appear in different hospital files)
→ Approach B and C build on this foundation (Projects 20, 17)

✅ WHAT SHOULD DO

1. USE job.setNumReduceTasks(0) — THE DEFINING LINE
   → This is what makes it a map-only job
   → Without this, Hadoop creates default 1 reducer
   → That 1 reducer becomes bottleneck for no reason

2. USE NullWritable FOR UNUSED KEY OR VALUE
   → context.write(NullWritable.get(), cleanRecord)
   → Output has no key prefix — just the clean data
   → If you used Text("") as key, output would have
     a leading TAB character on every line

3. USE COUNTERS EXTENSIVELY FOR DATA QUALITY REPORTING
   → From Project 7: counters are aggregated across ALL mappers
   → Define one counter per validation rule
   → After job: counter report = complete data quality audit
   → This is how production ETL tracks data health

4. VALIDATE EARLY, REJECT FAST
   → Check cheapest validations first (empty check before regex)
   → Once a record fails ANY rule, skip remaining checks
   → Don't waste CPU on records already known to be bad

5. STANDARDIZE DATA FORMAT DURING CLEANING
   → Trim whitespace from all fields
   → Uppercase codes (gender, status)
   → Normalize date formats to ISO 8601 (YYYY-MM-DD)
   → Cleaning = validation + normalization

6. PRESERVE THE ORIGINAL SCHEMA
   → Output should have same columns as input
   → Don't add/remove fields during cleaning
   → Downstream jobs expect consistent schema

🚫 WHAT SHOULD NOT DO

1. DO NOT ADD A REDUCER "JUST IN CASE"
   → Even 1 reducer means ALL data shuffled to ONE node
   → For 50GB input: 50GB transferred over network for nothing
   → Map-only: each mapper writes locally → zero network cost

2. DO NOT USE Text AS KEY WHEN KEY IS MEANINGLESS
   → context.write(new Text(""), value) → adds TAB to every line
   → context.write(new Text("clean"), value) → wastes space
   → NullWritable.get() → zero bytes, clean output

3. DO NOT SWALLOW ERRORS SILENTLY
   → BAD: if (invalid) { return; }  ← record vanishes, nobody knows
   → GOOD: if (invalid) { 
             context.getCounter("Errors","MISSING_NAME").increment(1);
             return;
           }
   → Counters provide audit trail

4. DO NOT LOAD ENTIRE FILE INTO MEMORY FOR DEDUP
   → HashSet of all patient IDs could exhaust mapper heap
   → For within-split dedup: bounded LRU cache (last N IDs)
   → For global dedup: use a separate MapReduce job with reducers
   → Or accept within-split dedup as "good enough"

5. DO NOT ASSUME INPUT ENCODING
   → Hospital systems may send UTF-8, Latin-1, or Windows-1252
   → Special characters in names (José, Müller, O'Brien)
   → Handle encoding explicitly or data corrupts silently

6. DO NOT MIX CLEANING WITH ANALYSIS
   → Cleaning job should ONLY clean — no aggregations
   → Analysis runs on clean output as a separate job
   → Separation of concerns → easier debugging
   → If analysis fails, you know it's not a data quality issue
   → This is the foundation of ETL pipeline thinking (Project 21)

DECISION RULE:
→ Does any record depend on another record? NO → Map-Only
→ Do you need to group/aggregate?           NO → Map-Only
→ Do you need sorted output?                NO → Map-Only
→ Is it row-by-row transformation?          YES → Map-Only

🚫 WHEN NOT TO USE

MAP-ONLY JOBS ARE WRONG WHEN:

1. YOU NEED GLOBAL DEDUPLICATION
   → Duplicate records may exist in DIFFERENT input splits
   → Mapper 1 sees record A, Mapper 5 also sees record A
   → Neither mapper knows about the other's copy
   → Need reducer to group by key and eliminate duplicates

2. YOU NEED SORTED OUTPUT
   → Map-only output preserves INPUT order (within each split)
   → No global sorting happens
   → If downstream needs sorted data → need reducer
   → (Or use TotalOrderPartitioner in Project 18)

3. YOU NEED CROSS-RECORD VALIDATION
   → "Patient cannot have two visits on the same day"
   → Requires seeing ALL records for a patient together
   → That means grouping by patient_id → needs reducer

4. OUTPUT MUST BE A SPECIFIC NUMBER OF FILES
   → Map-only produces one file PER INPUT SPLIT
   → 1000 input splits → 1000 output files
   → Cannot control output file count without reducer
   → With reducer: numReduceTasks controls output file count
   → (Project 10 demonstrated this)

5. YOU NEED AGGREGATED SUMMARY ALONGSIDE CLEANING
   → "Clean data AND produce count by department"
   → Aggregation requires reducer
   → Better: Two separate jobs (clean first, aggregate second)
   → Chain them (Project 21)

⚙️ HOW IT ONLY WORKS — INTERNAL MECHANICS

═══════════════════════════════════════════════════════════
MECHANIC 1: WHAT HAPPENS WHEN numReduceTasks = 0
═══════════════════════════════════════════════════════════

NORMAL JOB (numReduceTasks > 0):
  1. InputFormat creates splits
  2. Each split → one mapper task
  3. Mapper output → MapOutputBuffer (circular buffer in memory)
  4. Buffer partitioned by Partitioner (Project 10)
  5. Buffer sorted by key
  6. Combiner runs (Project 9)
  7. Spilled to local disk
  8. Shuffle: reducers PULL data over network
  9. Reduce merge-sorts pulled data
  10. Reducer processes and writes to HDFS

MAP-ONLY JOB (numReduceTasks = 0):
  1. InputFormat creates splits
  2. Each split → one mapper task
  3. Mapper calls context.write(key, value)
  4. Output goes DIRECTLY to HDFS via OutputFormat
  5. DONE

WHAT IS SKIPPED:
  ✗ MapOutputBuffer (no circular buffer needed)
  ✗ Partitioner (no reducers to route to)
  ✗ Sorting (no sort phase)
  ✗ Combiner (runs only as part of sort-spill, which doesn't happen)
  ✗ Spill to local disk (no intermediate files)
  ✗ Shuffle (no network transfer)
  ✗ Reducer (doesn't exist)

PERFORMANCE IMPACT:
  → CPU savings: no sorting
  → Memory savings: no 100MB output buffer
  → Disk savings: no intermediate spill files
  → Network savings: ZERO bytes transferred between nodes
  → Time savings: typically 40-70% faster than equivalent with reducer

═══════════════════════════════════════════════════════════
MECHANIC 2: OUTPUT FILE NAMING IN MAP-ONLY JOBS
═══════════════════════════════════════════════════════════

WITH reducers:
  Output files: part-r-00000, part-r-00001, ...
  → 'r' = reducer output
  → Number = reducer index
  → Count = numReduceTasks

WITHOUT reducers (map-only):
  Output files: part-m-00000, part-m-00001, ...
  → 'm' = mapper output
  → Number = mapper task index
  → Count = number of input splits

EXAMPLE:
  Input: 10GB file on HDFS
  Block size: 128MB
  Number of blocks/splits: ~80
  Map-only output: 80 files (part-m-00000 through part-m-00079)

  This is important because:
  → With reducer: you CONTROL output file count
  → Without reducer: output file count = input split count
  → Could result in many small output files
  → (Small file problem — discussed in Project 2, solved in Project 30)

═══════════════════════════════════════════════════════════
MECHANIC 3: NullWritable INTERNALS
═══════════════════════════════════════════════════════════

NullWritable is a SINGLETON:
  → NullWritable.get() always returns SAME instance
  → No memory allocation per call

Serialization:
  → write(DataOutput out) → writes ZERO bytes
  → readFields(DataInput in) → reads ZERO bytes
  → No disk/network overhead

When used as KEY:
  → Output line has no key prefix
  → No TAB separator before value
  → Output is JUST the value

  context.write(NullWritable.get(), new Text("clean record"))
  → Output file contains: "clean record\n"
  → NOT: "\tclean record\n" (which happens with empty Text key)

When used as VALUE:
  → Output line has only the key
  → Useful when key IS the complete output

  context.write(new Text("patient_id"), NullWritable.get())
  → Output file contains: "patient_id\n"

═══════════════════════════════════════════════════════════
MECHANIC 4: MAPPER LIFECYCLE IN MAP-ONLY JOB
═══════════════════════════════════════════════════════════

  1. JVM starts for mapper task
  2. setup(Context) called ONCE
     → Initialize validators, compile regex patterns
     → Open resources (NOT per-record — once for entire split)
  3. map(key, value, Context) called for EACH record
     → Validate → write or reject
  4. cleanup(Context) called ONCE after all records processed
     → Close resources, flush buffers
  5. OutputCommitter moves temporary output to final HDFS path
  6. JVM may be reused for next mapper task (if JVM reuse enabled)

  setup() is CRITICAL for performance:
  → Compile regex ONCE in setup(), reuse in every map() call
  → DON'T compile regex inside map() — called millions of times
  → DON'T open/close files inside map()

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "Output has TAB character at start of every line"
→ CAUSE: Using new Text("") as key instead of NullWritable
→ TextOutputFormat writes: key + TAB + value
→ Empty Text key = TAB + value
→ FIX: Use NullWritable.get() as key
→ OR: Set output format to suppress key:
      job.setOutputFormatClass(TextOutputFormat.class)
      conf.set("mapreduce.output.textoutputformat.separator", "")

PITFALL 2: "Too many small output files"
→ CAUSE: Map-only job creates one file per input split
→ 10,000 input splits → 10,000 output files
→ Each file might be only a few MB
→ Overwhelms NameNode (Project 2 discussed metadata overhead)
→ FIX OPTIONS:
  a) Use CombineTextInputFormat to merge small input splits
     → Fewer mappers → fewer output files
  b) Run a follow-up job to merge files
  c) Use a single reducer (but loses map-only speed benefit)

PITFALL 3: "Regex validation is extremely slow"
→ CAUSE: Pattern.compile() called INSIDE map() method
→ Called once per record = millions of compilations
→ FIX: Compile pattern ONCE in setup(), store as instance variable
→ 10x-100x speed improvement

PITFALL 4: "Mapper runs out of memory during dedup check"
→ CAUSE: HashSet storing all seen patient IDs grows unbounded
→ 80 million records × 10 bytes per ID = 800MB in HashSet
→ Mapper JVM default heap: 256MB-1GB
→ FIX: Use bounded cache (LRU with max size)
→ OR: Accept that map-only can only dedup ADJACENT records
→ OR: Use a separate MapReduce job with reducers for global dedup

PITFALL 5: "Clean output still has dirty records"
→ CAUSE: Validation logic has bugs
→ DIAGNOSIS: Spot-check output with grep for known bad patterns
→ PREVENTION: Unit test validation logic OUTSIDE Hadoop first
→ Write plain Java/Python tests for each validation rule
→ THEN wrap in mapper (testing covered in Project 38)

PITFALL 6: "Job succeeds but counter shows 0 accepted records"
→ CAUSE: All records failing validation
→ Common reason: wrong delimiter assumption
→ File uses COMMA but code splits on PIPE
→ Every record has "wrong number of fields"
→ PREVENTION: Print first 5 records in setup() for visual check

SCALING NUMBERS FOR OUR HOSPITAL SCENARIO:

Input: 50GB raw hospital records (daily)
Block size: 128MB
Input splits: ~400
Mappers: ~400 (one per split)

On a 50-node cluster:
  → ~8 mapper waves (400 mappers / 50 nodes)
  → Each mapper processes 128MB in ~30 seconds
  → Total wall time: ~4 minutes
  → Network transfer: 0 bytes
  → Output: 400 files (part-m-00000 to part-m-00399)

Same job WITH a reducer (1 reducer):
  → Map phase: same ~4 minutes
  → Shuffle: 50GB transferred to single reducer node
  → At 1Gbps network: ~7 minutes just for transfer
  → Reduce: single thread writes 30GB clean data: ~5 minutes
  → Total wall time: ~16 minutes
  → 4× SLOWER for zero benefit

LESSON: Map-only is not just simpler — it's fundamentally faster
for embarrassingly parallel tasks (where records are independent).

public class HospitalRecordCleaner extends Configured implements Tool {

    // ════════════════════════════════════════════════════════
    // MAPPER: Validate, clean, and emit only good records
    // ════════════════════════════════════════════════════════
    //
    // Input format (CSV with pipe delimiter):
    //   PAT_ID|NAME|DOB|GENDER|DEPARTMENT|VISIT_DATE|STATUS|AMOUNT
    //
    // Output: Same format, but ONLY valid records
    // Key: NullWritable (no key needed)
    // Value: Cleaned record (Text)

    public static class CleaningMapper
            extends Mapper<LongWritable, Text, NullWritable, Text> {

        // ── Error tracking counters ──
        // These aggregate across ALL mapper instances automatically
        // Accessed after job completion for data quality report
        enum CleaningCounters {
            TOTAL_INPUT_RECORDS,
            ACCEPTED_RECORDS,
            REJECTED_EMPTY_LINE,
            REJECTED_HEADER_ROW,
            REJECTED_WRONG_FIELD_COUNT,
            REJECTED_MISSING_PATIENT_ID,
            REJECTED_INVALID_PATIENT_ID,
            REJECTED_MISSING_NAME,
            REJECTED_INVALID_DOB,
            REJECTED_FUTURE_DOB,
            REJECTED_INVALID_GENDER,
            REJECTED_MISSING_DEPARTMENT,
            REJECTED_INVALID_VISIT_DATE,
            REJECTED_INVALID_STATUS,
            REJECTED_INVALID_AMOUNT,
            REJECTED_DUPLICATE_WITHIN_SPLIT
        }

        // ── Instance variables initialized ONCE in setup() ──
        // NOT created per record — critical for performance
        private Pattern patientIdPattern;
        private SimpleDateFormat dateFormat;
        private Set<String> validGenders;
        private Set<String> validStatuses;
        private Set<String> seenRecordKeys;  // For within-split dedup
        private Date today;
        private Text outputValue;

        private static final int EXPECTED_FIELDS = 8;
        private static final int MAX_DEDUP_CACHE_SIZE = 500_000;

        @Override
        protected void setup(Context context) throws IOException {
            // ── CRITICAL: Initialize expensive objects ONCE ──
            // This method runs once per mapper task
            // NOT once per record

            // Compile regex pattern ONCE
            // PAT_ followed by digits: PAT_001, PAT_12345, etc.
            patientIdPattern = Pattern.compile("^PAT_\\d{3,}$");

            // Date parser — NOT thread-safe, but one per mapper is fine
            dateFormat = new SimpleDateFormat("yyyy-MM-dd");
            dateFormat.setLenient(false);  // "2024-02-30" should FAIL

            // Valid code sets
            validGenders = new HashSet<>();
            validGenders.add("M");
            validGenders.add("F");

            validStatuses = new HashSet<>();
            validStatuses.add("COMPLETED");
            validStatuses.add("PENDING");
            validStatuses.add("CANCELLED");

            // Dedup cache — bounded to prevent OOM
            seenRecordKeys = new HashSet<>();

            // Today's date for future-date check
            today = new Date();

            // Reusable output Text object
            outputValue = new Text();
        }

        @Override
        protected void map(LongWritable offset, Text line, Context context)
                throws IOException, InterruptedException {

            context.getCounter(CleaningCounters.TOTAL_INPUT_RECORDS).increment(1);

            String record = line.toString();

            // ═══════════════════════════════════════
            // RULE 0: Skip empty lines
            // ═══════════════════════════════════════
            if (record == null || record.trim().isEmpty()) {
                context.getCounter(CleaningCounters.REJECTED_EMPTY_LINE).increment(1);
                return;
            }

            record = record.trim();

            // ═══════════════════════════════════════
            // RULE 1: Skip header rows
            // ═══════════════════════════════════════
            if (record.startsWith("PAT_ID") || record.startsWith("HEADER")
                    || record.toUpperCase().startsWith("PATIENT")) {
                context.getCounter(CleaningCounters.REJECTED_HEADER_ROW).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 2: Correct number of fields
            // ═══════════════════════════════════════
            String[] fields = record.split("\\|", -1);
            // -1 keeps trailing empty strings: "a|b|" → ["a","b",""]

            if (fields.length != EXPECTED_FIELDS) {
                context.getCounter(CleaningCounters.REJECTED_WRONG_FIELD_COUNT).increment(1);
                return;
            }

            // ── Extract and trim all fields ──
            String patientId   = fields[0].trim();
            String name        = fields[1].trim();
            String dobStr      = fields[2].trim();
            String gender      = fields[3].trim().toUpperCase();
            String department  = fields[4].trim();
            String visitDateStr= fields[5].trim();
            String status      = fields[6].trim().toUpperCase();
            String amountStr   = fields[7].trim();

            // ═══════════════════════════════════════
            // RULE 3: Patient ID validation
            // ═══════════════════════════════════════
            if (patientId.isEmpty()) {
                context.getCounter(CleaningCounters.REJECTED_MISSING_PATIENT_ID).increment(1);
                return;
            }
            if (!patientIdPattern.matcher(patientId).matches()) {
                context.getCounter(CleaningCounters.REJECTED_INVALID_PATIENT_ID).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 4: Name must not be empty
            // ═══════════════════════════════════════
            if (name.isEmpty()) {
                context.getCounter(CleaningCounters.REJECTED_MISSING_NAME).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 5: Date of birth validation
            // ═══════════════════════════════════════
            Date dob;
            try {
                dob = dateFormat.parse(dobStr);
            } catch (ParseException e) {
                context.getCounter(CleaningCounters.REJECTED_INVALID_DOB).increment(1);
                return;
            }
            // Birth date must not be in the future
            if (dob.after(today)) {
                context.getCounter(CleaningCounters.REJECTED_FUTURE_DOB).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 6: Gender validation
            // ═══════════════════════════════════════
            if (!validGenders.contains(gender)) {
                context.getCounter(CleaningCounters.REJECTED_INVALID_GENDER).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 7: Department must not be empty
            // ═══════════════════════════════════════
            if (department.isEmpty()) {
                context.getCounter(CleaningCounters.REJECTED_MISSING_DEPARTMENT).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 8: Visit date validation
            // ═══════════════════════════════════════
            try {
                dateFormat.parse(visitDateStr);
            } catch (ParseException e) {
                context.getCounter(CleaningCounters.REJECTED_INVALID_VISIT_DATE).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 9: Status validation
            // ═══════════════════════════════════════
            if (!validStatuses.contains(status)) {
                context.getCounter(CleaningCounters.REJECTED_INVALID_STATUS).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 10: Amount validation
            // ═══════════════════════════════════════
            double amount;
            try {
                amount = Double.parseDouble(amountStr);
            } catch (NumberFormatException e) {
                context.getCounter(CleaningCounters.REJECTED_INVALID_AMOUNT).increment(1);
                return;
            }
            if (amount < 0) {
                context.getCounter(CleaningCounters.REJECTED_INVALID_AMOUNT).increment(1);
                return;
            }

            // ═══════════════════════════════════════
            // RULE 11: Within-split deduplication
            // ═══════════════════════════════════════
            // Dedup key: patientId + visitDate (same patient, same day = duplicate)
            String dedupKey = patientId + "|" + visitDateStr;

            if (seenRecordKeys.size() < MAX_DEDUP_CACHE_SIZE) {
                if (seenRecordKeys.contains(dedupKey)) {
                    context.getCounter(
                        CleaningCounters.REJECTED_DUPLICATE_WITHIN_SPLIT).increment(1);
                    return;
                }
                seenRecordKeys.add(dedupKey);
            }
            // If cache is full, skip dedup check — don't OOM
            // Imperfect but safe

            // ═══════════════════════════════════════
            // ALL RULES PASSED — Build clean output
            // ═══════════════════════════════════════
            // Reconstruct with standardized fields
            String cleanRecord = String.join("|",
                patientId,
                name,
                dobStr,
                gender,          // Already uppercased
                department,
                visitDateStr,
                status,          // Already uppercased
                String.format("%.2f", amount)  // Normalize to 2 decimal places
            );

            outputValue.set(cleanRecord);

            // ── THE KEY LINE: NullWritable as key ──
            // Output will contain ONLY the clean record
            // No key prefix, no tab separator
            context.write(NullWritable.get(), outputValue);

            context.getCounter(CleaningCounters.ACCEPTED_RECORDS).increment(1);
        }

        @Override
        protected void cleanup(Context context) throws IOException {
            // Release dedup cache memory
            seenRecordKeys.clear();
            seenRecordKeys = null;
        }
    }


    // ════════════════════════════════════════════════════════
    // DRIVER: Configure and launch map-only job
    // ════════════════════════════════════════════════════════

    @Override
    public int run(String[] args) throws Exception {

        if (args.length != 2) {
            System.err.println("Usage: HospitalRecordCleaner <input_path> <output_path>");
            return 1;
        }

        Configuration conf = getConf();
        Job job = Job.getInstance(conf, "Hospital Record Cleaner - Map Only ETL");
        job.setJarByClass(HospitalRecordCleaner.class);

        // ── Set Mapper ──
        job.setMapperClass(CleaningMapper.class);

        // ══════════════════════════════════════════════════
        // THE CRITICAL LINE — ZERO REDUCERS = MAP-ONLY JOB
        // ══════════════════════════════════════════════════
        // Without this: default 1 reducer is created
        // All mapper output shuffled to 1 node for nothing
        // With this: mapper output goes directly to HDFS
        job.setNumReduceTasks(0);

        // ── Output types ──
        // Key: NullWritable (no key in output)
        // Value: Text (the clean record)
        job.setOutputKeyClass(NullWritable.class);
        job.setOutputValueClass(Text.class);

        // NOTE: No setReducerClass() — there IS no reducer
        // NOTE: No setPartitionerClass() — no partitioner without reducer
        //       (Project 10 partitioner is irrelevant here)
        // NOTE: No setCombinerClass() — combiner runs during sort-spill
        //       which doesn't happen in map-only jobs
        //       (Project 9 combiner is irrelevant here)

        // ── Input/Output paths ──
        FileInputFormat.addInputPath(job, new Path(args[0]));
        FileOutputFormat.setOutputPath(job, new Path(args[1]));

        // ── Run and produce quality report ──
        boolean success = job.waitForCompletion(true);

        if (success) {
            printDataQualityReport(job);
        }

        return success ? 0 : 1;
    }


    // ════════════════════════════════════════════════════════
    // DATA QUALITY REPORT — Read counters after job completion
    // ════════════════════════════════════════════════════════

    private void printDataQualityReport(Job job) throws IOException {
        System.out.println("\n");
        System.out.println("╔══════════════════════════════════════════════════╗");
        System.out.println("║         DATA QUALITY REPORT                      ║");
        System.out.println("╠══════════════════════════════════════════════════╣");

        long total = getCounter(job, "TOTAL_INPUT_RECORDS");
        long accepted = getCounter(job, "ACCEPTED_RECORDS");

        long emptyLine     = getCounter(job, "REJECTED_EMPTY_LINE");
        long headerRow     = getCounter(job, "REJECTED_HEADER_ROW");
        long wrongFields   = getCounter(job, "REJECTED_WRONG_FIELD_COUNT");
        long missingPid    = getCounter(job, "REJECTED_MISSING_PATIENT_ID");
        long invalidPid    = getCounter(job, "REJECTED_INVALID_PATIENT_ID");
        long missingName   = getCounter(job, "REJECTED_MISSING_NAME");
        long invalidDob    = getCounter(job, "REJECTED_INVALID_DOB");
        long futureDob     = getCounter(job, "REJECTED_FUTURE_DOB");
        long invalidGender = getCounter(job, "REJECTED_INVALID_GENDER");
        long missingDept   = getCounter(job, "REJECTED_MISSING_DEPARTMENT");
        long invalidVisit  = getCounter(job, "REJECTED_INVALID_VISIT_DATE");
        long invalidStatus = getCounter(job, "REJECTED_INVALID_STATUS");
        long invalidAmount = getCounter(job, "REJECTED_INVALID_AMOUNT");
        long duplicates    = getCounter(job, "REJECTED_DUPLICATE_WITHIN_SPLIT");

        long totalRejected = total - accepted;

        System.out.printf("║  Total Input Records:      %,10d           ║%n", total);
        System.out.println("║──────────────────────────────────────────────────║");
        System.out.printf("║  ✅ ACCEPTED (clean):       %,10d (%5.1f%%)   ║%n",
                accepted, pct(accepted, total));
        System.out.printf("║  ❌ REJECTED (dirty):       %,10d (%5.1f%%)   ║%n",
                totalRejected, pct(totalRejected, total));
        System.out.println("║──────────────────────────────────────────────────║");
        System.out.println("║  REJECTION BREAKDOWN:                            ║");
        System.out.printf("║    Empty lines:            %,10d             ║%n", emptyLine);
        System.out.printf("║    Header rows:            %,10d             ║%n", headerRow);
        System.out.printf("║    Wrong field count:      %,10d             ║%n", wrongFields);
        System.out.printf("║    Missing patient ID:     %,10d             ║%n", missingPid);
        System.out.printf("║    Invalid patient ID:     %,10d             ║%n", invalidPid);
        System.out.printf("║    Missing name:           %,10d             ║%n", missingName);
        System.out.printf("║    Invalid date of birth:  %,10d             ║%n", invalidDob);
        System.out.printf("║    Future date of birth:   %,10d             ║%n", futureDob);
        System.out.printf("║    Invalid gender:         %,10d             ║%n", invalidGender);
        System.out.printf("║    Missing department:     %,10d             ║%n", missingDept);
        System.out.printf("║    Invalid visit date:     %,10d             ║%n", invalidVisit);
        System.out.printf("║    Invalid status:         %,10d             ║%n", invalidStatus);
        System.out.printf("║    Invalid amount:         %,10d             ║%n", invalidAmount);
        System.out.printf("║    Duplicate (in-split):   %,10d             ║%n", duplicates);
        System.out.println("║──────────────────────────────────────────────────║");

        // Reconciliation check
        long reconciledTotal = accepted + emptyLine + headerRow + wrongFields
                + missingPid + invalidPid + missingName + invalidDob + futureDob
                + invalidGender + missingDept + invalidVisit + invalidStatus
                + invalidAmount + duplicates;

        System.out.printf("║  RECONCILIATION:  %,d + %,d = %,d           ║%n",
                accepted, totalRejected, reconciledTotal);
        System.out.printf("║  Matches input?   %s                             ║%n",
                reconciledTotal == total ? "✅ YES" : "❌ NO — DATA LOSS!");

        System.out.println("╚══════════════════════════════════════════════════╝");
        System.out.println();
    }

    private long getCounter(Job job, String counterName) throws IOException {
        return job.getCounters()
                  .findCounter(CleaningMapper.CleaningCounters.valueOf(counterName))
                  .getValue();
    }

    private double pct(long part, long total) {
        return total == 0 ? 0.0 : (part * 100.0 / total);
    }


    // ════════════════════════════════════════════════════════
    // MAIN
    // ════════════════════════════════════════════════════════

    public static void main(String[] args) throws Exception {
        int exitCode = ToolRunner.run(new Configuration(),
                                      new HospitalRecordCleaner(), args);
        System.exit(exitCode);
    }
}

        "Dan", "Liz", "Kim", "Roy", "Eva", "Max", "Ana", "Leo",
        "Mia", "Zoe", "Jay", "Uma", "Raj", "Nia", "Ben", "Joy"
    ]
    
    last_names = [
        "Smith", "Chen", "Patel", "Lee", "Brown", "Kim", "Adams",
        "Stone", "Cole", "Grant", "Lopez", "Ray", "Fox", "Wu",
        "Hill", "Yao", "Park", "Ruiz", "Hall", "Young"
    ]
    
    statuses = ["Completed", "Pending", "Cancelled"]
    
    error_types = [
        "missing_name", "invalid_date", "invalid_gender",
        "missing_dept", "future_dob", "invalid_status",
        "bad_amount", "empty_line", "wrong_fields",
        "bad_patient_id", "header_row"
    ]
    
    records_written = 0
    errors_injected = 0
    duplicate_source_ids = []
    
    with open(filepath, 'w') as f:
        for i in range(1, num_records + 1):
            
            # Decide if this record should be dirty
            inject_error = random.random() < error_rate
            
            if inject_error:
                error_type = random.choice(error_types)
                errors_injected += 1
            else:
                error_type = None
            
            # ── Generate base valid record ──
            patient_id = f"PAT_{i:06d}"
            name = f"{random.choice(first_names)} {random.choice(last_names)}"
            
            # Date of birth: between 1940 and 2005
            birth_year = random.randint(1940, 2005)
            birth_month = random.randint(1, 12)
            birth_day = random.randint(1, 28)  # Safe day range
            dob = f"{birth_year}-{birth_month:02d}-{birth_day:02d}"
            
            gender = random.choice(["M", "F"])
            department = random.choice(departments)
            
            # Visit date: random day in January 2024
            visit_day = random.randint(1, 31)
            visit_date = f"2024-01-{visit_day:02d}"
            
            status = random.choice(statuses)
            amount = round(random.uniform(50, 2000), 2)
            
            # ── Inject specific error type ──
            if error_type == "missing_name":
                name = ""
            
            elif error_type == "invalid_date":
                # Use wrong date format: DD/MM/YYYY instead of YYYY-MM-DD
                dob = f"{birth_day:02d}/{birth_month:02d}/{birth_year}"
            
            elif error_type == "invalid_gender":
                gender = random.choice(["X", "U", "N/A", "Other", ""])
            
            elif error_type == "missing_dept":
                department = ""
            
            elif error_type == "future_dob":
                # Birth date in the future: 2045-2060
                future_year = random.randint(2045, 2060)
                dob = f"{future_year}-{birth_month:02d}-{birth_day:02d}"
            
            elif error_type == "invalid_status":
                status = random.choice(["Unknown", "Done", "Active", "N/A", ""])
            
            elif error_type == "bad_amount":
                # Either non-numeric or negative
                amount_choice = random.choice(["text", "negative", "empty"])
                if amount_choice == "text":
                    amount = "abc"
                elif amount_choice == "negative":
                    amount = round(random.uniform(-500, -1), 2)
                else:
                    amount = ""
            
            elif error_type == "empty_line":
                f.write("\n")
                records_written += 1
                continue
            
            elif error_type == "wrong_fields":
                # Write record with wrong number of fields
                f.write(f"{patient_id}|{name}|{dob}\n")
                records_written += 1
                continue
            
            elif error_type == "bad_patient_id":
                patient_id = random.choice([
                    "", "PATXYZ", "12345", "PAT-001",
                    "pat_001", "P_1", "NULL"
                ])
            
            elif error_type == "header_row":
                f.write("PAT_ID|Name|DOB|Gender|Department|Visit_Date|Status|Amount\n")
                records_written += 1
                continue
            
            # ── Build and write the record ──
            record = f"{patient_id}|{name}|{dob}|{gender}|{department}|{visit_date}|{status}|{amount}"
            f.write(record + "\n")
            records_written += 1
            
            # ── Track some IDs for duplicate injection ──
            if error_type is None and len(duplicate_source_ids) < 1000:
                # Store some clean records to duplicate later
                duplicate_source_ids.append(record)
        
        # ── Inject duplicates at the end ──
        # Take 5% of stored clean records and write them again
        num_duplicates = max(1, int(len(duplicate_source_ids) * 0.05))
        duplicate_records = random.sample(
            duplicate_source_ids, 
            min(num_duplicates, len(duplicate_source_ids))
        )
        
        for dup_record in duplicate_records:
            f.write(dup_record + "\n")
            records_written += 1
            errors_injected += 1
    
    file_size_mb = os.path.getsize(filepath) / (1024 * 1024)
    
    print(f"═══ GENERATION SUMMARY ═══")
    print(f"File:              {filepath}")
    print(f"Size:              {file_size_mb:.1f} MB")
    print(f"Total lines:       {records_written:,}")
    print(f"Target error rate: {error_rate * 100:.0f}%")
    print(f"Errors injected:   {errors_injected:,} ({errors_injected * 100.0 / records_written:.1f}%)")
    print(f"Duplicates added:  {len(duplicate_records):,}")
    print()
    
    return records_written, errors_injected


def verify_cleaning_results(input_path, output_path):
    """
    Post-job verification: compare input and output statistics.
    Run this AFTER the Hadoop job completes.
    Uses local file copies (hdfs dfs -get first).
    """
    
    print("═══ POST-CLEANING VERIFICATION ═══\n")
    
    # ── Count input records ──
    input_count = 0
    with open(input_path, 'r') as f:
        for line in f:
            input_count += 1
    
    # ── Count and validate output records ──
    output_count = 0
    validation_failures = []
    
    with open(output_path, 'r') as f:
        for line_num, line in enumerate(f, 1):
            record = line.strip()
            if not record:
                continue
            
            output_count += 1
            fields = record.split('|')
            
            # ── Re-validate every output record ──
            issues = []
            
            # Field count
            if len(fields) != 8:
                issues.append(f"Wrong field count: {len(fields)}")
            else:
                pid, name, dob, gender, dept, visit, status, amount = fields
                
                # Patient ID format
                if not pid.startswith("PAT_") or not pid[4:].isdigit():
                    issues.append(f"Bad patient ID: {pid}")
                
                # Name present
                if not name.strip():
                    issues.append("Missing name")
                
                # Date format (basic check)
                try:
                    datetime.strptime(dob, "%Y-%m-%d")
                    dob_date = datetime.strptime(dob, "%Y-%m-%d")
                    if dob_date > datetime.now():
                        issues.append(f"Future DOB: {dob}")
                except ValueError:
                    issues.append(f"Invalid DOB: {dob}")
                
                # Gender
                if gender not in ("M", "F"):
                    issues.append(f"Invalid gender: {gender}")
                
                # Department
                if not dept.strip():
                    issues.append("Missing department")
                
                # Visit date
                try:
                    datetime.strptime(visit, "%Y-%m-%d")
                except ValueError:
                    issues.append(f"Invalid visit date: {visit}")
                
                # Status
                if status not in ("COMPLETED", "PENDING", "CANCELLED"):
                    issues.append(f"Invalid status: {status}")
                
                # Amount
                try:
                    amt = float(amount)
                    if amt < 0:
                        issues.append(f"Negative amount: {amount}")
                except ValueError:
                    issues.append(f"Non-numeric amount: {amount}")
            
            if issues:
                validation_failures.append({
                    "line": line_num,
                    "record": record[:80],
                    "issues": issues
                })
    
    # ── Print results ──
    print(f"Input records:       {input_count:,}")
    print(f"Output records:      {output_count:,}")
    print(f"Records removed:     {input_count - output_count:,}")
    print(f"Removal rate:        {(input_count - output_count) * 100.0 / input_count:.1f}%")
    print()
    
    if validation_failures:
        print(f"⚠️  DIRTY RECORDS IN OUTPUT: {len(validation_failures)}")
        print("   These records should NOT have passed cleaning:\n")
        for failure in validation_failures[:10]:  # Show first 10
            print(f"   Line {failure['line']}: {failure['record']}")
            for issue in failure['issues']:
                print(f"      → {issue}")
            print()
        if len(validation_failures) > 10:
            print(f"   ... and {len(validation_failures) - 10} more")
    else:
        print("✅ ALL OUTPUT RECORDS ARE VALID — Cleaning job is correct!")
    
    print()
