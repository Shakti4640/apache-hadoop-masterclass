SOLUTION: Migrate storage to object store, compute to managed clusters

PATH 1: LIFT AND SHIFT (Easiest, least optimized)
→ Move data from HDFS to S3/ADLS/GCS

→ Replace Hadoop entirely with serverless services
→ AWS: Athena + Glue + Redshift Spectrum
→ Azure: Synapse Analytics + Data Factory
→ GCP: BigQuery + Dataflow
→ No cluster management at all
→ Out of scope for this Hadoop guide

✅ WHAT SHOULD DO

1. START WITH DATA MIGRATION AND VALIDATION
   → Copy a SUBSET of data to cloud storage first
   → Run identical pipeline on-premise AND cloud
   → Compare outputs byte-for-byte
   → Only proceed when results match 100%

2. USE IAM ROLES, NOT ACCESS KEYS
   → Access keys are static secrets that can leak
   → IAM roles are attached to EC2/EMR instances
   → No credentials in code or configuration files
   → Temporary credentials rotate automatically

3. USE PARQUET/ORC FOR CLOUD STORAGE
   → Object stores have HIGH latency per request
   → Columnar formats minimize data read (column pruning)
   → Parquet with Snappy compression: best general choice
   → ORC for Hive-heavy workloads (Project 30 comparison)

4. PARTITION DATA BY QUERY PATTERNS
   → S3 "directories" are just key prefixes
   → s3://bucket/data/year=2024/month=01/day=15/
   → Spark/Hive partition pruning skips irrelevant partitions
   → Reading 1 day from 3 years: scans 0.1% of data instead of 100%

5. SIZE FILES CORRECTLY FOR OBJECT STORAGE
   → Small files problem (Project 2 pitfall) is WORSE on cloud
   → Each S3 GET request has ~50-100ms latency
   → 1 million small files = 1M requests = minutes of overhead
   → Target: 128MB-1GB files (coalesce small files)

6. USE SPOT/PREEMPTIBLE INSTANCES FOR WORKERS
   → 60-90% cheaper than on-demand
   → Spark handles node loss via RDD lineage (Project 48 mechanic)
   → Keep master nodes on-demand (stable)
   → Workers can be interrupted — Spark retries tasks

7. EXTERNALIZE THE HIVE METASTORE
   → Ephemeral clusters lose local metastore on termination
   → AWS Glue Catalog persists table definitions
   → Azure: External SQL Database for Hive metastore
   → Table created by morning cluster → readable by evening cluster

8. ENABLE S3 COMMITTERS FOR RELIABLE OUTPUT
   → Default OutputCommitter uses rename — S3 rename is NOT atomic
   → S3A Committer (magic/staging) provides correct commit protocol
   → Without this: partial/duplicate output files on job failure

🚫 WHAT SHOULD NOT DO

1. DO NOT assume data locality exists on cloud
   → On-premise HDFS: compute reads LOCAL disk (100-200 MB/s per disk)
   → Cloud: compute reads from S3 OVER NETWORK
   → Network: typically 10-25 Gbps per instance = 1.2-3.1 GB/s
   → S3 single-stream: ~100 MB/s, multi-stream: up to 10 GB/s
   → IMPLICATION: Increase parallelism, use columnar formats
   → DO NOT use TextInputFormat on huge CSV files on S3

2. DO NOT store credentials in code or hadoop configs
   → NEVER: fs.s3a.access.key = AKIA... in core-site.xml
   → ALWAYS: Use IAM instance profiles or environment credentials
   → Leaked keys = compromised S3 bucket = data breach

3. DO NOT create persistent clusters and leave them running
   → Defeats the entire cost advantage of cloud
   → "Cloud lift-and-shift" without ephemeral clusters
     = MORE expensive than on-premise (cloud markup + storage cost)
   → Design for: spin up → process → terminate

4. DO NOT ignore S3 eventual consistency (pre-2020 S3)
   → S3 now provides strong read-after-write consistency (Dec 2020)
   → BUT: S3 list operations can still be slow for large prefixes
   → EMRFS consistent view was needed before — less critical now
   → ADLS Gen2 has always been strongly consistent

5. DO NOT use HDFS on EMR as primary storage
   → EMR local HDFS lives on instance storage
   → When cluster terminates → ALL HDFS DATA IS LOST
   → Local HDFS is ONLY for temporary shuffle/intermediate data
   → ALL persistent data MUST be on S3/ADLS

6. DO NOT migrate all data at once
   → Start with one pipeline's data
   → Validate thoroughly
   → Then migrate next pipeline
   → Keep on-premise as fallback for 3-6 months

7. DO NOT use same file sizes as on-premise
   → HDFS: 128MB blocks work well (local disk I/O)
   → S3: 256MB-1GB files work better (amortize request latency)
   → Rewrite small files into larger ones during migration

8. DO NOT skip cost monitoring
   → Cloud costs are invisible until the bill arrives
   → Set budget alerts at 50%, 80%, 100% of expected
   → Monitor: S3 request costs (GET/PUT), data transfer, compute hours
   → A runaway Spark job reading 1PB from S3 = $5,000 in GET requests

🚫 WHEN NOT TO USE

DO NOT MIGRATE TO CLOUD WHEN:

1. DATA SOVEREIGNTY PROHIBITS IT
   → Some industries (government, healthcare, finance) in certain countries
     require data to stay within specific physical boundaries
   → Cloud regions may satisfy this — verify with legal team
   → If not satisfied → on-premise is the only option

2. NETWORK BANDWIDTH IS THE BOTTLENECK
   → Initial migration of 2PB over 1Gbps link = ~6 MONTHS
   → Even 10Gbps = ~18 days for 2PB
   → AWS Snowball (physical device) solves this for initial migration
   → Ongoing: if data generation exceeds upload bandwidth → problem

3. YOUR WORKLOAD IS 100% UTILIZED 24/7
   → Cloud compute has ~30-40% markup over equivalent on-premise
   → If cluster runs at 90%+ utilization continuously
   → On-premise + 3-year hardware cycle is cheaper
   → Rare in practice — most clusters have significant idle time

4. ULTRA-LOW LATENCY RANDOM ACCESS IS PRIMARY USE CASE
   → HBase on HDFS: <5ms random reads (local disk)
   → HBase on S3: 50-200ms per read (network round trip)
   → For real-time serving: keep HBase on-premise or use DynamoDB/Cosmos
   → (Reference: Project 27, 35 — HBase row key design)

5. YOUR TEAM LACKS CLOUD SKILLS
   → Cloud misconfiguration is the #1 cause of security breaches
   → Leaving S3 bucket public = data exposed to entire internet
   → Budget 2-3 months of cloud training before migration
   → Cloud complexity is different, not less, than on-premise

  Your code:     spark.read.parquet("s3a://bucket/data/logs/")

  2. For each object: GET object → simulates file read
  3. Streams bytes into InputFormat → records to Spark

═══════════════════════════════════════════════════════════
MECHANIC 2: EMRFS vs S3A — AWS-SPECIFIC
═══════════════════════════════════════════════════════════

AWS offers TWO ways to access S3 from Hadoop:

OPTION A: s3a:// (Open-source Apache connector)
  → Part of Hadoop project (hadoop-aws module)
  → Works on ANY Hadoop cluster (on-prem, EMR, Dataproc)
  → Uses AWS SDK directly
  → Configuration via core-site.xml
  → Path: s3a://my-bucket/path/to/data

OPTION B: s3:// via EMRFS (AWS proprietary, EMR only)
  → Built into Amazon EMR
  → Optimized for EMR-specific features
  → Consistent view via DynamoDB metadata (pre-2020)
  → Now less critical since S3 strong consistency (Dec 2020)
  → Path: s3://my-bucket/path/to/data (no 'a')

│ FEATURE             │ s3a://           │ s3:// (EMRFS)    │

═══════════════════════════════════════════════════════════
MECHANIC 3: THE OUTPUT COMMIT PROBLEM
═══════════════════════════════════════════════════════════

ON HDFS (how it works correctly):

  MapReduce/Spark write workflow:
  1. Each task writes to: _temporary/task_attempt_xxxx/part-00000
  2. Task succeeds → RENAME to: output/part-00000
  3. RENAME on HDFS is ATOMIC and INSTANT (metadata operation)
  4. If task fails → temporary file is simply deleted
  5. No partial results ever visible in output directory

ON S3 (how it BREAKS):

  1. Each task writes to: _temporary/task_attempt_xxxx/part-00000
  2. Task succeeds → RENAME = COPY + DELETE
  3. COPY: Read entire file from S3, write it back to new key
     For a 1GB file: this takes MINUTES, not milliseconds
  4. If copy succeeds but delete fails → DUPLICATE data
  5. If job crashes during copy → PARTIAL results
  6. 1000 tasks × 1GB each → 1TB of COPY operations at commit time

SOLUTION: S3A COMMITTERS

  ┌────────────────────────────────────────────────────────┐
  │  Magic Committer (recommended for Spark):               │
  │                                                         │
  │  1. Tasks write directly to final output path           │
  │  2. BUT: writes go through S3 MultipartUpload           │
  │  3. MultipartUpload parts are INVISIBLE until committed │
  │  4. On task success: complete the multipart upload       │
  │     (single API call — instant, no copy)                │
  │  5. On task failure: abort the multipart upload          │
  │     (parts are discarded — no partial output)            │
  │                                                         │
  │  Result: Atomic commit WITHOUT rename/copy               │
  │  Performance: No extra I/O at commit time                │
  └────────────────────────────────────────────────────────┘

  ┌────────────────────────────────────────────────────────┐
  │  Staging Committer (alternative):                       │
  │                                                         │
  │  1. Tasks write to LOCAL disk (HDFS on cluster)         │
  │  2. On task success: upload from local to S3             │
  │  3. Job commit: finalize all uploads                     │
  │                                                         │
  │  Trade-off: Needs local HDFS for staging                 │
  │  Safer for very large individual files                   │
  └────────────────────────────────────────────────────────┘

MECHANIC 6: PERFORMANCE WITHOUT DATA LOCALITY
═══════════════════════════════════════════════════════════

  On-premise data locality (Project 1 principle):
    HDFS block on same machine as Mapper → disk read at 200 MB/s
    
  Cloud without data locality:
    S3 object on different machine → network read
    
  HOW CLOUD COMPENSATES:

  1. HIGHER NETWORK BANDWIDTH
     → Modern cloud instances: 10-100 Gbps network
     → Single S3 stream: ~100 MB/s
     → Parallel streams: up to 10 GB/s per instance
     → With 100 instances: 1 TB/s aggregate read throughput
     → EXCEEDS what on-premise HDFS can deliver

  2. COLUMNAR FORMATS REDUCE DATA READ
     → Parquet on S3: read only needed columns
     → 50-column table, query needs 3 columns → read 6% of data
     → S3 Select: push predicate INTO S3 (read even less)
     → Predicate pushdown: skip entire Parquet row groups

  3. S3 REQUEST PARALLELISM
     → Each Spark partition reads from S3 independently
     → S3 scales to thousands of concurrent requests per prefix
     → No single-machine bottleneck (unlike HDFS NameNode)

  4. LARGER INSTANCE TYPES AVAILABLE
     → On-premise: limited by what you purchased
     → Cloud: r5.24xlarge = 96 cores, 768GB RAM per instance
     → Can process more data per node despite network reads

  NET EFFECT:
  → Simple single-pass ETL: 10-30% slower on cloud (network overhead)
  → Complex multi-stage pipeline: COMPARABLE or faster
     (S3 bandwidth × parallelism compensates for latency)
  → Cost per job: 60-80% cheaper (ephemeral + spot pricing)

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "S3 listing is incredibly slow for partitioned tables"
→ REASON: S3 LIST API returns 1000 objects per call
→ Table with 100,000 partitions = 100 LIST calls = 10-30 seconds
→ On HDFS: NameNode returns all metadata from memory instantly
→ FIX: Use Hive metastore for partition discovery (not S3 listing)
   spark.conf.set("spark.sql.hive.metastorePartitionPruning", "true")
   spark.conf.set("spark.sql.hive.filesourcePartitionFilePruning", "true")

PITFALL 2: "Job succeeded but output directory is empty/partial"
→ REASON: Default FileOutputCommitter uses rename (broken on S3)
→ FIX: Enable S3A Magic Committer:
   spark.hadoop.fs.s3a.committer.name=magic
   spark.hadoop.fs.s3a.committer.magic.enabled=true
   spark.hadoop.mapreduce.outputcommitter.factory.scheme.s3a=
     org.apache.hadoop.fs.s3a.commit.S3ACommitterFactory

PITFALL 3: "Cluster terminated but data is lost"
→ REASON: Data written to local HDFS (instance storage) not S3
→ FIX: ALWAYS write final output to s3a:// path
   Never rely on cluster-local HDFS for persistent data
   Configure Spark event logs to S3 BEFORE cluster terminates

PITFALL 4: "S3 throttling — 503 SlowDown errors"
→ REASON: S3 rate limit per prefix: 5,500 GET/s, 3,500 PUT/s
→ 1000 Spark tasks all reading same S3 prefix simultaneously
→ FIX: Distribute data across multiple S3 prefixes
   Use random prefixes: s3://bucket/a1b2c3/data/ (old technique)
   S3 now auto-scales (2018+) but initial burst can still throttle
   Enable retry: spark.hadoop.fs.s3a.retry.limit=10

PITFALL 5: "Hive tables disappear after cluster restart"
→ REASON: Default Hive metastore is embedded Derby DB
→ Stored in local filesystem → lost when cluster terminates
→ FIX: External metastore:
   AWS: Use Glue Data Catalog as Hive metastore
   Azure: External Azure SQL Database
   GCP: Dataproc Metastore service

PITFALL 6: "Spot instance terminated mid-job → job fails"
→ REASON: Spot instances reclaimed when price exceeds bid
→ FIX: 
   Use spot for TASK nodes only (no HDFS data)
   Keep CORE/MASTER nodes on-demand
   Enable Spark speculation for straggler handling
   Enable YARN retry: spark.yarn.maxAppAttempts=3
   Use Spark's fault tolerance — lost tasks automatically retried

PITFALL 7: "Cloud costs unexpectedly high"
→ COMMON CAUSES:
   - Cluster left running over weekend ($5,000 mistake)
   - Excessive S3 GET requests from small files
   - Data transfer between regions (cross-region = $0.02/GB)
   - Over-provisioned instances (using r5.24xlarge for simple ETL)
→ FIX: Set auto-termination on idle, budget alerts,
   tag all resources, review billing weekly

PITFALL 8: "Performance much worse than on-premise"
→ COMMON CAUSES:
   - Reading text/CSV instead of Parquet from S3
   - Millions of small files (each S3 GET = 50-100ms)
   - Insufficient parallelism (too few partitions)
   - Instances without enhanced networking enabled
→ FIX: Convert to Parquet, coalesce small files,
   increase spark.sql.shuffle.partitions,
   use instances with 10+ Gbps networking

def migrate_shipments(spark, source_hdfs, target_s3):
    """
    Migrate shipments CSV from HDFS to partitioned Parquet on S3.
    
    This combines:
    → Data migration (HDFS → S3)
    → Format conversion (CSV → Parquet) — Project 30
    → Partitioning (by region, status) — Project 25
    → File size optimization (coalesce for target size)
    """

    print(f"\n[MIGRATE] Source: {source_hdfs}")
    print(f"[MIGRATE] Target: {target_s3}")

    start = time.time()

    # ── Step 1: Read from HDFS (on-premise) ──
    shipment_schema = StructType([
        StructField("shipment_id", StringType(), True),
        StructField("customer_id", StringType(), True),
        StructField("product_id", StringType(), True),
        StructField("status", StringType(), True),
        StructField("region", StringType(), True),
        StructField("delivery_days", IntegerType(), True),
        StructField("cost", DoubleType(), True),
        StructField("timestamp", StringType(), True)
    ])

    df = spark.read \
        .option("header", "true") \
        .schema(shipment_schema) \
        .csv(source_hdfs)

    source_count = df.count()
    print(f"[MIGRATE] Source records: {source_count}")

    # ── Step 2: Clean and normalize during migration ──
    # Why not just copy? Because we're already reading the data —
    # might as well optimize it for cloud query patterns
    df_cleaned = df \
        .filter(
            (F.col("region").isNotNull()) &
            (F.trim(F.col("region")) != "") &
            (F.col("cost") > 0)
        ) \
        .withColumn("region", F.trim(F.upper(F.col("region")))) \
        .withColumn(
            "event_date",
            F.to_date(F.col("timestamp"), "yyyy-MM-dd'T'HH:mm:ss")
        ) \
        .withColumn(
            "event_year", F.year(F.col("event_date"))
        ) \
        .withColumn(
            "event_month", F.month(F.col("event_date"))
        )

    # ── Step 3: Repartition for optimal file sizes on S3 ──
    # Target: ~256MB per Parquet file
    # Estimate: 2M records × ~100 bytes each = ~200MB as Parquet
    # With partitioning by region (5) × status (5) = 25 partitions
    # Each partition: ~8MB (too small for S3!)
    # Solution: coalesce within each partition to reduce file count

    # For larger datasets, calculate:
    # target_file_size_mb = 256
    # estimated_total_mb = source_count * avg_row_size_bytes / (1024*1024)
    # num_output_files = max(1, int(estimated_total_mb / target_file_size_mb))

    df_optimized = df_cleaned.repartition(
        10,  # Target ~10 output files total
        F.col("region")
    )

    # ── Step 4: Write to S3 as partitioned Parquet ──
    df_optimized.write \
        .mode("overwrite") \
        .partitionBy("region", "status") \
        .option("maxRecordsPerFile", 500000) \
        .parquet(target_s3)

    elapsed = time.time() - start

    # ── Step 5: Validate migration ──
    print(f"\n[VALIDATE] Reading back from S3...")
    df_s3 = spark.read.parquet(target_s3)
    target_count = df_s3.count()

    print(f"[VALIDATE] Source records (HDFS):  {source_count}")
    print(f"[VALIDATE] Target records (S3):    {target_count}")
    print(f"[VALIDATE] Cleaned/removed:        {source_count - target_count}")
    print(f"[VALIDATE] Migration time:         {elapsed:.1f}s")

    # ── Step 6: Verify partition structure ──
    print(f"\n[VALIDATE] S3 partition structure:")
    partitions = df_s3.select("region", "status").distinct().collect()
    for p in sorted(partitions, key=lambda x: (x.region, x.status)):
        count = df_s3.filter(
            (F.col("region") == p.region) &
            (F.col("status") == p.status)
        ).count()
        print(f"  region={p.region}/status={p.status}: {count} records")

    # ── Step 7: Compare aggregates for data integrity ──
    print(f"\n[VALIDATE] Aggregate comparison:")

    hdfs_agg = df.filter(
        (F.col("region").isNotNull()) &
        (F.trim(F.col("region")) != "") &
        (F.col("cost") > 0)
    ).agg(
        F.sum("cost").alias("total_cost"),
        F.avg("cost").alias("avg_cost")
    ).collect()[0]

    s3_agg = df_s3.agg(
        F.sum("cost").alias("total_cost"),
        F.avg("cost").alias("avg_cost")
    ).collect()[0]

    cost_match = abs(hdfs_agg.total_cost - s3_agg.total_cost) < 0.01
    avg_match = abs(hdfs_agg.avg_cost - s3_agg.avg_cost) < 0.01

    print(f"  HDFS total cost: {hdfs_agg.total_cost:.2f}")
    print(f"  S3 total cost:   {s3_agg.total_cost:.2f}")
    print(f"  Match: {'✅' if cost_match else '❌'}")

    print(f"  HDFS avg cost:   {hdfs_agg.avg_cost:.2f}")
    print(f"  S3 avg cost:     {s3_agg.avg_cost:.2f}")
    print(f"  Match: {'✅' if avg_match else '❌'}")

    return cost_match and avg_match


def migrate_customers(spark, source_hdfs, target_s3):
    """Migrate small customer dimension table."""

    print(f"\n[MIGRATE] Customers: {source_hdfs} → {target_s3}")

    df = spark.read \
        .option("header", "true") \
        .option("inferSchema", "true") \
        .csv(source_hdfs)

    source_count = df.count()

    # Small table — single Parquet file is fine
    df.coalesce(1).write \
        .mode("overwrite") \
        .parquet(target_s3)

    target_count = spark.read.parquet(target_s3).count()
    match = source_count == target_count
    print(f"  Source: {source_count}, Target: {target_count}, Match: {'✅' if match else '❌'}")

    return match

def run_cloud_pipeline(spark, paths):
    """
    Exact same pipeline as Project 48, running on cloud storage.
    Demonstrates that application code is cloud-agnostic.
    """

    pipeline_start = time.time()

    print("\n" + "=" * 70)
    print("CLOUD LOGISTICS PIPELINE")
    print(f"Storage paths: {paths['shipments'][:30]}...")
    print("=" * 70)

    # ── STAGE 1: Ingest from cloud storage ──
    print("\n[STAGE 1] Ingesting from cloud storage...")
    t1 = time.time()

    # For migrated Parquet data
    try:
        shipments = spark.read.parquet(paths["shipments"])
        customers = spark.read.parquet(paths["customers"])
    except Exception:
        # Fallback to CSV if Parquet not available
        print("  Parquet not found, trying CSV...")
        shipments = spark.read \
            .option("header", "true") \
            .option("inferSchema", "true") \
            .csv(paths["shipments"])
        customers = spark.read \
            .option("header", "true") \
            .option("inferSchema", "true") \
            .csv(paths["customers"])

    ship_count = shipments.count()
    cust_count = customers.count()
    print(f"  Shipments: {ship_count} records")
    print(f"  Customers: {cust_count} records")
    print(f"  Time: {time.time() - t1:.2f}s")

    # ── STAGE 2: Clean (same as Project 48 Stage 2) ──
    print("\n[STAGE 2] Cleaning data...")
    t2 = time.time()

    shipments_cleaned = shipments \
        .filter(
            (F.col("region").isNotNull()) &
            (F.trim(F.col("region")) != "") &
            (F.col("cost") > 0) &
            (F.col("cost").isNotNull())
        ) \
        .withColumn("region", F.trim(F.upper(F.col("region"))))

    # Cache — works the same on cloud
    # Data cached in executor JVM memory (not on S3)
    shipments_cleaned.cache()
    clean_count = shipments_cleaned.count()
    print(f"  Cleaned records: {clean_count}")
    print(f"  Time: {time.time() - t2:.2f}s")

    # ── STAGE 3: Join (same as Project 48 Stage 3) ──
    print("\n[STAGE 3] Customer enrichment join...")
    t3 = time.time()

    # Broadcast join — small table broadcast to all executors
    # Works identically on cloud — broadcast over cluster network
    enriched = shipments_cleaned.join(
        F.broadcast(customers),
        "customer_id",
        "inner"
    )
    enriched.cache()
    enriched_count = enriched.count()
    print(f"  Enriched records: {enriched_count}")
    print(f"  Time: {time.time() - t3:.2f}s")

    # Show join strategy
    print("  Join strategy:")
    enriched.explain(mode="simple")

    # ── STAGE 4: Aggregation (same as Project 48 Stage 4) ──
    print("\n[STAGE 4] Region aggregation...")
    t4 = time.time()

    region_metrics = enriched \
        .groupBy("region") \
        .agg(
            F.count("*").alias("shipment_count"),
            F.round(F.sum("cost"), 2).alias("total_cost"),
            F.round(F.avg("cost"), 2).alias("avg_cost"),
            F.countDistinct("customer_id").alias("unique_customers"),
            F.sum(F.when(F.col("status") == "delivered", 1).otherwise(0))
                .alias("delivered_count")
        ) \
        .withColumn(
            "delivery_rate",
            F.round(
                F.col("delivered_count") / F.col("shipment_count") * 100, 1
            )
        ) \
        .orderBy(F.desc("shipment_count"))

    print("  Region metrics:")
    region_metrics.show(truncate=False)
    print(f"  Time: {time.time() - t4:.2f}s")

    # ── STAGE 5: Top products (same as Project 48 Stage 6) ──
    print("\n[STAGE 5] Top products per region...")
    t5 = time.time()

    product_counts = enriched \
        .groupBy("region", "product_id") \
        .agg(F.count("*").alias("product_count"))

    window_spec = Window \
        .partitionBy("region") \
        .orderBy(F.desc("product_count"))

    top_products = product_counts \
        .withColumn("rank", F.row_number().over(window_spec)) \
        .filter(F.col("rank") <= 3) \
        .orderBy("region", "rank")

    print("  Top 3 products per region:")
    top_products.show(20, truncate=False)
    print(f"  Time: {time.time() - t5:.2f}s")

    # ── STAGE 6: Write results to cloud storage ──
    print("\n[STAGE 6] Writing results to cloud storage...")
    t6 = time.time()

    output_base = paths["output"]

    # Write region metrics as Parquet
    region_metrics.write \
        .mode("overwrite") \
        .parquet(f"{output_base}/region_metrics")
    print(f"  Region metrics → {output_base}/region_metrics")

    # Write top products as Parquet
    top_products.write \
        .mode("overwrite") \
        .parquet(f"{output_base}/top_products")
    print(f"  Top products → {output_base}/top_products")

    # Write enriched data partitioned (for downstream queries)
    enriched.write \
        .mode("overwrite") \
        .partitionBy("region", "status") \
        .parquet(f"{output_base}/enriched_partitioned")
    print(f"  Enriched data → {output_base}/enriched_partitioned")

    print(f"  Time: {time.time() - t6:.2f}s")

    # ── Cleanup ──
    shipments_cleaned.unpersist()
    enriched.unpersist()

    total_time = time.time() - pipeline_start

    print("\n" + "=" * 70)
    print(f"PIPELINE COMPLETE")

def compact_small_files(spark, input_path, output_path, target_size_mb=256):
    """
    Compact small files into larger ones for cloud storage efficiency.
    
    This is the cloud equivalent of dealing with the small files
    problem from Project 2 (NameNode memory) — but on cloud it's
    about S3 request costs and listing performance.
    """

    print(f"\n[COMPACT] Compacting: {input_path}")
    print(f"[COMPACT] Target file size: {target_size_mb}MB")

    df = spark.read.parquet(input_path)
    record_count = df.count()

    # Estimate bytes per record (approximate)
    # Sample 1000 records, measure size
    sample_df = df.limit(1000)
    temp_path = f"{output_path}/_size_estimate/"
    sample_df.write.mode("overwrite").parquet(temp_path)

    # Read back to estimate
    import subprocess
    try:
        result = subprocess.run(
            ["hdfs", "dfs", "-du", "-s", temp_path],
            capture_output=True, text=True
        )
        sample_bytes = int(result.stdout.strip().split()[0])
        bytes_per_record = sample_bytes / 1000
    except Exception:
        bytes_per_record = 200  # Conservative estimate

    total_bytes = record_count * bytes_per_record
    total_mb = total_bytes / (1024 * 1024)
    target_files = max(1, int(total_mb / target_size_mb))

    print(f"  Records: {record_count}")
    print(f"  Estimated size: {total_mb:.0f}MB")
    print(f"  Target files: {target_files}")

    # Coalesce to target number of files
    df.coalesce(target_files) \
        .write \
        .mode("overwrite") \
        .parquet(output_path)

    print(f"  ✅ Compacted to {target_files} files at {output_path}")

    # Cleanup temp
    try:
        subprocess.run(
            ["hdfs", "dfs", "-rm", "-r", "-f", temp_path],
            capture_output=True
        )
    except Exception:
        pass

    return target_files

def setup_persistent_tables(spark, storage_base):

    and Project 25 (partitioning/bucketing).
    """

    print("EXTERNAL METASTORE: Persistent Tables Across Clusters")
    print("=" * 70)

    spark.sql("USE logistics_cloud")

    # ── Create EXTERNAL table for shipments ──
    # EXTERNAL = data managed by US, not Hive
    # DROP TABLE does NOT delete the S3 data

    spark.sql(f"""
        CREATE EXTERNAL TABLE IF NOT EXISTS shipments_enriched (
            shipment_id     STRING,
            customer_id     STRING,
            product_id      STRING,
            delivery_days   INT,
            cost            DOUBLE,
            event_date      DATE,

            event_month     INT,
            customer_name   STRING,
            tier            STRING,
            signup_year     INT

        STORED AS PARQUET
        LOCATION '{storage_base}/enriched_partitioned/'
    """)

    # S3 has the data in region=NORTH/status=delivered/ directories

        SELECT region, status, COUNT(*) as cnt, ROUND(AVG(cost), 2) as avg_cost
        FROM shipments_enriched

def generate_airflow_dag():
    """
    Apache Airflow DAG — works on any cloud (GCP Cloud Composer,
    AWS MWAA, or self-hosted).
    Most portable orchestration option.
    """

    airflow_dag_code = '''
# Save as: dags/logistics_pipeline_dag.py
# Deploy to Airflow (Cloud Composer / MWAA / self-hosted)

from airflow import DAG
from airflow.providers.google.cloud.operators.dataproc import (
    DataprocCreateClusterOperator,
    DataprocSubmitPySparkJobOperator,
    DataprocDeleteClusterOperator
)
# For AWS EMR, use:
# from airflow.providers.amazon.aws.operators.emr import (
#     EmrCreateJobFlowOperator,
#     EmrAddStepsOperator,
#     EmrTerminateJobFlowOperator
# )
from airflow.operators.python import BranchPythonOperator
from airflow.operators.email import EmailOperator
from airflow.utils.dates import days_ago
from datetime import timedelta


# ── DAG Definition ──
default_args = {
    "owner": "data-engineering",
    "depends_on_past": False,
    "email": ["alerts@company.com"],
    "email_on_failure": True,
    "email_on_retry": False,
    "retries": 2,
    "retry_delay": timedelta(minutes=10),
}

with DAG(
    dag_id="logistics_pipeline_cloud",
    default_args=default_args,
    description="Nightly logistics ETL pipeline on cloud Hadoop",
    schedule_interval="0 22 * * *",  # Every night at 10 PM
    start_date=days_ago(1),
    catchup=False,
    tags=["logistics", "spark", "cloud"],
) as dag:

    # ── Step 1: Create ephemeral cluster ──
    create_cluster = DataprocCreateClusterOperator(
        task_id="create_dataproc_cluster",
        project_id="my-gcp-project",
        cluster_name="logistics-{{ ds_nodash }}",
        region="us-central1",
        cluster_config={
            "master_config": {
                "num_instances": 1,
                "machine_type_uri": "n2-standard-8",
                "disk_config": {"boot_disk_size_gb": 100}
            },
            "worker_config": {
                "num_instances": 5,
                "machine_type_uri": "n2-highmem-16",
                "disk_config": {"boot_disk_size_gb": 200}
            },
            "secondary_worker_config": {
                "num_instances": 15,
                "machine_type_uri": "n2-highmem-16",
                "is_preemptible": True
            },
            "software_config": {
                "image_version": "2.1",
                "properties": {
                    "spark:spark.sql.adaptive.enabled": "true",
                    "spark:spark.dynamicAllocation.enabled": "true",
                    "spark:spark.executor.memory": "12g"
                }
            },
            "lifecycle_config": {
                "idle_delete_ttl": {"seconds": 1800},  # 30 min idle → delete
                "auto_delete_time": "2024-01-16T06:00:00Z"  # Force delete by 6AM
            }
        }
    )

    # ── Step 2: Run main pipeline ──
    run_pipeline = DataprocSubmitPySparkJobOperator(
        task_id="run_logistics_pipeline",
        main="gs://my-data-lake/scripts/cloud_logistics_pipeline.py",
        arguments=["gcp"],
        cluster_name="logistics-{{ ds_nodash }}",
        region="us-central1",
        dataproc_pyspark_properties={
            "spark.executor.memory": "12g",
            "spark.executor.cores": "4",
            "spark.sql.shuffle.partitions": "200"
        }
    )

    # ── Step 3: Run data quality checks ──
    run_quality = DataprocSubmitPySparkJobOperator(
        task_id="run_data_quality",
        main="gs://my-data-lake/scripts/data_quality_checks.py",
        cluster_name="logistics-{{ ds_nodash }}",
        region="us-central1"
    )

    # ── Step 4: Refresh metastore ──
    refresh_metastore = DataprocSubmitPySparkJobOperator(
        task_id="refresh_metastore",
        main="gs://my-data-lake/scripts/external_metastore_setup.py",
        cluster_name="logistics-{{ ds_nodash }}",
        region="us-central1"
    )

    # ── Step 5: Delete cluster ──
    delete_cluster = DataprocDeleteClusterOperator(
        task_id="delete_dataproc_cluster",
        project_id="my-gcp-project",
        cluster_name="logistics-{{ ds_nodash }}",
        region="us-central1",
        trigger_rule="all_done"  # Delete even if upstream failed
    )

    # ── Step 6: Send notification ──
    notify_success = EmailOperator(
        task_id="notify_success",
        to="team@company.com",
        subject="✅ Logistics Pipeline {{ ds }} Complete",
        html_content="Pipeline completed successfully for {{ ds }}",
        trigger_rule="all_success"
    )

    # ── DAG Dependencies (equivalent to Oozie workflow arrows) ──
    create_cluster >> run_pipeline >> run_quality >> refresh_metastore
    refresh_metastore >> [delete_cluster, notify_success]
'''

    print("\n" + "=" * 70)
    print("APACHE AIRFLOW DAG — PORTABLE CLOUD ORCHESTRATION")
    print("=" * 70)
    print(airflow_dag_code)

    return airflow_dag_code

# Save as: cloud_migration_readiness.py
# Run on existing on-premise cluster to assess migration readiness

from pyspark.sql import SparkSession
from pyspark.sql import functions as F
import subprocess
import sys
import os


def assess_migration_readiness():
    
    spark = SparkSession.builder \
        .appName("CloudMigrationReadiness") \
        .enableHiveSupport() \
        .getOrCreate()
    
    print("=" * 70)
    print("CLOUD MIGRATION READINESS ASSESSMENT")
    print("=" * 70)
    
    results = []
    
    # ════════════════════════════════════════
    # CHECK 1: Data Inventory
    # ════════════════════════════════════════
    
    print("\n[CHECK 1] Data Inventory on HDFS")
    try:
        result = subprocess.run(
            ["hdfs", "dfs", "-du", "-s", "-h", "/data/"],
            capture_output=True, text=True, timeout=60
        )
        print(f"  HDFS usage:\n  {result.stdout.strip()}")
        
        # Count total files
        result_count = subprocess.run(
            ["hdfs", "dfs", "-count", "/data/"],
            capture_output=True, text=True, timeout=60
        )
        parts = result_count.stdout.strip().split()
        if len(parts) >= 3:
            dir_count = int(parts[0])
            file_count = int(parts[1])
            byte_count = int(parts[2])
            
            print(f"  Directories: {dir_count}")
            print(f"  Files:       {file_count}")
            print(f"  Total bytes: {byte_count / (1024**3):.1f} GB")
            
            # Estimate migration time
            # Assume 1 Gbps upload bandwidth
            transfer_seconds = byte_count / (125 * 1024 * 1024)  # 1Gbps = 125MB/s
            transfer_hours = transfer_seconds / 3600
            
            print(f"\n  Estimated migration time at 1 Gbps: {transfer_hours:.1f} hours")
            print(f"  Estimated migration time at 10 Gbps: {transfer_hours/10:.1f} hours")
            
            if transfer_hours > 168:  # More than 1 week
                print(f"  ⚠️ RECOMMEND: Use AWS Snowball for initial migration")
            
            # Check for small files problem
            if file_count > 0:
                avg_file_size = byte_count / file_count
                avg_file_mb = avg_file_size / (1024 * 1024)
                print(f"\n  Average file size: {avg_file_mb:.1f} MB")
                
                if avg_file_mb < 10:
                    print(f"  ⚠️ WARNING: Average file size < 10MB")
                    print(f"     Small files are EXPENSIVE on cloud storage")
                    print(f"     Each S3 GET: $0.0004/1000 requests")
                    print(f"     {file_count} files × daily scan = ${file_count * 0.0004:.2f}/scan")
                    print(f"     ACTION: Compact files during migration")
                    results.append(("Small Files", False))
                else:
                    print(f"  ✅ File sizes OK for cloud")
                    results.append(("Small Files", True))
            
            results.append(("Data Inventory", True))
        else:
            results.append(("Data Inventory", False))
            
    except Exception as e:
        print(f"  ❌ Could not inventory: {e}")
        results.append(("Data Inventory", False))
    
    # ════════════════════════════════════════
    # CHECK 2: File Format Assessment
    # ════════════════════════════════════════
    
    print("\n[CHECK 2] File Format Assessment")
    
    format_counts = {"text/csv": 0, "parquet": 0, "orc": 0, "avro": 0, 
                     "sequence": 0, "other": 0}
    
    try:
        result = subprocess.run(
            ["hdfs", "dfs", "-find", "/data/", "-name", "*.csv"],
            capture_output=True, text=True, timeout=120
        )
        format_counts["text/csv"] = len(result.stdout.strip().split("\n")) if result.stdout.strip() else 0
        
        result = subprocess.run(
            ["hdfs", "dfs", "-find", "/data/", "-name", "*.parquet"],
            capture_output=True, text=True, timeout=120
        )
        format_counts["parquet"] = len(result.stdout.strip().split("\n")) if result.stdout.strip() else 0
        
        result = subprocess.run(
            ["hdfs", "dfs", "-find", "/data/", "-name", "*.orc"],
            capture_output=True, text=True, timeout=120
        )
        format_counts["orc"] = len(result.stdout.strip().split("\n")) if result.stdout.strip() else 0
        
        print(f"  Format distribution:")
        for fmt, count in format_counts.items():
            if count > 0:
                print(f"    {fmt}: {count} files")
        
        columnar_pct = (format_counts["parquet"] + format_counts["orc"]) / \
            max(sum(format_counts.values()), 1) * 100
        
        if columnar_pct > 70:
            print(f"  ✅ {columnar_pct:.0f}% columnar — cloud-ready")
            results.append(("File Formats", True))
        elif columnar_pct > 30:
            print(f"  ⚠️ {columnar_pct:.0f}% columnar — convert remaining during migration")
            results.append(("File Formats", None))
        else:
            print(f"  ❌ {columnar_pct:.0f}% columnar — MUST convert to Parquet/ORC")
            print(f"     CSV/Text on S3 is 5-50x slower than Parquet")
            results.append(("File Formats", False))
            
    except Exception as e:
        print(f"  Could not assess formats: {e}")
        results.append(("File Formats", None))
    
    # ════════════════════════════════════════
    # CHECK 3: Hive Table Inventory
    # ════════════════════════════════════════
    
    print("\n[CHECK 3] Hive Table Inventory")
    try:
        databases = spark.sql("SHOW DATABASES").collect()
        total_tables = 0
        managed_tables = 0
        external_tables = 0
        
        for db_row in databases:
            db_name = db_row.databaseName
            if db_name == "default":
                continue
            
            tables = spark.sql(f"SHOW TABLES IN {db_name}").collect()
            for table_row in tables:
                total_tables += 1
                try:
                    desc = spark.sql(
                        f"DESCRIBE EXTENDED {db_name}.{table_row.tableName}"
                    ).collect()
                    for row in desc:
                        if hasattr(row, 'col_name') and row.col_name == "Type":
                            if "EXTERNAL" in str(row.data_type).upper():
                                external_tables += 1
                            else:
                                managed_tables += 1
                except Exception:
                    pass
        
        print(f"  Total databases: {len(databases)}")
        print(f"  Total tables: {total_tables}")
        print(f"  Managed tables: {managed_tables}")
        print(f"  External tables: {external_tables}")
        
        if managed_tables > 0:
            print(f"\n  ⚠️ {managed_tables} MANAGED tables found")
            print(f"     Managed tables store data in Hive warehouse directory")
            print(f"     Must convert to EXTERNAL tables for cloud migration")
            print(f"     EXTERNAL tables point to S3 — data independent of metastore")
        
        results.append(("Hive Tables", total_tables >= 0))
        
    except Exception as e:
        print(f"  Could not inventory Hive: {e}")
        results.append(("Hive Tables", None))
    
    # ════════════════════════════════════════
    # CHECK 4: Network Connectivity to Cloud
    # ════════════════════════════════════════
    
    print("\n[CHECK 4] Network Connectivity")
    
    # Test S3 reachability
    try:
        result = subprocess.run(
            ["curl", "-s", "-o", "/dev/null", "-w", "%{http_code}",
             "--connect-timeout", "5",
             "https://s3.amazonaws.com"],
            capture_output=True, text=True, timeout=10
        )
        s3_reachable = result.stdout.strip() in ["200", "301", "307", "403"]
        print(f"  S3 reachable: {'✅ Yes' if s3_reachable else '❌ No'}")
        
        if not s3_reachable:
            print(f"  ACTION: Configure NAT gateway or VPN to AWS")
        
        results.append(("Network S3", s3_reachable))
    except Exception:
        print(f"  Could not test S3 connectivity")
        results.append(("Network S3", None))
    
    # ════════════════════════════════════════
    # CHECK 5: Security Model Assessment
    # ════════════════════════════════════════
    
    print("\n[CHECK 5] Security Model")
    
    kerberos_enabled = False
    try:
        result = subprocess.run(
            ["hdfs", "getconf", "-confKey", "hadoop.security.authentication"],
            capture_output=True, text=True, timeout=10
        )
        auth_type = result.stdout.strip()
        kerberos_enabled = auth_type.lower() == "kerberos"
        print(f"  Authentication: {auth_type}")
        
        if kerberos_enabled:
            print(f"  ⚠️ Kerberos enabled (Project 41)")
            print(f"     Cloud equivalent: IAM roles (AWS) / AAD (Azure) / IAM (GCP)")
            print(f"     Migration requires security model redesign")
            print(f"     Kerberos → IAM is NOT a 1:1 mapping")
        else:
            print(f"  Simple authentication — easier to migrate")
        
        results.append(("Security", True))
    except Exception:
        print(f"  Could not determine security model")
        results.append(("Security", None))
    
    # Check encryption
    try:
        result = subprocess.run(
            ["hdfs", "crypto", "-listZones"],
            capture_output=True, text=True, timeout=10
        )
        if result.stdout.strip():
            print(f"  Encryption zones detected (Project 42)")
            print(f"  Cloud equivalent: SSE-KMS (AWS) / Azure Storage Encryption")
        else:
            print(f"  No HDFS encryption zones")
    except Exception:
        pass
    
    # ════════════════════════════════════════
    # CHECK 6: Pipeline Dependency Map
    # ════════════════════════════════════════
    
    print("\n[CHECK 6] Pipeline Dependencies")
    print(f"""
    Components to migrate:

        )

        live_nodes = 0
        total_capacity = 0
        
        for line in lines:
            if "Live datanodes" in line:
                live_nodes = int(line.split("(")[1].split(")")[0])
            if "Present Capacity" in line:
                parts = line.split(":")
                if len(parts) > 1:
                    capacity_str = parts[1].strip().split()[0]
                    total_capacity = int(capacity_str)
        
        total_tb = total_capacity / (1024**4) if total_capacity > 0 else 0
        
        print(f"  Live DataNodes: {live_nodes}")
        print(f"  Total HDFS capacity: {total_tb:.1f} TB")
        
        # Cloud cost estimate
        s3_monthly = total_tb * 1024 * 0.023  # $0.023/GB/month S3 Standard
        compute_monthly = live_nodes * 1.0 * 8 * 30  # $1/hr × 8hrs/day × 30 days
        compute_spot = compute_monthly * 0.3  # 70% spot savings
        
        print(f"\n  Cloud cost estimate (monthly):")
        print(f"    Storage (S3 Standard):     ${s3_monthly:,.0f}")

        print(f"    Total (annual):             ${(s3_monthly + compute_spot) * 12:,.0f}/year")
