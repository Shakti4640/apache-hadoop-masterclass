AGGREGATION: AVERAGE TEMPERATURE PER STATION

  ⚠️ THIS IS TRICKY — you CANNOT just average partial averages!

  WRONG APPROACH (gives incorrect results):
    Mapper 1 sees STN_001: [20, 22, 24] → emits average = 22.0
    Mapper 2 sees STN_001: [30]         → emits average = 30.0
    Reducer averages: (22.0 + 30.0) / 2 = 26.0
    CORRECT ANSWER: (20 + 22 + 24 + 30) / 4 = 24.0 ← DIFFERENT!

  WHY WRONG:
    Mapper 1 had 3 values, Mapper 2 had 1 value
    Averaging averages ignores the WEIGHT (count) of each partial average
    This is because AVERAGE is NOT an ASSOCIATIVE operation

  RIGHT APPROACH — emit BOTH sum and count:
    Mapper: emit("STN_001", "sum:66.0,count:3")   ← from mapper 1
    Mapper: emit("STN_001", "sum:30.0,count:1")   ← from mapper 2
    Reducer: total_sum = 66.0 + 30.0 = 96.0
             total_count = 3 + 1 = 4
             average = 96.0 / 4 = 24.0 ← CORRECT!

✅ WHAT SHOULD DO

1. USE DoubleWritable for temperature values (not IntWritable)
   → Temperatures have decimal precision: 22.5, -3.7
   → IntWritable truncates to integer (22.5 → 22 — WRONG)
   → DoubleWritable preserves full precision

2. HANDLE negative temperatures
   → Many locations have temperatures below zero
   → Initialize max to Double.MIN_VALUE (not 0!)
   → Initialize min to Double.MAX_VALUE (not 0!)
   → If you start max at 0, you'll miss all negative-only stations

3. HANDLE missing/invalid sensor readings
   → Sensors malfunction: temperature might be "N/A" or "-9999"
   → -9999 is a common sentinel for "no reading"
   → Filter these out in mapper — don't let them corrupt aggregations
   → Use counters to track how many readings were filtered

4. VALIDATE temperature range
   → Earth's recorded range: -89.2°C to 56.7°C
   → Anything outside is likely a sensor error
   → Filter unreasonable values in mapper

5. FOR AVERAGE: Always emit sum AND count from mapper
   → Never emit a partial average
   → Reducer: total_average = total_sum / total_count
   → This pattern is correct regardless of data distribution across mappers

6. REUSE Writable objects (same as Project 4)
   → Create once as class fields
   → Call .set() per record

🚫 WHAT SHOULD NOT DO

1. DO NOT compute average by averaging partial averages
   → Already explained above — gives WRONG results
   → Average is NOT associative
   → ALWAYS use sum/count approach

2. DO NOT initialize max to 0 or min to 0
   → WRONG: double max = 0; → misses all negative temperatures
   → WRONG: double min = 0; → misses all positive-only stations
   → RIGHT: double max = Double.NEGATIVE_INFINITY;
   → RIGHT: double min = Double.POSITIVE_INFINITY;

3. DO NOT use Float when you need Double
   → FloatWritable: 4 bytes, ~7 significant digits
   → DoubleWritable: 8 bytes, ~15 significant digits
   → Temperature 22.123456789 → FloatWritable loses precision
   → For scientific data: ALWAYS use DoubleWritable

4. DO NOT parse CSV by splitting on ALL commas blindly
   → If a field contains a comma (quoted): split breaks
   → Our weather data has no quoted fields — split(",") is safe
   → For complex CSVs: use a proper CSV parser library
   → (Covered more in Project 12)

5. DO NOT emit the header line's values as temperature data
   → "temp_c" parsed as Double → NumberFormatException
   → Same header-skipping pattern from Project 4 applies
   → Check content: if line starts with "station_id" → skip

6. DO NOT forget that values iterator is consumed once (Project 4 rule)
   → For combined aggregation: compute ALL aggregates in ONE pass
   → Cannot iterate for max, then iterate again for min

🚫 WHEN NOT TO USE

1. DO NOT use MapReduce for MEDIAN computation
   → Median requires ALL values sorted and the middle one selected
   → With distributed data: no single reducer has ALL values (unless 1 reducer)
   → Even with 1 reducer: must hold all values in memory
   → For 1 billion readings: impossible to fit in reducer memory
   → USE INSTEAD: Approximate median algorithms, or Hive percentile functions

2. DO NOT use separate MapReduce jobs when one combined job works
   → 4 separate jobs = 4× input reads = 4× I/O cost
   → Combined job reads input ONCE = 4× faster
   → Only use separate jobs when aggregations need different keys

3. DO NOT use MapReduce for simple aggregations on small data
   → 100MB of sensor data? Python pandas computes all aggregations in seconds
   → MapReduce overhead (30-60 seconds) exceeds the actual computation time
   → RULE: MapReduce for > 1GB, simpler tools for less

4. DO NOT use DoubleWritable for currency/financial calculations
   → Double has floating-point precision issues (0.1 + 0.2 ≠ 0.3)
   → For money: use LongWritable with amounts in cents (integer arithmetic)
   → Or use Text with BigDecimal parsing in reducer

MECHANIC 2: REDUCER-SIDE MERGE AND GROUPING

┌──────────────────────────────────────────────────────────────────────┐
│ INSIDE A REDUCER TASK                                                │
│                                                                      │
│ ① COPY PHASE (shuffle):                                             │
│    → Reducer starts parallel HTTP fetches from ALL mappers          │
│    → Each mapper serves its partition-N data over HTTP              │
│    → Fetched data stored in reducer's memory buffer                 │
│    → If buffer fills → spill to reducer's local disk                │
│                                                                      │
│ ② SORT PHASE (merge):                                               │
│    → All fetched data (from all mappers) must be merged             │
│    → Since each mapper's output is already SORTED:                  │
│      → Multi-way merge sort (efficient)                             │
│    → Result: single sorted stream of all (key, value) pairs        │
│                                                                      │
│    Merged stream:                                                    │
│    ("STN_001",22.5), ("STN_001",23.1), ("STN_001",24.0),           │
│    ("STN_001",21.8), ("STN_003",15.2), ("STN_003",16.1)            │
│                                                                      │
│ ③ GROUPING:                                                         │
│    → Consecutive records with SAME KEY grouped together             │
│    → Presented to reduce() as (key, Iterable<values>)              │
│                                                                      │
│    reduce("STN_001", [22.5, 23.1, 24.0, 21.8]) ← your code called │
│    reduce("STN_003", [15.2, 16.1])              ← your code called │
│                                                                      │
│ ④ YOUR reduce() runs and emits output                               │
│    → Output written directly to HDFS (part-r-NNNNN)                │
│                                                                      │
│ TIMING NOTE:                                                         │
│ → Copy phase can OVERLAP with map phase                             │
│   (reducers start fetching as soon as mappers complete)             │
│ → Sort phase starts after ALL map output fetched                    │
│ → Reduce phase starts after sort complete                           │
│ → Progress: 0-33% = copy, 33-66% = sort, 66-100% = reduce         │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

MECHANIC 3: WHY THE VALUES ITERATOR IS SINGLE-PASS

┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│ When reduce() receives: ("STN_001", [22.5, 23.1, 24.0, 21.8])     │
│                                                                      │
│ The Iterable is NOT a List in memory!                               │
│ It is a STREAMING READ from the sorted merge output:                │
│                                                                      │
│   Sorted file on disk:                                               │
│   ... | STN_001 | 22.5 | STN_001 | 23.1 | STN_001 | 24.0 | ...   │
│           ↑ cursor                                                   │
│                                                                      │
│   First iteration: read 22.5, advance cursor                        │
│   ... | STN_001 | 22.5 | STN_001 | 23.1 | STN_001 | 24.0 | ...   │
│                            ↑ cursor                                  │
│                                                                      │
│   Second iteration: read 23.1, advance cursor                       │
│   ... | STN_001 | 22.5 | STN_001 | 23.1 | STN_001 | 24.0 | ...   │
│                                             ↑ cursor                 │
│                                                                      │
│   When key changes (STN_003 found): stop iteration for STN_001     │
│   Call reduce("STN_003", ...) — cursor continues from new position  │
│                                                                      │
│ WHY STREAMING?                                                       │
│ → A station might have 1 BILLION readings                           │
│ → Cannot fit all in memory                                          │
│ → Streaming reads them one at a time from disk                      │
│ → After reading: the value is GONE (cursor moved forward)           │
│ → Second iteration: cursor is already past all values → empty       │
│                                                                      │
│ ALSO: The IntWritable/DoubleWritable OBJECT is REUSED               │
│ → Same object in memory, .set() called with new value each time     │
│ → Storing a reference stores a pointer to an object that CHANGES    │
│ → Always extract the primitive: val.get() returns int/double        │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "Max temperature shows -Infinity for some stations"
→ REASON: Initialized max = Double.NEGATIVE_INFINITY but NO valid readings
→ All readings for that station were filtered (malformed/sentinel)
→ FIX: Track count alongside max, skip emit if count == 0

PITFALL 2: "Average calculation gives slightly different results each run"
→ REASON: Floating-point addition is NOT perfectly associative
→ (a + b) + c ≠ a + (b + c) at machine precision level
→ Different shuffle orderings → different addition order → tiny difference
→ ACCEPTABLE: Differences at 10th+ decimal place are normal
→ For exact precision: use integer arithmetic (multiply temps by 100)

PITFALL 3: "Job completes but many stations are missing from output"
→ REASON: Mapper filtered too aggressively
→ Check counters: "filtered_readings" count vs "total_readings"
→ If filtered > 50%: your filter criteria are too strict
→ FIX: Log sample filtered values, adjust thresholds

PITFALL 4: "Reducer runs out of memory on combined aggregation"
→ REASON: Storing too much data per key in reducer
→ For max/min/avg/count: only need 4 running variables — MINIMAL memory
→ If you accidentally collect all values into a List: memory explosion
→ FIX: ALWAYS use running aggregation, never collect all values

PITFALL 5: "DoubleWritable precision lost — 22.5 becomes 22.0"
→ REASON: Using IntWritable instead of DoubleWritable
→ Or: parsing with Integer.parseInt() instead of Double.parseDouble()
→ FIX: Match Writable type to data type precisely

PITFALL 6: "Combined job output has commas in values — hard to parse"
→ REASON: Using Text value with comma-separated fields
→ When TextOutputFormat writes key\tvalue: value contains commas
→ FIX: Use a clean delimiter (pipe |) or custom OutputFormat
→ Or: emit multiple separate lines per station (less efficient)

MAPPER OUTPUT SIZE — THE SCALING BOTTLENECK:

For our weather data:
→ Input: 2GB/day × 30 days = 60GB
→ Each line ~80 bytes: ~750M records
→ Mapper emits (station_id, temperature) per record
→ Each emit: ~20 bytes (8 bytes key + 8 bytes double + overhead)
→ Total mapper output: ~15GB → ALL goes through shuffle network

OPTIMIZATION PREVIEW (Project 9):
→ Combiner: local pre-aggregation on mapper side

→ Without combiner: emits 4 pairs → 4 transfers over network
→ With combiner: emits 1 pair (max=24.0) → 1 transfer

// INPUT:  CSV: station_id,timestamp,temp_celsius,humidity,wind,pressure
// OUTPUT: station_id    max:X|min:Y|avg:Z|count:N
//
// Demonstrates:
// - DoubleWritable for decimal values
// - Combined aggregation in single pass
// - Proper average computation (sum/count, NOT average of averages)

// - Custom counters for monitoring
// ══════════════════════════════════════════════════════════════════

import java.io.IOException;

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.io.DoubleWritable;
import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Job;
import org.apache.hadoop.mapreduce.Mapper;
import org.apache.hadoop.mapreduce.Reducer;
import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;
import org.apache.hadoop.mapreduce.lib.output.FileOutputFormat;


public class WeatherAnalysis {

    // ── Temperature validation constants ──
    private static final double MIN_VALID_TEMP = -90.0;   // Coldest recorded: -89.2°C
    private static final double MAX_VALID_TEMP = 60.0;    // Hottest recorded: 56.7°C
    private static final double SENTINEL_VALUE = -9999.0; // Common "no reading" marker
    
    
    // ================================================================
    // MAPPER: Extracts (station_id, temperature) from each CSV line
    // ================================================================
    
    public static class WeatherMapper 
            extends Mapper<LongWritable, Text, Text, DoubleWritable> {
        
        // Reusable output objects (Project 4 pattern)
        private final Text outputKey = new Text();
        private final DoubleWritable outputValue = new DoubleWritable();
        
        // Column indices
        private static final int COL_STATION = 0;
        private static final int COL_TIMESTAMP = 1;
        private static final int COL_TEMP = 2;
        
        @Override
        protected void map(LongWritable key, Text value, Context context)
                throws IOException, InterruptedException {
            
            String line = value.toString();
            
            // Skip header (Project 4 pattern — check content, not position)
            if (line.startsWith("station_id")) {
                context.getCounter("Weather", "header_lines").increment(1);
                return;
            }
            
            try {
                String[] fields = line.split(",");
                
                if (fields.length < 6) {
                    context.getCounter("Weather", "malformed_lines").increment(1);
                    return;
                }
                
                String stationId = fields[COL_STATION].trim();
                double temperature = Double.parseDouble(fields[COL_TEMP].trim());
                
                // ── FILTER: Sentinel values ──
                if (temperature == SENTINEL_VALUE) {
                    context.getCounter("Weather", "sentinel_filtered").increment(1);
                    return;
                }
                
                // ── FILTER: Out-of-range values ──
                if (temperature < MIN_VALID_TEMP || temperature > MAX_VALID_TEMP) {
                    context.getCounter("Weather", "range_filtered").increment(1);
                    return;
                }
                
                // ── EMIT: Valid reading ──
                outputKey.set(stationId);
                outputValue.set(temperature);
                context.write(outputKey, outputValue);
                
                context.getCounter("Weather", "valid_readings").increment(1);
                
            } catch (NumberFormatException e) {
                context.getCounter("Weather", "number_format_errors").increment(1);
            } catch (Exception e) {
                context.getCounter("Weather", "other_errors").increment(1);
            }
        }
    }
    
    
    // ================================================================
    // REDUCER: Computes max, min, average, count in SINGLE PASS
    // ================================================================
    
    public static class WeatherReducer 
            extends Reducer<Text, DoubleWritable, Text, Text> {
        
        // Note: Output value is TEXT (formatted string with all aggregations)
        // This is different from Project 4 where output value was IntWritable
        private final Text outputValue = new Text();
        
        @Override
        protected void reduce(Text key, Iterable<DoubleWritable> values, Context context)
                throws IOException, InterruptedException {
            
            // ── Initialize aggregation variables ──
            // CRITICAL: Use NEGATIVE_INFINITY for max, POSITIVE_INFINITY for min
            // NOT zero — zero would be wrong for negative-only or positive-only stations
            double max = Double.NEGATIVE_INFINITY;
            double min = Double.POSITIVE_INFINITY;
            double sum = 0.0;
            long count = 0;
            
            // ── SINGLE PASS through values iterator ──
            // REMEMBER: This iterator can only be traversed ONCE (Project 4 lesson)
            // We compute ALL FOUR aggregations in this one loop
            
            for (DoubleWritable val : values) {
                double temp = val.get();  // Extract primitive double
                
                // Update max
                if (temp > max) {
                    max = temp;
                }
                
                // Update min
                if (temp < min) {
                    min = temp;
                }
                
                // Accumulate for average
                sum += temp;
                count++;
            }
            
            // ── Compute average ──
            // CORRECT: total_sum / total_count
            // All values for this station arrived at this ONE reducer
            // So this is the TRUE average (not average of averages)
            
            if (count > 0) {
                double avg = sum / count;
                
                // Format output: max:X|min:Y|avg:Z|count:N
                String result = String.format(
                    "max:%.1f|min:%.1f|avg:%.2f|count:%d",
                    max, min, avg, count
                );
                
                outputValue.set(result);
                context.write(key, outputValue);
            }
            // If count == 0: station had no valid readings — skip output
        }
    }
    
    
    // ================================================================
    // SIMPLE MAX REDUCER (for comparison with combined approach)
    // ================================================================
    
    public static class MaxTemperatureReducer 
            extends Reducer<Text, DoubleWritable, Text, DoubleWritable> {
        
        private final DoubleWritable outputValue = new DoubleWritable();
        
        @Override
        protected void reduce(Text key, Iterable<DoubleWritable> values, Context context)
                throws IOException, InterruptedException {
            
            double max = Double.NEGATIVE_INFINITY;
            
            for (DoubleWritable val : values) {
                double temp = val.get();
                if (temp > max) {
                    max = temp;
                }
            }
            
            if (max > Double.NEGATIVE_INFINITY) {
                outputValue.set(max);
                context.write(key, outputValue);
            }
        }
    }
    
    
    // ================================================================
    // SIMPLE MIN REDUCER
    // ================================================================
    
    public static class MinTemperatureReducer 
            extends Reducer<Text, DoubleWritable, Text, DoubleWritable> {
        
        private final DoubleWritable outputValue = new DoubleWritable();
        
        @Override
        protected void reduce(Text key, Iterable<DoubleWritable> values, Context context)
                throws IOException, InterruptedException {
            
            double min = Double.POSITIVE_INFINITY;
            
            for (DoubleWritable val : values) {
                double temp = val.get();
                if (temp < min) {
                    min = temp;
                }
            }
            
            if (min < Double.POSITIVE_INFINITY) {
                outputValue.set(min);
                context.write(key, outputValue);
            }
        }
    }
    
    
    // ================================================================
    // DRIVER
    // ================================================================
    
    public static void main(String[] args) throws Exception {
        
        if (args.length < 3) {
            System.err.println("Usage: WeatherAnalysis <mode> <input_path> <output_path>");
            System.err.println("  mode: 'max', 'min', or 'combined'");
            System.err.println("");
            System.err.println("Examples:");
            System.err.println("  hadoop jar WeatherAnalysis.jar max /weather/raw/2024/01/ /weather/output/max_temps");
            System.err.println("  hadoop jar WeatherAnalysis.jar combined /weather/raw/2024/01/ /weather/output/all_stats");
            System.exit(1);
        }
        
        String mode = args[0].toLowerCase();
        String inputPath = args[1];
        String outputPath = args[2];
        
        Configuration conf = new Configuration();
        
        // Clean output directory
        FileSystem fs = FileSystem.get(conf);
        Path outPath = new Path(outputPath);
        if (fs.exists(outPath)) {
            fs.delete(outPath, true);
        }
        
        Job job = Job.getInstance(conf);
        job.setJarByClass(WeatherAnalysis.class);
        
        // Mapper is the same for all modes
        job.setMapperClass(WeatherMapper.class);
        
        // Map output types are always the same
        job.setMapOutputKeyClass(Text.class);
        job.setMapOutputValueClass(DoubleWritable.class);
        
        // ── SELECT REDUCER BASED ON MODE ──
        switch (mode) {
            case "max":
                job.setJobName("Weather Analysis — Max Temperature");
                job.setReducerClass(MaxTemperatureReducer.class);
                job.setOutputKeyClass(Text.class);
                job.setOutputValueClass(DoubleWritable.class);
                break;
                
            case "min":
                job.setJobName("Weather Analysis — Min Temperature");
                job.setReducerClass(MinTemperatureReducer.class);
                job.setOutputKeyClass(Text.class);
                job.setOutputValueClass(DoubleWritable.class);
                break;
                
            case "combined":
                job.setJobName("Weather Analysis — Combined Stats");
                job.setReducerClass(WeatherReducer.class);
                // NOTE: Combined reducer outputs Text values (not DoubleWritable)
                job.setOutputKeyClass(Text.class);
                job.setOutputValueClass(Text.class);
                break;
                
            default:
                System.err.println("Unknown mode: " + mode);
                System.err.println("Use: max, min, or combined");
                System.exit(1);
        }
        
        // One reducer — output fits in single file for 500 stations
        job.setNumReduceTasks(1);
        
        FileInputFormat.addInputPath(job, new Path(inputPath));
        FileOutputFormat.setOutputPath(job, outPath);
        
        System.out.println("============================================");
        System.out.println("Weather Analysis — Mode: " + mode.toUpperCase());
        System.out.println("Input:  " + inputPath);
        System.out.println("Output: " + outputPath);
        System.out.println("============================================");
        
        long startTime = System.currentTimeMillis();
        boolean success = job.waitForCompletion(true);
        long elapsed = System.currentTimeMillis() - startTime;
        
        System.out.println("============================================");
        System.out.println(success ? "JOB SUCCEEDED" : "JOB FAILED");
        System.out.println("Time: " + elapsed / 1000 + " seconds");
        System.out.println("============================================");
        
        System.exit(success ? 0 : 1);
    }
}
