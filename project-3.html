💡 HOW CAN WE SOLVE IT

THE MENTAL MODEL — HDFS CLI vs. LINUX CLI:

Almost every HDFS command mirrors a Linux command,
but operates on the DISTRIBUTED filesystem instead of local:

┌──────────────────────┬──────────────────────┬───────────────────────┐
│ OPERATION            │ LINUX COMMAND         │ HDFS COMMAND          │
├──────────────────────┼──────────────────────┼───────────────────────┤
│ List files           │ ls -la               │ hdfs dfs -ls          │
│ Create directory     │ mkdir -p             │ hdfs dfs -mkdir -p    │
│ Copy file in         │ cp local remote      │ hdfs dfs -put         │
│ Copy file out        │ cp remote local      │ hdfs dfs -get         │
│ Read file            │ cat file             │ hdfs dfs -cat         │
│ Read last lines      │ tail file            │ hdfs dfs -tail        │
│ Move file            │ mv src dest          │ hdfs dfs -mv          │
│ Copy within          │ cp src dest          │ hdfs dfs -cp          │
│ Remove file          │ rm file              │ hdfs dfs -rm          │
│ Remove directory     │ rm -r dir            │ hdfs dfs -rm -r       │
│ Disk usage           │ du -sh               │ hdfs dfs -du -s -h    │
│ Free space           │ df -h                │ hdfs dfs -df -h       │
│ Change permissions   │ chmod 755            │ hdfs dfs -chmod 755   │
│ Change owner         │ chown user:group     │ hdfs dfs -chown       │
│ Word count           │ wc -l                │ (no direct equivalent)│
│ Find files           │ find / -name "*.csv" │ hdfs dfs -find / -name│
└──────────────────────┴──────────────────────┴───────────────────────┘

KEY DIFFERENCE:
→ Linux commands operate on LOCAL filesystem (ext4, XFS)
→ HDFS commands send requests to NameNode/DataNodes over RPC
→ Every HDFS command involves NETWORK communication
→ This means HDFS commands have HIGHER LATENCY than local commands
→ "hdfs dfs -ls" takes ~200ms (network round trip to NameNode)
→ "ls" locally takes ~1ms

✅ WHAT SHOULD DO

1. ALWAYS use -p flag with mkdir (creates parent directories)
   → hdfs dfs -mkdir -p /telecom/raw/call_records/2024/01
   → Without -p: fails if /telecom doesn't exist yet

2. ALWAYS check before overwriting
   → hdfs dfs -test -e /path/to/file    (returns 0 if exists)
   → hdfs dfs -put will FAIL if file already exists (safe default)
   → hdfs dfs -put -f FORCES overwrite (DANGEROUS)

3. USE -du regularly to monitor space usage
   → hdfs dfs -du -s -h /telecom/*
   → Shows which directories consume most space
   → Catch runaway data growth BEFORE cluster fills up

4. USE getmerge for downloading multi-part output
   → MapReduce produces part-r-00000, part-r-00001, etc.
   → hdfs dfs -getmerge /output/dir/ local_file.txt
   → Concatenates all parts into one local file

5. ALWAYS verify after upload
   → Upload: hdfs dfs -put local_file /hdfs/path/
   → Verify: hdfs dfs -ls /hdfs/path/local_file
   → Checksum: hdfs dfs -checksum /hdfs/path/local_file

6. USE -text instead of -cat for compressed/binary files
   → hdfs dfs -cat compressed.gz → shows binary garbage
   → hdfs dfs -text compressed.gz → decompresses and shows text
   → -text auto-detects: gzip, SequenceFile, Avro

7. ORGANIZE by date partitions from the beginning
   → /data/year=2024/month=01/day=15/file.csv
   → Enables efficient data pruning in queries
   → Prevents "one giant directory" antipattern

🚫 WHAT SHOULD NOT DO

1. DO NOT use "hdfs dfs -rm -r /" 
   → Deletes EVERYTHING in HDFS — ENTIRE filesystem gone
   → There is NO "are you sure?" prompt
   → Unlike Linux, there is NO system protection for root
   → ALWAYS double-check paths before -rm -r

2. DO NOT use -put -f (force overwrite) in automated scripts
   → Silently overwrites existing data
   → If a script reruns due to a bug, previous day's output is GONE
   → Instead: check if file exists first, fail if it does

3. DO NOT download large HDFS files to your local machine
   → hdfs dfs -get /50GB_file.csv /local/path/ → fills local disk
   → Process data WHERE IT LIVES (on the cluster)
   → Download only RESULTS (small aggregated output)

4. DO NOT create deep directory hierarchies unnecessarily
   → /a/b/c/d/e/f/g/h/file.txt → 8 directory entries in NameNode
   → Each directory costs NameNode memory
   → Keep paths meaningful but not excessively deep

5. DO NOT store files without a retention/cleanup plan
   → HDFS fills up silently over weeks/months
   → Old temporary files, failed job outputs, duplicate uploads
   → Implement: hdfs dfs -rm -r /tmp/staging/* (daily cleanup cron)

6. DO NOT ignore the Trash mechanism
   → By default, -rm moves to .Trash (not actually deleted)
   → Disk space NOT freed until trash emptied
   → Use -skipTrash ONLY when certain
   → Run hdfs dfs -expunge periodically

🚫 WHEN NOT TO USE

WHEN NOT TO USE SPECIFIC COMMANDS:

1. DO NOT use -cat on files larger than ~100MB
   → Transfers entire file through network to your terminal
   → Terminal hangs, network saturated, nothing useful displayed
   → USE INSTEAD: hdfs dfs -cat /path | head -50

2. DO NOT use -get for processing data
   → Downloading to local machine defeats the purpose of Hadoop
   → Process data ON the cluster using MapReduce (Project 4)
   → -get is for SMALL result files only

3. DO NOT use -cp for large files within HDFS unnecessarily
   → -cp creates a FULL COPY (doubles storage usage)
   → -mv is metadata-only (INSTANT, no data movement)
   → If you just need to rename/reorganize: use -mv

4. DO NOT use -ls -R on directories with millions of files
   → NameNode must enumerate every file
   → Output overwhelms terminal
   → Takes minutes to complete
   → USE INSTEAD: hdfs dfs -count /path (gives summary)

5. DO NOT use -appendToFile as a regular write pattern
   → HDFS append opens the last block, extends it
   → Extremely slow compared to writing new files
   → Creates under-replicated block temporarily
   → USE INSTEAD: Write a new file, then -mv to replace

6. DO NOT use hdfs dfs commands in tight loops from scripts
   → Each command spawns a new JVM (1-2 seconds overhead)
   → 1000 iterations × 2s = 33 minutes of JVM startup alone
   → USE INSTEAD: HDFS Java API or WebHDFS REST API for bulk operations

MECHANIC 3: WHY "hdfs dfs -mv" IS INSTANT

┌──────────────────────────────────────────────────────────────────────┐
│ hdfs dfs -mv /telecom/raw/file.csv /telecom/archive/file.csv       │
│                                                                      │
│ WHAT DOES NOT HAPPEN:                                                │
│   → Data is NOT moved between DataNodes                              │
│   → Blocks are NOT copied or relocated                               │
│   → No DataNode is contacted AT ALL                                  │
│                                                                      │
│ WHAT ACTUALLY HAPPENS:                                               │
│   → Client sends RPC to NameNode: rename(old_path, new_path)        │
│   → NameNode updates its in-memory namespace:                        │
│     BEFORE: /telecom/raw/ → [file.csv → blk_001, blk_002, ...]     │
│     AFTER:  /telecom/archive/ → [file.csv → blk_001, blk_002, ...] │
│   → NameNode writes to edits log: "rename old_path → new_path"      │
│   → DONE. Takes milliseconds.                                       │
│                                                                      │
│ THIS IS WHY:                                                         │
│   → Moving a 10TB file takes the same time as moving a 10KB file    │
│   → It's just a metadata update in NameNode's memory                 │
│   → ALWAYS prefer -mv over -cp + -rm for reorganization             │
│                                                                      │
│ EXCEPTION:                                                           │
│   → -mv BETWEEN different HDFS namespaces (Federation) is NOT       │
│     instant — requires actual data copy (Project 43)                 │
└──────────────────────────────────────────────────────────────────────┘

│                                                                      │

MECHANIC 5: HOW "hdfs dfs -rm" AND TRASH WORK

┌──────────────────────────────────────────────────────────────────────┐
│ hdfs dfs -rm /telecom/raw/old_file.csv                              │
│                                                                      │
│ WITH TRASH ENABLED (fs.trash.interval > 0 in core-site.xml):        │
│   → NameNode RENAMES file to /user/<username>/.Trash/Current/...    │
│   → Blocks are NOT deleted from DataNodes                            │
│   → Disk space is NOT freed                                          │
│   → File can be RECOVERED by moving from .Trash back                 │
│   → After trash interval expires: NameNode deletes permanently       │
│                                                                      │
│ WITH TRASH DISABLED (default in some configs):                       │
│   → NameNode removes file from namespace immediately                 │
│   → NameNode marks blocks as "to be deleted"                         │
│   → On next heartbeat from DataNodes:                                │
│     NameNode → DataNode: "Delete blk_001, blk_002, ..."             │
│   → DataNodes delete block files from local disk                     │
│   → Disk space freed ASYNCHRONOUSLY (not immediate)                  │
│                                                                      │
│ hdfs dfs -rm -skipTrash /path:                                       │
│   → Bypasses trash even if enabled                                   │
│   → Permanent immediate deletion                                     │
│   → NO RECOVERY possible                                             │
│                                                                      │
│ hdfs dfs -expunge:                                                    │
│   → Empties trash (creates checkpoint of current trash)              │
│   → Previous checkpoint is deleted permanently                       │
└──────────────────────────────────────────────────────────────────────┘

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "hdfs dfs -put fails with 'File already exists'"
→ REASON: HDFS -put does NOT overwrite by default (safety feature)
→ WRONG FIX: hdfs dfs -put -f (silently overwrites — dangerous)
→ RIGHT FIX: Check first, then decide:
   hdfs dfs -test -e /path/file && echo "EXISTS" || hdfs dfs -put local /path/

PITFALL 2: "hdfs dfs -ls shows different size than du"
→ REASON: -ls shows LOGICAL size (actual data bytes)
→         -du shows LOGICAL size by default
→         -du with replication: multiply by replication factor
→ EXAMPLE:
   -ls shows: 500MB (logical)
   Actual disk used: 500MB × 3 replicas = 1.5GB
→ USE: hdfs dfs -du -s -h /path → logical size
→ Physical = logical × replication factor

PITFALL 3: "Deleted files but HDFS still shows full"
→ REASON 1: Files went to Trash (not actually deleted)
  → FIX: hdfs dfs -expunge
→ REASON 2: Block deletion is asynchronous via heartbeats
  → FIX: Wait a few minutes, DataNodes will delete blocks
→ REASON 3: Under-replicated blocks being re-replicated
  → FIX: Wait for replication to complete

PITFALL 4: "Permission denied on HDFS commands"
→ REASON: HDFS has its own permission system (mirrors POSIX)
→ Default: simple authentication (trusts OS username)
→ User "alice" cannot write to directory owned by "bob" with 755
→ FIX: hdfs dfs -chmod 777 /path (as superuser) — NOT recommended
→ BETTER: hdfs dfs -chown alice:team /path

PITFALL 5: "hdfs dfs -getmerge produces file with wrong line endings"
→ REASON: getmerge concatenates files BYTE-BY-BYTE
→ If part-r-00000 doesn't end with newline, it merges into next file's first line
→ FIX: Ensure MapReduce output always ends with newline (default does)
→ OPTION: hdfs dfs -getmerge -nl /src/ /local/file 
  → -nl adds newline between merged files

PITFALL 6: "Script using hdfs dfs in a loop is incredibly slow"
→ REASON: Each command spawns a new JVM (~2 seconds)
→ 500 file operations × 2s = 16 minutes of JVM startup
→ FIX: Use HDFS Java API or WebHDFS REST for batch operations
→ FIX: Combine operations: upload directory instead of individual files
  → hdfs dfs -put /local/dir/* /hdfs/dir/ (one JVM, many files)

import subprocess
import sys
import os
from datetime import datetime, timedelta

class HDFSManager:
    """
    Automates common HDFS operations for the telecom data pipeline.
    Wraps hdfs dfs commands with safety checks and logging.
    """
    
    def __init__(self, base_path="/telecom"):
        self.base_path = base_path
        self.log = []
    
    def _run(self, cmd):
        """Execute HDFS command and return (success, stdout, stderr)."""
        result = subprocess.run(cmd, shell=True, capture_output=True, text=True)
        success = result.returncode == 0
        self.log.append({
            "command": cmd,
            "success": success,
            "stdout": result.stdout.strip(),
            "stderr": result.stderr.strip()
        })
        return success, result.stdout.strip(), result.stderr.strip()
    
    def exists(self, hdfs_path):
        """Check if a path exists in HDFS."""
        success, _, _ = self._run(f"hdfs dfs -test -e {hdfs_path}")
        return success
    
    def is_directory(self, hdfs_path):
        """Check if a path is a directory."""
        success, _, _ = self._run(f"hdfs dfs -test -d {hdfs_path}")
        return success
    
    def is_file(self, hdfs_path):
        """Check if a path is a file."""
        success, _, _ = self._run(f"hdfs dfs -test -f {hdfs_path}")
        return success
    
    def is_empty_file(self, hdfs_path):
        """Check if a file has zero length."""
        success, _, _ = self._run(f"hdfs dfs -test -z {hdfs_path}")
        return success
    
    def mkdir(self, hdfs_path):
        """Create directory with parents (safe — idempotent)."""
        success, out, err = self._run(f"hdfs dfs -mkdir -p {hdfs_path}")
        if success:
            print(f"  [MKDIR]  ✓ {hdfs_path}")
        else:
            print(f"  [MKDIR]  ✗ {hdfs_path} — {err}")
        return success
    
    def safe_upload(self, local_path, hdfs_path, overwrite=False):
        """
        Upload with safety checks:
        - Verifies local file exists
        - Checks if HDFS target already exists
        - Only overwrites if explicitly requested
        - Verifies upload via size comparison
        """
        # Check local file exists
        if not os.path.exists(local_path):
            print(f"  [UPLOAD] ✗ Local file not found: {local_path}")
            return False
        
        local_size = os.path.getsize(local_path)
        
        # Check if HDFS target already exists
        if self.exists(hdfs_path):
            if not overwrite:
                print(f"  [UPLOAD] ⚠ SKIPPED — already exists: {hdfs_path}")
                print(f"           Set overwrite=True to force replacement")
                return False
            else:
                print(f"  [UPLOAD] ⚠ OVERWRITING existing file: {hdfs_path}")
                flag = "-f"
        else:
            flag = ""
        
        # Ensure parent directory exists
        parent_dir = "/".join(hdfs_path.split("/")[:-1])
        if not self.exists(parent_dir):
            self.mkdir(parent_dir)
        
        # Upload
        success, out, err = self._run(f"hdfs dfs -put {flag} {local_path} {hdfs_path}")
        
        if success:
            # Verify by checking HDFS file size
            hdfs_size = self.get_file_size(hdfs_path)
            if hdfs_size == local_size:
                print(f"  [UPLOAD] ✓ {local_path} → {hdfs_path}")
                print(f"           Size verified: {local_size:,} bytes")
            else:
                print(f"  [UPLOAD] ⚠ Size mismatch! Local: {local_size}, HDFS: {hdfs_size}")
        else:
            print(f"  [UPLOAD] ✗ Failed: {err}")
        
        return success
    
    def get_file_size(self, hdfs_path):
        """Get file size in bytes."""
        success, out, _ = self._run(f'hdfs dfs -stat "%b" {hdfs_path}')
        if success and out:
            return int(out)
        return -1
    
    def get_replication(self, hdfs_path):
        """Get replication factor of a file."""
        success, out, _ = self._run(f'hdfs dfs -stat "%r" {hdfs_path}')
        if success and out:
            return int(out)
        return -1
    
    def safe_delete(self, hdfs_path, skip_trash=False, recursive=False):
        """
        Delete with safety checks:
        - Refuses to delete base paths (/, /user, /telecom)
        - Confirms path exists before attempting delete
        - Logs the operation for audit
        """
        # SAFETY: Refuse to delete critical paths
        protected_paths = ["/", "/user", "/tmp", self.base_path]
        if hdfs_path.rstrip("/") in protected_paths:
            print(f"  [DELETE] ✗ REFUSED — protected path: {hdfs_path}")
            print(f"           Protected paths: {protected_paths}")
            return False
        
        # Check exists
        if not self.exists(hdfs_path):
            print(f"  [DELETE] ⚠ Path does not exist: {hdfs_path}")
            return False
        
        # Build command
        flags = ""
        if recursive:
            flags += "-r "
        if skip_trash:
            flags += "-skipTrash "
        
        success, out, err = self._run(f"hdfs dfs -rm {flags}{hdfs_path}")
        
        if success:
            trash_note = "(PERMANENT — skipped trash)" if skip_trash else "(moved to trash)"
            print(f"  [DELETE] ✓ {hdfs_path} {trash_note}")
        else:
            print(f"  [DELETE] ✗ Failed: {err}")
        
        return success
    
    def safe_move(self, src_path, dest_path):
        """
        Move with safety checks:
        - Verifies source exists
        - Verifies destination parent exists
        - Confirms move completed
        """
        if not self.exists(src_path):
            print(f"  [MOVE]   ✗ Source not found: {src_path}")
            return False
        
        # Ensure destination parent exists
        dest_parent = "/".join(dest_path.split("/")[:-1])
        if not self.exists(dest_parent):
            self.mkdir(dest_parent)
        
        success, out, err = self._run(f"hdfs dfs -mv {src_path} {dest_path}")
        
        if success:
            print(f"  [MOVE]   ✓ {src_path} → {dest_path}")
            print(f"           (metadata-only — instant regardless of file size)")
        else:
            print(f"  [MOVE]   ✗ Failed: {err}")
        
        return success
    
    def list_files(self, hdfs_path, recursive=False, human_readable=True):
        """List files with parsed output."""
        flags = ""
        if recursive:
            flags += "-R "
        if human_readable:
            flags += "-h "
        
        success, out, err = self._run(f"hdfs dfs -ls {flags}{hdfs_path}")
        
        if not success:
            print(f"  [LIST]   ✗ Cannot list: {hdfs_path} — {err}")
            return []
        
        files = []
        for line in out.split("\n"):
            if line.startswith("Found") or not line.strip():
                continue
            parts = line.split()
            if len(parts) >= 8:
                files.append({
                    "permissions": parts[0],
                    "replication": parts[1],
                    "owner": parts[2],
                    "group": parts[3],
                    "size": parts[4],
                    "date": parts[5],
                    "time": parts[6],
                    "path": parts[7]
                })
        
        return files
    
    def get_disk_usage(self, hdfs_path):
        """Get disk usage summary."""
        success, out, _ = self._run(f"hdfs dfs -du -s -h {hdfs_path}")
        if success:
            return out
        return "Unable to determine"
    
    def get_cluster_status(self):
        """Get overall HDFS cluster status."""
        success, out, _ = self._run("hdfs dfs -df -h /")
        if success:
            return out
        return "Unable to determine"
    
    def read_head(self, hdfs_path, num_lines=10):
        """Safely read first N lines of a file."""
        success, out, _ = self._run(f"hdfs dfs -cat {hdfs_path} | head -{num_lines}")
        if success:
            return out
        return ""
    
    def count_lines(self, hdfs_path):
        """Count lines in an HDFS file (streams entire file — use sparingly)."""
        success, out, _ = self._run(f"hdfs dfs -cat {hdfs_path} | wc -l")
        if success and out:
            return int(out.strip())
        return -1
    
    def get_block_info(self, hdfs_path):
        """Get block-level information for a file."""
        success, out, _ = self._run(
            f"hdfs fsck {hdfs_path} -files -blocks -locations 2>/dev/null"
        )
        if success:
            return out
        return "Unable to retrieve block info"
    
    def verify_integrity(self, hdfs_path):
        """Verify file integrity via checksum."""
        success, out, _ = self._run(f"hdfs dfs -checksum {hdfs_path}")
        if success:
            return out
        return "Unable to verify"
    
    def archive_old_data(self, source_dir, archive_dir, days_old=30):
        """
        Move files older than N days from source to archive.
        Real-world pattern for data lifecycle management.
        """
        print(f"\n  [ARCHIVE] Scanning {source_dir} for files older than {days_old} days...")
        
        files = self.list_files(source_dir)
        cutoff_date = datetime.now() - timedelta(days=days_old)
        moved_count = 0
        
        for f in files:
            if f["permissions"].startswith("d"):
                continue  # skip directories
            
            try:
                file_date = datetime.strptime(f["date"], "%Y-%m-%d")
                if file_date < cutoff_date:
                    filename = f["path"].split("/")[-1]
                    dest = f"{archive_dir}/{filename}"
                    if self.safe_move(f["path"], dest):
                        moved_count += 1
            except ValueError:
                continue
        
        print(f"  [ARCHIVE] Moved {moved_count} files to {archive_dir}")
        return moved_count
    
    def generate_storage_report(self):
        """Generate a comprehensive storage report."""
        print(f"\n{'='*70}")
        print(f"  HDFS STORAGE REPORT — {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
        print(f"{'='*70}")
        
        # Cluster capacity
        print(f"\n  CLUSTER CAPACITY:")
        print(f"  {self.get_cluster_status()}")
        
        # Directory breakdown
        print(f"\n  DIRECTORY BREAKDOWN:")
        success, out, _ = self._run(f"hdfs dfs -du -h {self.base_path}/")
        if success:
            for line in out.split("\n"):
                if line.strip():
                    print(f"    {line}")
        
        # File counts
        print(f"\n  FILE COUNTS:")
        success, out, _ = self._run(f"hdfs dfs -count {self.base_path}/raw/")
        if success:
            parts = out.split()
            if len(parts) >= 3:
                print(f"    Raw data:    {parts[0]} dirs, {parts[1]} files, {int(parts[2]):,} bytes")
        
        success, out, _ = self._run(f"hdfs dfs -count {self.base_path}/processed/")
        if success:
            parts = out.split()
            if len(parts) >= 3:
                print(f"    Processed:   {parts[0]} dirs, {parts[1]} files, {int(parts[2]):,} bytes")
        
        success, out, _ = self._run(f"hdfs dfs -count {self.base_path}/archive/")
        if success:
            parts = out.split()
            if len(parts) >= 3:
                print(f"    Archive:     {parts[0]} dirs, {parts[1]} files, {int(parts[2]):,} bytes")
        
        # Largest files
        print(f"\n  LARGEST FILES (Top 5):")
        files = self.list_files(f"{self.base_path}/", recursive=True)
        file_entries = [f for f in files if not f["permissions"].startswith("d")]
        
        # Sort by size (approximate — human readable makes exact sort difficult)
        for i, f in enumerate(file_entries[:5]):
            print(f"    {i+1}. {f['size']:>8}  {f['path']}")
        
        # Operation log summary
        print(f"\n  OPERATIONS THIS SESSION:")
        success_count = sum(1 for entry in self.log if entry["success"])
        fail_count = sum(1 for entry in self.log if not entry["success"])
        print(f"    Total commands: {len(self.log)}")
        print(f"    Successful:     {success_count}")
        print(f"    Failed:         {fail_count}")
        
        print(f"\n{'='*70}")
    
    def print_operation_log(self, last_n=None):
        """Print the operation audit log."""

        print(f"\n  OPERATION AUDIT LOG (last {len(entries)} operations):")
        print(f"  {'─'*65}")
        
        for i, entry in enumerate(entries):

            cmd = entry["command"]
            if len(cmd) > 60:
                cmd = cmd[:57] + "..."
            print(f"  {status}  {cmd}")
        
        print(f"  {'─'*65}")
