PROJECT 32: WRITING A CUSTOM INPUTFORMAT TO PARSE BINARY SENSOR DATA

🔴 HYPOTHETICAL PROBLEM

SCENARIO:
→ You are a data engineer at an industrial IoT company
→ 50,000 sensors on factory floors generate readings every second
→ Each sensor writes data in a PROPRIETARY BINARY FORMAT
→ Daily volume: ~200GB of binary sensor files

WHAT FAILS WITH STANDARD INPUTFORMATS:

→ TextInputFormat (Project 12): Splits on newlines (\n)
  → Binary data has NO newlines — or has accidental 0x0A bytes
  → Completely garbled output

→ SequenceFile (Project 30): Expects Hadoop's SequenceFile format
  → These files are NOT SequenceFiles
  → Cannot read them at all

→ NLineInputFormat (Project 12): Still assumes text lines
  → Same problem as TextInputFormat

→ WholeFileInputFormat: One mapper per file
  → With 50,000 files, that's 50,000 mappers — cluster melts
  → Each file is 4MB — too small for individual mappers

THE QUESTION:
→ How do you teach Hadoop to read a completely custom binary format
  that no existing InputFormat understands?
→ How do you ensure CORRECT block splitting so binary records
  don't get cut in the middle?
→ How do you handle the file header that appears only at offset 0?

THE ANSWER: Write a Custom InputFormat + Custom RecordReader

STEP 2: Identify the CRITICAL challenge — split boundaries

  PROBLEM: HDFS splits files at BYTE boundaries (128MB blocks)
  
  A 400MB binary file with 28-byte records:
  
  Block 1: bytes 0 — 134,217,727 (128MB)
  Block 2: bytes 134,217,728 — 268,435,455
  Block 3: bytes 268,435,456 — 400,000,000
  
  The split boundary at byte 134,217,727 almost certainly
  falls IN THE MIDDLE of a 28-byte record:
  
  ... [Record N: bytes 134,217,710 — 134,217,737] ...
                                    ↑
                          SPLIT BOUNDARY HERE
                          (cuts record in half!)
  
  TextInputFormat handles this for TEXT by:
  → First mapper reads past split end to finish the last line
  → Second mapper skips to the first complete line after split start
  
  WE MUST DO THE SAME for binary records:
  → Align to record boundaries
  → Skip the file header (32 bytes) only in the first split
  → Handle partial records at split boundaries

💡 HOW CAN WE SOLVE IT

DESIGN THE SPLIT AND READ STRATEGY:

═══════════════════════════════════════════════════════════
STRATEGY: Record-Aligned Splitting
═══════════════════════════════════════════════════════════

FILE LAYOUT (simplified):
  [HEADER: 32 bytes][Record 0][Record 1]...[Record N]
                     28 bytes  28 bytes     28 bytes

DATA START OFFSET = 32 (after header)
RECORD SIZE = 28 bytes
RECORDS ARE FIXED-LENGTH → we can calculate exact boundaries

SPLIT ALIGNMENT LOGIC:

  Given: split start byte = S, split length = L
  
  CASE 1: S == 0 (first split)
    → Start reading at byte 32 (skip header)
    → Read records until reaching S + L
    → If last record extends past S + L, STILL READ IT
      (complete the partial record)
  
  CASE 2: S > 0 (subsequent splits)
    → S might be in the middle of a record
    → Calculate: records_before = (S - HEADER_SIZE) / RECORD_SIZE
    → If (S - HEADER_SIZE) % RECORD_SIZE != 0:
        → We're mid-record — advance to next record boundary
        → adjusted_start = HEADER_SIZE + (records_before + 1) * RECORD_SIZE
    → Read records from adjusted_start until S + L
    → Complete the last partial record as in Case 1

VISUAL EXAMPLE:

  File: [HDR|R0|R1|R2|R3|R4|R5|R6|R7|R8|R9|R10|R11|...]
  
  Split 1: bytes 0-99
    → Skip header (32 bytes)
    → Start at byte 32, read R0 (bytes 32-59)
    → Read R1 (bytes 60-87)
    → R2 starts at byte 88, ends at byte 115 → PAST split end
    → Still read R2 completely (read past split boundary)
    → Split 1 processes: R0, R1, R2
  
  Split 2: bytes 100-199
    → byte 100 is inside R2 (bytes 88-115) → skip to R3
    → R3 starts at byte 116, read it
    → Continue reading until byte 199...
    → Split 2 processes: R3, R4, R5 (etc.)
  
  GUARANTEE: Every record is read EXACTLY ONCE
  → No duplicates, no gaps

═══════════════════════════════════════════════════════════
ALTERNATIVE STRATEGY: Non-Splittable Files
═══════════════════════════════════════════════════════════

  For simpler cases, you can mark files as NON-SPLITTABLE:
  → Override isSplitable() to return false
  → Each file = one mapper

✅ WHAT SHOULD DO

1. EXTEND FileInputFormat, NOT InputFormat DIRECTLY
   → FileInputFormat provides file listing, split calculation, status reporting
   → You only override createRecordReader() and optionally isSplitable()
   → Writing InputFormat from scratch means reimplementing 500+ lines of split logic

2. USE FIXED-LENGTH RECORDS WHEN POSSIBLE
   → Fixed-length records allow mathematical split alignment
   → Variable-length records require scanning for delimiters — much harder
   → If you control the format: CHOOSE fixed-length

3. READ BINARY DATA USING DataInputStream OR FSDataInputStream
   → FSDataInputStream supports seek() for random access
   → Essential for jumping to split start positions
   → readInt(), readLong(), readFloat() handle endianness

4. HANDLE BYTE ORDER EXPLICITLY
   → Java uses BIG-ENDIAN by default
   → Many hardware devices write LITTLE-ENDIAN
   → Use ByteBuffer with ByteOrder.LITTLE_ENDIAN if needed
   → Getting this wrong → all numbers are garbage

5. IMPLEMENT getProgress() ACCURATELY
   → Framework uses this for speculative execution decisions
   → If progress appears stuck, Hadoop may launch duplicate tasks
   → Calculate: (current_position - split_start) / split_length

6. WRITE A DATA GENERATOR FOR TESTING
   → You need binary test files to validate your InputFormat
   → Generate files with known values → verify MapReduce output matches
   → Include edge cases: empty files, single-record files, exact split boundaries

🚫 WHAT SHOULD NOT DO

1. DO NOT ASSUME SPLIT START == FILE START
   → Only the FIRST split starts at byte 0
   → Subsequent splits start at arbitrary byte offsets
   → You MUST handle mid-record start positions

2. DO NOT READ PAST YOUR SPLIT WITHOUT BOUNDS
   → Reading one extra record past split end is CORRECT (completing partial record)
   → Reading ALL remaining records past split end is WRONG
   → That causes DUPLICATE processing — next split's mapper also reads those records

3. DO NOT IGNORE THE FILE HEADER IN NON-FIRST SPLITS
   → Split 2 starts at byte 134,217,728
   → There is NO header at this position
   → Only seek past header when split starts at byte 0
   → Reading 32 bytes of record data as header → corruption

4. DO NOT USE BufferedReader OR InputStreamReader
   → These are for TEXT data (character decoding)
   → Binary data must use raw byte streams
   → DataInputStream or ByteBuffer for structured binary reads

5. DO NOT FORGET TO CLOSE STREAMS IN close()
   → RecordReader.close() is called when mapper finishes
   → Leaked file handles accumulate across thousands of tasks
   → Eventually: "Too many open files" error crashes DataNode

6. DO NOT MAKE RecordReader STATEFUL BEYOND CURRENT RECORD
   → Each call to nextKeyValue() should advance to next record
   → Don't accumulate records in memory (that's the Mapper's job)
   → RecordReader is a CURSOR, not a COLLECTION

🚫 WHEN NOT TO USE

1. WHEN A STANDARD INPUTFORMAT WORKS
   → Custom InputFormat is complex — 200+ lines of code
   → If you can convert binary to text first (preprocessing step) → simpler
   → Only build custom when conversion is impractical at scale

2. WHEN FILES ARE VERY SMALL (< 1MB each)
   → Custom InputFormat won't fix the small files problem (Project 1 pitfall)
   → NameNode still overwhelmed by metadata for millions of tiny files
   → FIRST: merge small files into larger SequenceFiles or HAR archives
   → THEN: read with SequenceFileInputFormat

3. WHEN FORMAT CHANGES FREQUENTLY
   → Custom InputFormat is compiled Java — needs rebuild + redeploy
   → If sensor firmware updates change the binary format every month
   → Consider: Avro (schema evolution) or a flexible preprocessing layer

4. WHEN YOU NEED COLUMNAR ACCESS
   → Custom binary format reads ALL fields for every record
   → If queries only need 2 of 5 fields → wasted I/O
   → Convert to Parquet/ORC (Project 30) for columnar access
   → Use custom InputFormat for INGESTION, then convert for ANALYSIS

MECHANIC 2: HOW SPLIT ALIGNMENT WORKS FOR BINARY RECORDS
═══════════════════════════════════════════════════════════════

  FILE: 1000 bytes total
  HEADER: 32 bytes
  RECORD SIZE: 28 bytes
  DATA REGION: bytes 32 — 999 = 968 bytes = 34.57 records → 34 complete records
  
  HDFS creates 2 splits (hypothetical small split size for illustration):
  
  Split 0: start=0,    length=500
  Split 1: start=500,  length=500
  
  SPLIT 0 PROCESSING:
  ┌─────────────────────────────────────────────────────────┐
  │ initialize():                                            │
  │   splitStart = 0                                         │
  │   splitEnd = 0 + 500 = 500                               │
  │   isFirstSplit = (splitStart == 0) → TRUE                │
  │   dataStart = 32 (skip header)                           │
  │   seek(32)                                               │
  │   position = 32                                          │
  │                                                          │
  │ nextKeyValue() calls:                                    │
  │   Read record at byte 32  (R0)  → position = 60  → TRUE │
  │   Read record at byte 60  (R1)  → position = 88  → TRUE │
  │   Read record at byte 88  (R2)  → position = 116 → TRUE │
  │   ...                                                    │
  │   Read record at byte 480 (R16) → position = 508 → TRUE │
  │   ↑ 508 > splitEnd(500) BUT we started reading at 480   │
  │     which is < splitEnd, so we COMPLETE this record      │
  │   position = 508 > splitEnd → STOP                       │
  │   nextKeyValue() → FALSE                                 │
  │                                                          │
  │ Records processed: R0 through R16 (17 records)           │
  └─────────────────────────────────────────────────────────┘

  SPLIT 1 PROCESSING:
  ┌─────────────────────────────────────────────────────────┐
  │ initialize():                                            │
  │   splitStart = 500                                       │
  │   splitEnd = 500 + 500 = 1000                            │
  │   isFirstSplit = (splitStart == 0) → FALSE               │
  │                                                          │
  │   ALIGNMENT CALCULATION:                                 │
  │   bytesIntoDataRegion = 500 - 32 = 468                   │
  │   offsetInRecord = 468 % 28 = 16                         │
  │   → We are 16 bytes INTO a record                        │
  │   → Skip remaining 28 - 16 = 12 bytes to next boundary  │
  │   alignedStart = 500 + 12 = 512                          │
  │   seek(512)                                              │
  │   position = 512                                         │
  │                                                          │
  │   VERIFICATION:                                          │
  │   (512 - 32) / 28 = 480 / 28 = 17.14... WRONG!          │
  │   Wait — 512 - 32 = 480, 480 / 28 = 17.14               │
  │   That means 512 is NOT a record boundary!               │
  │                                                          │
  │   CORRECT ALIGNMENT:                                     │
  │   nextRecordStart = HEADER + ceil((500 - HEADER) / RECORD) * RECORD│
  │   = 32 + ceil(468 / 28) * 28                             │
  │   = 32 + 17 * 28                                         │
  │   = 32 + 476 = 508                                       │
  │   seek(508) → This is R17's start                        │
  │   position = 508                                         │
  │                                                          │
  │   NOTICE: R16 ended at byte 508, R17 starts at 508      │
  │   Split 0 read R16 (bytes 480-507), Split 1 starts R17  │
  │   → NO GAP, NO OVERLAP ✓                                │
  │                                                          │
  │ nextKeyValue() calls:                                    │
  │   Read record at byte 508 (R17) → position = 536 → TRUE │
  │   Read record at byte 536 (R18) → position = 564 → TRUE │
  │   ...                                                    │
  │   Read until splitEnd (1000) or EOF                      │
  └─────────────────────────────────────────────────────────┘

MECHANIC 3: FSDataInputStream SEEK BEHAVIOR
═══════════════════════════════════════════════════════════════

  HDFS files are stored as blocks on DataNodes.
  FSDataInputStream abstracts this:

  ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
  │ Block 1      │      │ Block 2      │      │ Block 3      │
  │ (DataNode A) │      │ (DataNode B) │      │ (DataNode C) │
  │ bytes 0-127M │      │ bytes 128M-  │      │ bytes 256M-  │
  └──────────────┘      └──────────────┘      └──────────────┘
         ↑                     ↑                     ↑
         └─────────────────────┼─────────────────────┘
                               │
                    ┌──────────┴──────────┐
                    │  FSDataInputStream   │
                    │                      │
                    │  seek(300MB)         │
                    │  → Connects to       │
                    │    DataNode C        │
                    │  → Reads from local  │
                    │    block position    │
                    │                      │
                    │  read(buffer, 0, 28) │
                    │  → Returns 28 bytes  │
                    │    of record data    │
                    └─────────────────────┘

  KEY BEHAVIORS:
  → seek() is O(1) — doesn't read intervening data
  → Reading past a block boundary transparently switches to next block
  → Data locality: if mapper runs on DataNode A, reading Block 1 is LOCAL
  → Reading Block 2 from DataNode A requires NETWORK transfer

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "All values are garbage numbers"
→ CAUSE: Byte order (endianness) mismatch
→ Java DataInputStream reads BIG-ENDIAN
→ If sensor writes LITTLE-ENDIAN: temperature "22.5°C" reads as "1.46E-37"
→ SOLUTION: Use ByteBuffer with explicit byte order (shown in code)

PITFALL 2: "Some records appear duplicated"
→ CAUSE: Split alignment off by one record
→ Both Split N and Split N+1 read the boundary record
→ SOLUTION: Strict rule — a record belongs to the split that
  contains its FIRST byte. Read past split end to complete,
  skip partial record at split start.

PITFALL 3: "First mapper works, subsequent mappers produce garbage"
→ CAUSE: Non-first mappers trying to read file header as record data
→ byte 500 is NOT a header — it's mid-data
→ SOLUTION: Only read header when split start == 0

PITFALL 4: "EOFException at random points"
→ CAUSE: Last record in file is incomplete (file truncated)
→ Sensor crashed mid-write → partial record at end
→ SOLUTION: Wrap reads in try-catch, skip incomplete final records
→ Log a counter (Project 7) for truncated records

PITFALL 5: "CRC checksum error on read"
→ CAUSE: Binary file was corrupted during HDFS upload
→ HDFS verifies checksums on every read
→ SOLUTION: Re-upload the file, check source for corruption
→ NOT an InputFormat bug — it's a data integrity issue

PITFALL 6: "Mapper runs but processes 0 records"
→ CAUSE: Aligned start position > split end
→ The entire split falls within ONE record that the previous split already read
→ This is CORRECT behavior for very small splits — not an error
→ SOLUTION: Ensure split size >> record size (default 128MB >> 28 bytes = fine)

📈 SCALABILITY CONSIDERATIONS

1. RECORD SIZE vs. SPLIT SIZE RATIO
   → Record: 28 bytes, Split: 128MB = 4.7 million records per split
   → Alignment overhead: at most 1 record per split = negligible
   → If records were 100MB each → 1 record per split → alignment matters more

2. HEADER PARSING OVERHEAD
   → Header read once per FILE (not per split)
   → Only first split reads header
   → For metadata needed by all mappers → use Distributed Cache (Project 15)

3. FILE COUNT vs. SPLIT COUNT
   ┌───────────────────────────────────┬─────────────────────────┐
   │ SCENARIO                          │ MAPPERS LAUNCHED         │
   ├───────────────────────────────────┼─────────────────────────┤
   │ 1 × 10GB file (splittable)       │ ~80 mappers (128MB each)│
   │ 50,000 × 4MB files (splittable)  │ 50,000 mappers (1 each) │
   │ 50,000 × 4MB files (non-split)   │ 50,000 mappers (1 each) │
   │ 100 × 2GB merged files (split)   │ ~1,600 mappers          │
   └───────────────────────────────────┴─────────────────────────┘
   
   → 50,000 mappers is TOO MANY for small files
   → SOLUTION: Merge sensor files by hour/day before processing
   → Use CombineFileInputFormat for automatic merging of small files

4. SEEK PERFORMANCE ON REMOTE BLOCKS
   → seek() to a remote block requires network connection setup
   → Each mapper ideally reads LOCAL blocks (data locality from Project 1)
   → Custom InputFormat inherits FileInputFormat's locality-aware split placement

5. MEMORY USAGE IN RecordReader
   → Each record: 28 bytes — negligible memory
   → Buffer for reading: configurable (default 4KB adequate)
   → No accumulation — stream processing pattern
   → SAFE even for multi-TB files

package com.iot.sensor;

import org.apache.hadoop.io.Writable;

import java.io.DataInput;
import java.io.DataOutput;
import java.io.IOException;
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.TimeZone;

public class SensorReading implements Writable {

    // Fields matching the binary record structure
    private String sensorId;       // From file header (shared across records)
    private long timestamp;        // 8 bytes — milliseconds since epoch
    private float temperature;     // 4 bytes — Celsius
    private float humidity;        // 4 bytes — percentage
    private float pressure;        // 4 bytes — hectopascals
    private int status;            // 4 bytes — enum: 0/1/2/3

    // Status code constants
    public static final int STATUS_NORMAL = 0;
    public static final int STATUS_WARNING = 1;
    public static final int STATUS_CRITICAL = 2;
    public static final int STATUS_OFFLINE = 3;

    // Date formatter for human-readable output
    private static final SimpleDateFormat DATE_FORMAT;
    static {
        DATE_FORMAT = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        DATE_FORMAT.setTimeZone(TimeZone.getTimeZone("UTC"));
    }

    // Default constructor required by Hadoop serialization
    public SensorReading() {}

    public SensorReading(String sensorId, long timestamp, 
                         float temperature, float humidity, 
                         float pressure, int status) {
        this.sensorId = sensorId;
        this.timestamp = timestamp;
        this.temperature = temperature;
        this.humidity = humidity;
        this.pressure = pressure;
        this.status = status;
    }

    // ── Writable interface: serialize to binary ──
    @Override
    public void write(DataOutput out) throws IOException {
        out.writeUTF(sensorId);
        out.writeLong(timestamp);
        out.writeFloat(temperature);
        out.writeFloat(humidity);
        out.writeFloat(pressure);
        out.writeInt(status);
    }

    // ── Writable interface: deserialize from binary ──
    @Override
    public void readFields(DataInput in) throws IOException {
        sensorId = in.readUTF();
        timestamp = in.readLong();
        temperature = in.readFloat();
        humidity = in.readFloat();
        pressure = in.readFloat();
        status = in.readInt();
    }

    // ── Getters ──
    public String getSensorId() { return sensorId; }
    public long getTimestamp() { return timestamp; }
    public float getTemperature() { return temperature; }
    public float getHumidity() { return humidity; }
    public float getPressure() { return pressure; }
    public int getStatus() { return status; }

    // ── Setters (used by RecordReader to reuse object) ──
    public void setSensorId(String sensorId) { this.sensorId = sensorId; }
    public void setTimestamp(long timestamp) { this.timestamp = timestamp; }
    public void setTemperature(float temperature) { this.temperature = temperature; }
    public void setHumidity(float humidity) { this.humidity = humidity; }
    public void setPressure(float pressure) { this.pressure = pressure; }
    public void setStatus(int status) { this.status = status; }

    // ── Derived getters ──
    public String getTimestampString() {
        return DATE_FORMAT.format(new Date(timestamp));
    }

    public String getHourKey() {
        SimpleDateFormat hourFormat = new SimpleDateFormat("yyyy-MM-dd-HH");
        hourFormat.setTimeZone(TimeZone.getTimeZone("UTC"));
        return hourFormat.format(new Date(timestamp));
    }

    public String getStatusString() {
        switch (status) {
            case STATUS_NORMAL:   return "NORMAL";
            case STATUS_WARNING:  return "WARNING";
            case STATUS_CRITICAL: return "CRITICAL";
            case STATUS_OFFLINE:  return "OFFLINE";
            default:              return "UNKNOWN(" + status + ")";
        }
    }

    @Override
    public String toString() {
        return String.format("%s\t%s\t%.2f\t%.2f\t%.2f\t%s",
            sensorId, getTimestampString(), temperature, 
            humidity, pressure, getStatusString());
    }
}

// FILE 2: SensorRecordReader.java — THE CORE: READS BINARY DATA
// ═══════════════════════════════════════════════════════════════
// This is where the REAL work happens.
// Reads raw binary bytes and produces SensorReading objects.

package com.iot.sensor;

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.fs.FSDataInputStream;
import org.apache.hadoop.fs.FileSystem;
import org.apache.hadoop.fs.Path;
import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.mapreduce.InputSplit;
import org.apache.hadoop.mapreduce.RecordReader;
import org.apache.hadoop.mapreduce.TaskAttemptContext;
import org.apache.hadoop.mapreduce.lib.input.FileSplit;
import org.apache.hadoop.mapreduce.Counter;

import java.io.IOException;
import java.nio.ByteBuffer;
import java.nio.ByteOrder;

public class SensorRecordReader extends RecordReader<LongWritable, SensorReading> {

    // ── Binary format constants ──
    // These match the sensor file specification from the problem statement
    private static final int HEADER_SIZE = 32;
    private static final int RECORD_SIZE = 28;
    private static final byte[] MAGIC_BYTES = "SNSR".getBytes();
    private static final int MAGIC_SIZE = 4;
    private static final int VERSION_SIZE = 2;
    private static final int SENSOR_ID_SIZE = 10;
    private static final int RECORD_COUNT_SIZE = 8;
    private static final int RESERVED_SIZE = 8;

    // ── State variables ──
    private FSDataInputStream fileIn;        // The file stream
    private long splitStart;                 // Where this split begins (byte offset)
    private long splitEnd;                   // Where this split ends
    private long currentPosition;            // Current read position
    private long alignedStart;               // Actual start after alignment
    private boolean isFirstSplit;            // Whether this split starts at byte 0

    // ── Parsed header data ──
    private String sensorId;                 // Extracted from file header
    private short fileVersion;               // Format version from header
    private long expectedRecordCount;        // Record count from header

    // ── Output key-value pair (reused to reduce GC pressure) ──
    private LongWritable currentKey = new LongWritable();
    private SensorReading currentValue = new SensorReading();
    private long recordsRead = 0;

    // ── Buffer for reading one record ──
    private byte[] recordBuffer = new byte[RECORD_SIZE];
    private ByteBuffer byteBuffer;

    // ── Counters for monitoring ──
    private Counter totalRecordsCounter;
    private Counter skippedAlignmentCounter;
    private Counter truncatedRecordsCounter;
    private Counter invalidHeaderCounter;

    // ── Byte order configuration ──
    // Sensors often use little-endian (Intel byte order)
    // Java default is big-endian — we must handle this explicitly
    private ByteOrder byteOrder = ByteOrder.BIG_ENDIAN;

    @Override
    public void initialize(InputSplit genericSplit, TaskAttemptContext context) 
            throws IOException {
        
        // ── Step 1: Extract split information ──
        FileSplit split = (FileSplit) genericSplit;
        Configuration conf = context.getConfiguration();
        Path filePath = split.getPath();
        
        splitStart = split.getStart();
        splitEnd = splitStart + split.getLength();
        isFirstSplit = (splitStart == 0);

        // ── Step 2: Configure byte order from job configuration ──
        String orderStr = conf.get("sensor.byte.order", "BIG_ENDIAN");
        byteOrder = orderStr.equals("LITTLE_ENDIAN") 
                    ? ByteOrder.LITTLE_ENDIAN 
                    : ByteOrder.BIG_ENDIAN;
        byteBuffer = ByteBuffer.wrap(recordBuffer).order(byteOrder);

        // ── Step 3: Initialize counters ──
        totalRecordsCounter = context.getCounter(
            "SensorInputFormat", "TOTAL_RECORDS_READ");
        skippedAlignmentCounter = context.getCounter(
            "SensorInputFormat", "RECORDS_SKIPPED_ALIGNMENT");
        truncatedRecordsCounter = context.getCounter(
            "SensorInputFormat", "RECORDS_TRUNCATED");
        invalidHeaderCounter = context.getCounter(
            "SensorInputFormat", "INVALID_MAGIC_HEADER");

        // ── Step 4: Open the file ──
        FileSystem fs = filePath.getFileSystem(conf);
        fileIn = fs.open(filePath);

        // ── Step 5: Read file header (always — we need sensorId) ──
        // Even non-first splits need the sensorId from the header
        // We seek to byte 0, read header, then seek to our split position
        readFileHeader();

        // ── Step 6: Calculate aligned start position ──
        if (isFirstSplit) {
            // First split: start right after the header
            alignedStart = HEADER_SIZE;
        } else {
            // Non-first split: align to next record boundary
            // 
            // ALGORITHM:
            // 1. Calculate how many bytes into the data region our split starts
            // 2. If not on a record boundary, advance to the next one
            // 3. The previous split's RecordReader will have read past its
            //    split end to complete the last record — so the boundary
            //    record is already handled
            
            long bytesIntoData = splitStart - HEADER_SIZE;
            long remainder = bytesIntoData % RECORD_SIZE;
            
            if (remainder == 0) {
                // Lucky — split starts exactly on a record boundary
                alignedStart = splitStart;
            } else {
                // Skip to next record boundary
                long bytesToSkip = RECORD_SIZE - remainder;
                alignedStart = splitStart + bytesToSkip;
                skippedAlignmentCounter.increment(1);
            }
        }

        // ── Step 7: Seek to our aligned start position ──
        fileIn.seek(alignedStart);
        currentPosition = alignedStart;
    }

    /**
     * Read and validate the 32-byte file header.
     * Structure: [Magic:4][Version:2][SensorID:10][RecordCount:8][Reserved:8]
     */
    private void readFileHeader() throws IOException {
        // Save current position (we might need to seek back)
        fileIn.seek(0);

        // Read magic bytes
        byte[] magic = new byte[MAGIC_SIZE];
        fileIn.readFully(magic);
        
        // Validate magic bytes
        for (int i = 0; i < MAGIC_SIZE; i++) {
            if (magic[i] != MAGIC_BYTES[i]) {
                invalidHeaderCounter.increment(1);
                throw new IOException(
                    "Invalid sensor file: expected magic 'SNSR', got '" 
                    + new String(magic) + "'");
            }
        }

        // Read version (2 bytes)
        byte[] versionBytes = new byte[VERSION_SIZE];
        fileIn.readFully(versionBytes);
        ByteBuffer vb = ByteBuffer.wrap(versionBytes).order(byteOrder);
        fileVersion = vb.getShort();

        // Read sensor ID (10 bytes, ASCII, null-padded)
        byte[] sensorIdBytes = new byte[SENSOR_ID_SIZE];
        fileIn.readFully(sensorIdBytes);
        sensorId = new String(sensorIdBytes).trim();

        // Read expected record count (8 bytes)
        byte[] countBytes = new byte[RECORD_COUNT_SIZE];
        fileIn.readFully(countBytes);
        ByteBuffer cb = ByteBuffer.wrap(countBytes).order(byteOrder);
        expectedRecordCount = cb.getLong();

        // Skip reserved bytes (8 bytes)
        fileIn.readFully(new byte[RESERVED_SIZE]);

        // At this point, fileIn is at position 32 (HEADER_SIZE)
    }

    /**
     * THE CRITICAL METHOD: Read the next binary record.
     * 
     * Called repeatedly by the MapReduce framework.
     * Returns true if a record was successfully read.
     * Returns false when this split is exhausted.
     */
    @Override
    public boolean nextKeyValue() throws IOException {
        
        // ── Boundary check ──
        // We read records as long as the record STARTS before splitEnd.
        // If a record starts before splitEnd but extends past it,
        // we STILL READ IT (to avoid gap between splits).
        // The next split's RecordReader will align past this record.
        if (currentPosition >= splitEnd) {
            return false;
        }

        // ── Attempt to read exactly RECORD_SIZE bytes ──
        int bytesRead = 0;
        int remaining = RECORD_SIZE;
        
        while (remaining > 0) {
            int n = fileIn.read(recordBuffer, bytesRead, remaining);
            if (n < 0) {
                // End of file reached
                if (bytesRead > 0) {
                    // Partial record at end of file — truncated data
                    truncatedRecordsCounter.increment(1);
                }
                return false;
            }
            bytesRead += n;
            remaining -= n;
        }

        // ── Parse the binary record ──
        // Record structure: [timestamp:8][temp:4][humidity:4][pressure:4][status:4]
        byteBuffer.clear();  // Reset buffer position for reading
        
        long timestamp = byteBuffer.getLong();       // bytes 0-7
        float temperature = byteBuffer.getFloat();   // bytes 8-11
        float humidity = byteBuffer.getFloat();       // bytes 12-15
        float pressure = byteBuffer.getFloat();       // bytes 16-19
        int statusCode = byteBuffer.getInt();         // bytes 20-23
        // bytes 24-27: padding to align to 28 bytes (read but unused)

        // ── Populate key-value pair ──
        // Key: file position (useful for debugging, deduplication)
        currentKey.set(currentPosition);
        
        // Value: parsed sensor reading
        currentValue.setSensorId(sensorId);
        currentValue.setTimestamp(timestamp);
        currentValue.setTemperature(temperature);
        currentValue.setHumidity(humidity);
        currentValue.setPressure(pressure);
        currentValue.setStatus(statusCode);

        // ── Advance position ──
        currentPosition += RECORD_SIZE;
        recordsRead++;
        totalRecordsCounter.increment(1);

        return true;
    }

    @Override
    public LongWritable getCurrentKey() {
        return currentKey;
    }

    @Override
    public SensorReading getCurrentValue() {
        return currentValue;
    }

    @Override
    public float getProgress() throws IOException {
        if (splitStart == splitEnd) {
            return 1.0f;
        }
        return Math.min(1.0f, 
            (currentPosition - splitStart) / (float)(splitEnd - splitStart));
    }

    @Override
    public void close() throws IOException {
        if (fileIn != null) {
            fileIn.close();
        }
    }
}

package com.iot.sensor;

import org.apache.hadoop.fs.Path;
import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.mapreduce.InputSplit;
import org.apache.hadoop.mapreduce.JobContext;
import org.apache.hadoop.mapreduce.RecordReader;
import org.apache.hadoop.mapreduce.TaskAttemptContext;
import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;

import java.io.IOException;

public class SensorBinaryInputFormat 
        extends FileInputFormat<LongWritable, SensorReading> {

    /**
     * Factory method: create a RecordReader for one split.
     * Called by each mapper task at initialization.
     */
    @Override
    public RecordReader<LongWritable, SensorReading> createRecordReader(
            InputSplit split, TaskAttemptContext context) 
            throws IOException, InterruptedException {
        
        // Return our custom reader — initialize() will be called by framework
        return new SensorRecordReader();
    }

    /**
     * Determine if a file can be split across multiple mappers.
     * 
     * We return TRUE because:
     * → Our records are fixed-length (28 bytes)
     * → Our RecordReader handles split alignment correctly
     * → Large files (multi-GB) NEED splitting for parallelism
     * 
     * Return FALSE if:
     * → Records are variable-length with no reliable delimiter
     * → File format requires sequential reading from start
     * → Files are compressed with non-splittable codec (Project 14)
     */
    @Override
    protected boolean isSplitable(JobContext context, Path filename) {
        // Check if file is compressed — compressed binary cannot be split
        // (Unless using a splittable compression like LZO with index)
        String name = filename.getName().toLowerCase();
        if (name.endsWith(".gz") || name.endsWith(".snappy") || name.endsWith(".bz2")) {
            return false;
        }
        return true;
    }
}

// FILE 5: SensorAnalysisMapper.java — MAPPER USING CUSTOM INPUT
// ═══════════════════════════════════════════════════════════════
// This mapper receives SensorReading objects from our RecordReader.
// It emits (sensorId_hour, reading) for hourly aggregation.

package com.iot.sensor;

import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Mapper;

import java.io.IOException;

public class SensorAnalysisMapper 
        extends Mapper<LongWritable, SensorReading, Text, Text> {
    
    // Note: Input types are <LongWritable, SensorReading>
    // These come from our custom RecordReader
    // NOT the usual <LongWritable, Text> from TextInputFormat
    
    private Text outputKey = new Text();
    private Text outputValue = new Text();
    
    enum SensorCounters {
        NORMAL_READINGS,
        WARNING_READINGS,
        CRITICAL_READINGS,
        OFFLINE_READINGS,
        TEMPERATURE_OUT_OF_RANGE,
        TOTAL_MAPPED
    }
    
    @Override
    protected void map(LongWritable byteOffset, SensorReading reading, 
                       Context context) 
            throws IOException, InterruptedException {
        
        context.getCounter(SensorCounters.TOTAL_MAPPED).increment(1);
        
        // ── Track status distribution ──
        switch (reading.getStatus()) {
            case SensorReading.STATUS_NORMAL:
                context.getCounter(SensorCounters.NORMAL_READINGS).increment(1);
                break;
            case SensorReading.STATUS_WARNING:
                context.getCounter(SensorCounters.WARNING_READINGS).increment(1);
                break;
            case SensorReading.STATUS_CRITICAL:
                context.getCounter(SensorCounters.CRITICAL_READINGS).increment(1);
                break;
            case SensorReading.STATUS_OFFLINE:
                context.getCounter(SensorCounters.OFFLINE_READINGS).increment(1);
                break;
        }
        
        // ── Validate temperature range ──
        // Physical reality check: temperatures outside -50 to 150°C are errors
        if (reading.getTemperature() < -50.0f || reading.getTemperature() > 150.0f) {
            context.getCounter(SensorCounters.TEMPERATURE_OUT_OF_RANGE).increment(1);
            return;  // Skip this record
        }
        
        // ── Skip offline sensors ──
        if (reading.getStatus() == SensorReading.STATUS_OFFLINE) {
            return;  // No meaningful data from offline sensors
        }
        
        // ── Emit: key = sensorId_hour, value = temp,humidity,pressure,status ──
        // Grouping by sensor + hour enables hourly aggregation in reducer
        String hourKey = reading.getSensorId() + "_" + reading.getHourKey();
        outputKey.set(hourKey);
        
        String valueStr = String.format("%.2f,%.2f,%.2f,%d",
            reading.getTemperature(),
            reading.getHumidity(),
            reading.getPressure(),
            reading.getStatus());
        outputValue.set(valueStr);
        
        context.write(outputKey, outputValue);
    }
}

import java.io.IOException;

public class SensorAnalysisReducer extends Reducer<Text, Text, Text, Text> {
    
    private Text outputValue = new Text();
    
    @Override
    protected void reduce(Text key, Iterable<Text> values, Context context)
            throws IOException, InterruptedException {
        
        // ── Aggregation accumulators ──
        double minTemp = Double.MAX_VALUE;
        double maxTemp = Double.MIN_VALUE;
        double sumTemp = 0;
        double sumHumidity = 0;
        double sumPressure = 0;
        long count = 0;
        long warningCount = 0;
        long criticalCount = 0;
        
        for (Text val : values) {
            String[] parts = val.toString().split(",");
            if (parts.length != 4) continue;
            
            try {
                double temp = Double.parseDouble(parts[0]);
                double humidity = Double.parseDouble(parts[1]);
                double pressure = Double.parseDouble(parts[2]);
                int status = Integer.parseInt(parts[3]);
                
                minTemp = Math.min(minTemp, temp);
                maxTemp = Math.max(maxTemp, temp);
                sumTemp += temp;
                sumHumidity += humidity;
                sumPressure += pressure;
                count++;
                
                if (status == SensorReading.STATUS_WARNING) {
                    warningCount++;
                }
                if (status == SensorReading.STATUS_CRITICAL) {
                    criticalCount++;
                }
            } catch (NumberFormatException e) {
                context.getCounter("SensorReducer", "PARSE_ERRORS").increment(1);
            }
        }
        
        if (count == 0) return;
        
        // ── Build output ──
        // Format: count | min_temp | max_temp | avg_temp | avg_humidity | avg_pressure | warnings | criticals
        String result = String.format(
            "%d\t%.2f\t%.2f\t%.2f\t%.2f\t%.2f\t%d\t%d",
            count,
            minTemp,
            maxTemp,
            sumTemp / count,
            sumHumidity / count,
            sumPressure / count,
            warningCount,
            criticalCount
        );
        
        outputValue.set(result);
        context.write(key, outputValue);
    }
}

package com.iot.sensor;

import java.io.*;
import java.nio.ByteBuffer;
import java.nio.ByteOrder;
import java.util.Random;

public class SensorDataGenerator {

    private static final int HEADER_SIZE = 32;
    private static final int RECORD_SIZE = 28;
    private static final byte[] MAGIC = "SNSR".getBytes();
    private static final short VERSION = 2;

    /**
     * Generate a single binary sensor file.
     *
     * @param outputPath   Local file path to write
     * @param sensorId     Sensor identifier (max 10 chars)
     * @param numRecords   Number of records to generate
     * @param byteOrder    BIG_ENDIAN or LITTLE_ENDIAN
     * @param startTime    Starting timestamp in millis
     * @param intervalMs   Milliseconds between readings
     */
    public static void generateFile(
            String outputPath, String sensorId, long numRecords,
            ByteOrder byteOrder, long startTime, long intervalMs) 
            throws IOException {
        
        Random rand = new Random(sensorId.hashCode());
        
        // ── Pad sensorId to exactly 10 bytes ──
        byte[] sensorIdBytes = new byte[10];
        byte[] idRaw = sensorId.getBytes();
        System.arraycopy(idRaw, 0, sensorIdBytes, 0, 
                         Math.min(idRaw.length, 10));
        
        try (DataOutputStream out = new DataOutputStream(
                new BufferedOutputStream(new FileOutputStream(outputPath)))) {
            
            // ══════════════════════════════════════
            // WRITE FILE HEADER (32 bytes)
            // ══════════════════════════════════════
            
            // Magic bytes (4 bytes)
            out.write(MAGIC);
            
            // Version (2 bytes)
            ByteBuffer versionBuf = ByteBuffer.allocate(2).order(byteOrder);
            versionBuf.putShort(VERSION);
            out.write(versionBuf.array());
            
            // Sensor ID (10 bytes)
            out.write(sensorIdBytes);
            
            // Record count (8 bytes)
            ByteBuffer countBuf = ByteBuffer.allocate(8).order(byteOrder);
            countBuf.putLong(numRecords);
            out.write(countBuf.array());
            
            // Reserved (8 bytes)
            out.write(new byte[8]);
            
            // ══════════════════════════════════════
            // WRITE RECORDS (28 bytes each)
            // ══════════════════════════════════════
            
            ByteBuffer recordBuf = ByteBuffer.allocate(RECORD_SIZE).order(byteOrder);
            
            // Base values for this sensor (simulate realistic ranges)
            float baseTemp = 20.0f + rand.nextFloat() * 10.0f;
            float baseHumidity = 40.0f + rand.nextFloat() * 20.0f;
            float basePressure = 1000.0f + rand.nextFloat() * 30.0f;
            
            for (long i = 0; i < numRecords; i++) {
                recordBuf.clear();
                
                // Timestamp (8 bytes)
                long timestamp = startTime + (i * intervalMs);
                recordBuf.putLong(timestamp);
                
                // Temperature with noise (4 bytes)
                float temp = baseTemp + (rand.nextFloat() - 0.5f) * 5.0f;
                recordBuf.putFloat(temp);
                
                // Humidity with noise (4 bytes)
                float humidity = Math.max(0, Math.min(100, 
                    baseHumidity + (rand.nextFloat() - 0.5f) * 10.0f));
                recordBuf.putFloat(humidity);
                
                // Pressure with noise (4 bytes)
                float pressure = basePressure + (rand.nextFloat() - 0.5f) * 5.0f;
                recordBuf.putFloat(pressure);
                
                // Status (4 bytes)
                // 95% normal, 3% warning, 1.5% critical, 0.5% offline
                int status;
                float statusRoll = rand.nextFloat();
                if (statusRoll < 0.95f) {
                    status = SensorReading.STATUS_NORMAL;
                } else if (statusRoll < 0.98f) {
                    status = SensorReading.STATUS_WARNING;
                } else if (statusRoll < 0.995f) {
                    status = SensorReading.STATUS_CRITICAL;
                } else {
                    status = SensorReading.STATUS_OFFLINE;
                }
                recordBuf.putInt(status);
                
                // Padding to reach 28 bytes (4 bytes unused)
                // Already at position 24 after the fields above
                // putLong(8) + putFloat(4) + putFloat(4) + putFloat(4) + putInt(4) = 24
                recordBuf.putInt(0);  // 4 bytes padding → total 28
                
                out.write(recordBuf.array());
            }
        }
        
        // Calculate file size for verification
        File f = new File(outputPath);
        long expectedSize = HEADER_SIZE + (numRecords * RECORD_SIZE);
        
        System.out.printf("  Generated: %s%n", outputPath);
        System.out.printf("    Sensor ID:    %s%n", sensorId);
        System.out.printf("    Records:      %,d%n", numRecords);
        System.out.printf("    File size:    %,d bytes (%.2f MB)%n", 
                          f.length(), f.length() / (1024.0 * 1024.0));
        System.out.printf("    Expected:     %,d bytes%n", expectedSize);
        System.out.printf("    Size match:   %s%n", 
                          f.length() == expectedSize ? "✅ YES" : "🔴 NO");
    }

            byte[] countBytes = new byte[8];
            in.readFully(countBytes);
            long recordCount = ByteBuffer.wrap(countBytes)
                                         .order(ByteOrder.BIG_ENDIAN).getLong();

            // Skip reserved
            in.readFully(new byte[8]);

            System.out.println("\n  First 5 records:");

            System.out.println("  " + "-".repeat(80));
            
            byte[] recordBuf = new byte[RECORD_SIZE];
            int displayCount = (int) Math.min(5, recordCount);
            
            for (int i = 0; i < displayCount; i++) {
                in.readFully(recordBuf);
                ByteBuffer bb = ByteBuffer.wrap(recordBuf)
                                          .order(ByteOrder.BIG_ENDIAN);
                
                long timestamp = bb.getLong();
                float temp = bb.getFloat();
                float humidity = bb.getFloat();
                float pressure = bb.getFloat();
                int status = bb.getInt();
                
                String statusStr;
                switch (status) {
                    case 0: statusStr = "NORMAL"; break;
                    case 1: statusStr = "WARNING"; break;
                    case 2: statusStr = "CRITICAL"; break;
                    case 3: statusStr = "OFFLINE"; break;
                    default: statusStr = "UNKNOWN(" + status + ")"; break;
                }
                
                java.text.SimpleDateFormat sdf = 
                    new java.text.SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
                sdf.setTimeZone(java.util.TimeZone.getTimeZone("UTC"));
                
                System.out.printf("  %-6d %-24s %10.2f %10.2f %10.2f %10s%n",
                    i, sdf.format(new java.util.Date(timestamp)),
                    temp, humidity, pressure, statusStr);
            }
            
            // ── Verify split alignment math ──
            long fileSize = new File(filePath).length();
            long dataSize = fileSize - HEADER_SIZE;
            long actualRecords = dataSize / RECORD_SIZE;
            long remainder = dataSize % RECORD_SIZE;
            
            System.out.println("\n  ── Split Alignment Verification ──");
            System.out.printf("  File size:          %,d bytes%n", fileSize);
            System.out.printf("  Header size:        %d bytes%n", HEADER_SIZE);
            System.out.printf("  Data region:        %,d bytes%n", dataSize);
            System.out.printf("  Record size:        %d bytes%n", RECORD_SIZE);
            System.out.printf("  Complete records:   %,d%n", actualRecords);
            System.out.printf("  Remainder bytes:    %d%n", remainder);
            System.out.printf("  Header count match: %s%n", 
                actualRecords == recordCount ? "✅ YES" : "🔴 NO");
            
            // Simulate split boundaries
            long blockSize = 128 * 1024 * 1024;  // 128MB
            if (fileSize > blockSize) {
                System.out.println("\n  ── Simulated Split Boundaries ──");
                int splitNum = 0;
                long pos = 0;
                while (pos < fileSize) {
                    long splitEnd = Math.min(pos + blockSize, fileSize);
                    
                    // Calculate alignment for this split
                    long alignedStart;
                    if (pos == 0) {
                        alignedStart = HEADER_SIZE;
                    } else {
                        long bytesIntoData = pos - HEADER_SIZE;
                        long rem = bytesIntoData % RECORD_SIZE;
                        alignedStart = (rem == 0) ? pos : pos + (RECORD_SIZE - rem);
                    }
                    
                    long recordsInSplit = (splitEnd - alignedStart) / RECORD_SIZE;
                    
                    System.out.printf("  Split %d: bytes %,d — %,d | aligned start: %,d | ~%,d records%n",
                        splitNum, pos, splitEnd, alignedStart, recordsInSplit);
                    
                    pos = splitEnd;
                    splitNum++;
                }
            }
        }
    }
