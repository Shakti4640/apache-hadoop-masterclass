PROJECT 15: SHARING A LOOKUP TABLE ACROSS ALL MAPPERS WITHOUT SHIPPING IT EVERY TIME

🔴 HYPOTHETICAL PROBLEM

SCENARIO:
→ You are a data engineer at the ride-sharing company (continuing from Project 14)
→ The trip records contain DRIVER_ID but NOT driver details:
    T001|D042|Manhattan|Brooklyn|8.5|24.50|5.00|2024-01-15 09:30:00
→ The analytics team wants ENRICHED output:
    T001|D042|Carlos Rivera|4.8|Toyota Camry|Manhattan|Brooklyn|24.50
→ Driver details live in a separate small file (5,000 drivers):
    D001|Alice Wu|4.9|Honda Civic
    D002|Tom Ray|4.5|Ford Focus
    D042|Carlos Rivera|4.8|Toyota Camry
    ...

THE NAIVE APPROACHES (ALL FLAWED):

APPROACH 1: "Read the driver file inside map() for each record"
→ Mapper opens HDFS file for EVERY record
→ 500 million trips × file open/read = 500 million HDFS calls
→ NameNode overwhelmed → cluster grinds to halt
→ Each read: ~50ms network latency → 500M × 50ms = 289 DAYS

APPROACH 2: "Include driver data in every trip record"
→ Pre-join before loading to HDFS
→ 5,000 drivers × 100 bytes = 500KB of driver data
→ Duplicated across 500 million records = 250GB of redundancy
→ Storage waste: 250GB × 3 replication = 750GB
→ Shuffle carries driver data unnecessarily

APPROACH 3: "Use a Reduce-Side Join (Project 17 pattern)"
→ Mapper 1: reads trips → emit(DRIVER_ID, trip_data)
→ Mapper 2: reads drivers → emit(DRIVER_ID, driver_data)
→ Reducer: matches trips with driver details
→ WORKS but: shuffles 500 million trip records over network
→ The driver file is TINY (500KB) — overkill to involve reducer

THE RIGHT APPROACH:
→ Ship the small driver file to EVERY mapper node ONCE
→ Each mapper loads it into memory (500KB = trivial)
→ For each trip record: look up driver in memory HashMap
→ NO network calls during processing
→ NO shuffle of driver data
→ NO reducer needed for the join
→ THIS IS THE DISTRIBUTED CACHE

STEP 2: Understand the API

Two ways to add files to distributed cache:

METHOD 1: job.addCacheFile(URI)
  → File available via context.getCacheFiles()
  → Returns URI[] of cached file paths
  → You open and read the file yourself

METHOD 2: job.addCacheFile(URI) with symlink
  → URI with fragment: new URI("/path/drivers.txt#drivers")
  → Creates symlink "drivers" in task working directory
  → Access via: new File("drivers") — simpler path

METHOD 3 (Legacy): DistributedCache.addCacheFile()
  → Deprecated in Hadoop 2.x+
  → Use job.addCacheFile() instead

STEP 3: Understand setup() loading pattern

The critical pattern:
  → Load cache file in setup() — runs ONCE per mapper task
  → Store in instance variable (HashMap) — available to all map() calls
  → DO NOT load in map() — called per record = millions of loads

  │  class MyMapper extends Mapper {                     │

✅ WHAT COULD BE DONE

APPROACH A: Distributed Cache + Map-only enrichment (OUR CHOICE)
→ Small file in cache, loaded into HashMap in setup()
→ Map-only job (zero reducers) — from Project 11
→ Each record enriched independently
→ Fastest possible approach for small lookup tables

APPROACH B: Distributed Cache + Reduce-side processing
→ Same cache loading but with reducers
→ Useful if you need aggregation AFTER enrichment
→ e.g., "Total fare per driver name" requires reduce
→ Cache still avoids shuffling driver data

APPROACH C: Reduce-Side Join (Project 17)
→ Both datasets as mapper input
→ Shuffle brings matching keys together
→ Reducer joins them
→ Works for ANY size datasets
→ Overkill when one dataset is small

APPROACH D: HBase lookup per record
→ Store driver data in HBase (Project 27)
→ Mapper queries HBase for each record
→ Works for very large lookup tables (millions of entries)
→ But: network latency per lookup (~5ms)
→ 500M records × 5ms = 29 days — too slow for batch

WE CHOOSE APPROACH A because:
→ Driver file is small (< 100MB) — fits easily in memory
→ No reducer needed — fastest execution
→ Simplest code — one HashMap, one lookup
→ Demonstrates the most common Hadoop enrichment pattern
→ Bridges directly to Map-Side Join (Project 16)

✅ WHAT SHOULD DO

1. VERIFY LOOKUP FILE SIZE BEFORE USING DISTRIBUTED CACHE
   → Cache file must fit in mapper JVM memory
   → Default mapper heap: 256MB - 1GB
   → HashMap overhead: ~2-3× the raw file size
   → RULE: Cache file should be < 200MB raw
   → 5,000 drivers × 100 bytes = 500KB — well within limits

2. USE SYMLINK FRAGMENT FOR CLEANER FILE ACCESS
   → job.addCacheFile(new URI("/path/file.txt#lookup"))
   → Access in mapper: new BufferedReader(new FileReader("lookup"))
   → Without fragment: must parse full URI path
   → Symlink is simpler and more portable

3. LOAD CACHE FILE IN setup() — NEVER IN map()
   → setup() runs ONCE per mapper task
   → map() runs per record (millions of times)
   → Loading in map() = reading file millions of times
   → From Project 11: setup() for one-time initialization

4. HANDLE MISSING LOOKUPS GRACEFULLY
   → Trip has D999 but driver file has no D999
   → Options:
     a) Skip record (counter: DRIVER_NOT_FOUND)
     b) Emit with "UNKNOWN" placeholder
     c) Emit original record unchanged
   → NEVER throw exception — one missing driver kills entire mapper

5. USE COUNTER TO TRACK CACHE EFFECTIVENESS
   → Count: CACHE_HIT (driver found)
   → Count: CACHE_MISS (driver not found)
   → After job: hit_rate = hits / (hits + misses)
   → Low hit rate → cache file may be stale or incomplete

6. CONSIDER CACHE FILE FRESHNESS
   → Driver file may be updated daily (new drivers, rating changes)
   → Cache is loaded ONCE at job start
   → If driver file changes during job execution → stale data
   → Best practice: Use a snapshot (dated copy) as cache file
   → /data/lookups/drivers_20240115.txt

🚫 WHAT SHOULD NOT DO

1. DO NOT CACHE LARGE FILES (> 200MB)
   → Each mapper loads entire file into memory
   → 100 mappers × 500MB cache = 50GB of cluster RAM just for cache
   → HashMap overhead: 500MB file → ~1.5GB in memory
   → Large lookups: use Reduce-Side Join (Project 17) or HBase (Project 27)

2. DO NOT READ CACHE FILE USING HDFS API IN MAPPER
   → Cache file is on LOCAL disk — use java.io.FileReader
   → NOT: FileSystem.get(conf).open(new Path(...))
   → HDFS read goes over network to DataNode
   → Local read goes to local disk — 100× faster
   → The entire point of distributed cache is LOCAL access

3. DO NOT CACHE FILES THAT CHANGE DURING JOB EXECUTION
   → Cache is snapshot at job submission time
   → If source file changes mid-job, some mappers may see old data
   → Actually: all mappers see the SAME snapshot (job submission time)
   → But if you THINK it's live data, you'll debug the wrong thing
   → ALWAYS use immutable/dated cache files

4. DO NOT ADD CACHE FILES INSIDE MAPPER CODE
   → job.addCacheFile() must be in DRIVER, not mapper
   → Cache files are distributed BEFORE mappers start
   → Cannot dynamically add cache files during execution
   → Plan all needed cache files at job configuration time

5. DO NOT FORGET TO HANDLE FILE ENCODING
   → Cache file read with FileReader uses system default encoding
   → If file is UTF-8 but system default is Latin-1 → garbled data
   → Explicit: new BufferedReader(new InputStreamReader(
       new FileInputStream("lookup"), StandardCharsets.UTF_8))

6. DO NOT USE DISTRIBUTED CACHE FOR BINARY RESOURCES WITHOUT CARE
   → JAR files, native libraries can also be cached
   → job.addCacheArchive() — for .tar.gz, .zip (auto-extracted)
   → job.addFileToClassPath() — for JARs needed at runtime
   → These are different from data cache files

🚫 WHEN NOT TO USE

1. LOOKUP TABLE LARGER THAN MAPPER MEMORY
   → 5GB lookup file → HashMap = ~15GB in memory
   → Mapper heap = 1GB → OutOfMemoryError
   → USE: Reduce-Side Join or HBase instead

2. LOOKUP DATA NEEDS REAL-TIME UPDATES
   → Driver ratings change every hour
   → Cache is static snapshot — doesn't refresh
   → USE: HBase (live reads) or re-run job with updated cache

3. LOOKUP REQUIRES COMPLEX QUERIES
   → "Find all drivers within 5 miles of pickup location"
   → HashMap supports only exact key match
   → USE: Spatial index in memory, or external service

4. ONLY A FEW CONFIGURATION VALUES NEEDED
   → "Tax rate = 8.5%" needed by all mappers
   → Distributed cache is overkill for one value
   → USE: conf.set("tax.rate", "8.5")
   → Read in mapper: context.getConfiguration().get("tax.rate")

5. CACHE FILE IS ON LOCAL FILESYSTEM (NOT HDFS)
   → Distributed cache requires file to be on HDFS
   → Cannot cache from local path directly
   → Must upload to HDFS first: hdfs dfs -put local_file /cache/
   → Then: job.addCacheFile(new URI("hdfs:///cache/local_file"))

═══════════════════════════════════════════════════════════
MECHANIC 1: DISTRIBUTED CACHE FILE LIFECYCLE
═══════════════════════════════════════════════════════════

PHASE 1: JOB SUBMISSION (Client side)

  Driver calls: job.addCacheFile(new URI("/data/drivers.txt#drivers"))
  
  What happens:
  1. URI stored in Job configuration under:
     "mapreduce.job.cache.files" = "/data/drivers.txt#drivers"
  2. Configuration serialized and submitted to ResourceManager
  3. NO file transfer happens yet

PHASE 2: TASK SCHEDULING (ResourceManager + NodeManager)

  When ResourceManager assigns a mapper task to Node X:
  1. NodeManager on Node X receives task description
  2. NodeManager checks: "Does this job need cache files?"
  3. Reads "mapreduce.job.cache.files" from configuration
  4. For each cache file:
     a. Check LOCAL cache directory: is file already present?
        → YES: skip download (shared across tasks on same node)
        → NO: download from HDFS to local directory
     b. Download location: 
        ${yarn.nodemanager.local-dirs}/filecache/<hash>/drivers.txt
     c. Create symlink in task working directory:
        ./drivers → /local/cache/path/drivers.txt

  CRITICAL OPTIMIZATION:
  → If Node X runs 10 mapper tasks for this job,
    file is downloaded ONCE, shared by all 10 tasks
  → If Node X already has this file from a PREVIOUS job
    with same file (same path + same modification time),
    it may reuse the cached copy

  ┌──────────────────────────────────────────────────────┐
  │  Node X:                                              │
  │                                                       │
  │  Local Cache:                                         │
  │  /yarn/local/filecache/abc123/drivers.txt  (500KB)   │
  │           ↑                                           │
  │           │ (one physical copy)                       │
  │           │                                           │
  │  Task 1 working dir: ./drivers → symlink              │
  │  Task 2 working dir: ./drivers → symlink              │
  │  Task 3 working dir: ./drivers → symlink              │
  │  (all point to same local file)                       │
  └──────────────────────────────────────────────────────┘

PHASE 3: TASK EXECUTION (Mapper JVM)

  1. Mapper JVM starts
  2. setup() is called
  3. Code reads symlink: new FileReader("drivers")
     → Resolves to local file → fast local disk read
  4. HashMap populated from file contents
  5. map() called for each input record
     → HashMap.get(key) → O(1) lookup
  6. cleanup() called → HashMap garbage collected

PHASE 4: CLEANUP (After job completes)

  → NodeManager MAY keep cache files for future jobs
  → Governed by: yarn.nodemanager.localizer.cache.target-size-mb
  → Default: 10GB of cache per node
  → LRU eviction when cache exceeds limit
  → Files from frequently-run jobs stay cached longer

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "FileNotFoundException in mapper setup()"
→ CAUSE: Symlink name doesn't match fragment in URI
→ Driver: job.addCacheFile(new URI("/path/drivers.txt#lookup"))
→ Mapper: new FileReader("drivers")  ← WRONG name
→ Should be: new FileReader("lookup")  ← matches fragment
→ FIX: Symlink name in code must EXACTLY match URI fragment

PITFALL 2: "Cache file has stale data"
→ CAUSE: Driver file on HDFS was updated after job started
→ Cache is snapshot at task localization time
→ If file changes between task localizations on different nodes,
   different mappers may see different versions
→ FIX: Use immutable dated files: drivers_20240115.txt
→ FIX: Copy to job-specific path before submitting

PITFALL 3: "OutOfMemoryError loading cache file"
→ CAUSE: Cache file too large for mapper heap
→ 500MB file → ~1.5GB HashMap → exceeds 1GB heap
→ FIX: Increase heap: conf.set("mapreduce.map.java.opts", "-Xmx2g")
→ BETTER FIX: If file > 200MB, use Reduce-Side Join instead

PITFALL 4: "Mapper runs but all lookups return null"
→ CAUSE: Key format mismatch
→ Cache file has: "D042|Carlos Rivera|4.8|Toyota Camry"
→ Trip file has: "D042 " (trailing space)
→ HashMap.get("D042 ") returns null (no match for "D042 ")
→ FIX: Trim keys when loading AND when looking up
→ ALWAYS: key.trim() on both sides

PITFALL 5: "Job works in local mode but fails on cluster"
→ CAUSE: File path differences between local and HDFS
→ Local mode: cache file resolved relative to local filesystem
→ Cluster mode: cache file must be on HDFS
→ FIX: Always upload cache file to HDFS before job submission
→ FIX: Use fully qualified HDFS URI: hdfs://namenode:9000/path/file

PITFALL 6: "Cache file works for first few mappers then fails"
→ CAUSE: Node local disk full — cannot cache file
→ NodeManager cache directory has limited space
→ yarn.nodemanager.local-dirs disk full → localization fails
→ FIX: Monitor disk space on worker nodes
→ FIX: Clean old cache: yarn.nodemanager.localizer.cache.cleanup.interval-ms

NETWORK IMPACT:
  Cache file distributed to N nodes (not N mappers)
  50-node cluster, 500KB cache file:
    → 500KB × 50 nodes = 25MB total network transfer
    → Happens ONCE before job starts
    → Negligible compared to any MapReduce job

  50-node cluster, 200MB cache file:
    → 200MB × 50 nodes = 10GB total network transfer
    → Happens before job starts → delays job launch by ~80 seconds
    → Still worthwhile if alternative is shuffling 200MB per mapper

CLUSTER MEMORY IMPACT:
  100 mappers × 1MB HashMap = 100MB total cluster memory for cache
  100 mappers × 100MB HashMap = 10GB total cluster memory for cache
  → At 100MB+, cache competes with actual processing memory
  → Monitor: if mapper GC time increases after adding cache → too big

MULTIPLE CACHE FILES:
  → Can add MULTIPLE files: drivers.txt + vehicles.txt + zones.txt

import org.apache.hadoop.conf.Configured;
import org.apache.hadoop.fs.Path;

public class TripEnrichmentJob extends Configured implements Tool {

    // ════════════════════════════════════════════════════════
    // MAPPER: Load driver lookup from cache, enrich each trip
    // ════════════════════════════════════════════════════════
    //
    // INPUT (trip records, pipe-delimited):
    //   TRIP_ID|DRIVER_ID|PICKUP_ZONE|DROPOFF_ZONE|DISTANCE|FARE|TIP|TIMESTAMP
    //   T001|D042|Manhattan|Brooklyn|8.5|24.50|5.00|2024-01-15 09:30:00
    //
    // CACHE FILE (driver lookup, pipe-delimited):
    //   DRIVER_ID|NAME|RATING|VEHICLE
    //   D042|Carlos Rivera|4.8|Toyota Camry
    //
    // OUTPUT (enriched, pipe-delimited):
    //   TRIP_ID|DRIVER_ID|DRIVER_NAME|RATING|VEHICLE|PICKUP|DROPOFF|DISTANCE|FARE|TIP|TIMESTAMP
    //
    // KEY: NullWritable (map-only, no key needed — Project 11)

    public static class EnrichmentMapper
            extends Mapper<LongWritable, Text, NullWritable, Text> {

        // ── Lookup table: loaded ONCE in setup() ──
        // Key: driver ID (e.g., "D042")
        // Value: driver details (e.g., "Carlos Rivera|4.8|Toyota Camry")
        private Map<String, String> driverLookup;
        private Text enrichedRecord = new Text();
        private long setupTimeMs;

        // ── Counters for monitoring ──
        enum CacheCounters {
            CACHE_LOAD_TIME_MS,
            CACHE_SIZE_ENTRIES,
            TOTAL_RECORDS,
            CACHE_HIT,
            CACHE_MISS,
            INVALID_RECORDS,
            HEADER_SKIPPED,
            ENRICHED_RECORDS
        }

        @Override
        protected void setup(Context context) throws IOException {
            // ══════════════════════════════════════════════
            // CRITICAL: Load cache file ONCE per mapper
            // ══════════════════════════════════════════════
            // This runs before any map() call
            // File is on LOCAL disk (distributed cache put it there)
            // Reading local file: ~1ms per MB (fast)

            long startTime = System.currentTimeMillis();
            driverLookup = new HashMap<>();

            // ── Read the symlinked cache file ──
            // URI fragment "drivers" creates local symlink named "drivers"
            // File is LOCAL — do NOT use HDFS FileSystem API
            try (BufferedReader reader = new BufferedReader(new FileReader("drivers"))) {
                String line;
                int lineCount = 0;

                while ((line = reader.readLine()) != null) {
                    lineCount++;
                    line = line.trim();

                    // Skip empty lines and headers
                    if (line.isEmpty() || line.startsWith("DRIVER_ID")) {
                        continue;
                    }

                    // Parse: DRIVER_ID|NAME|RATING|VEHICLE
                    String[] parts = line.split("\\|");
                    if (parts.length >= 4) {
                        String driverId = parts[0].trim();
                        // Store remaining fields as concatenated value
                        String driverInfo = parts[1].trim() + "|"
                                          + parts[2].trim() + "|"
                                          + parts[3].trim();
                        driverLookup.put(driverId, driverInfo);
                    }
                }
            }

            setupTimeMs = System.currentTimeMillis() - startTime;

            // ── Report cache loading metrics via counters ──
            context.getCounter(CacheCounters.CACHE_LOAD_TIME_MS)
                   .increment(setupTimeMs);
            context.getCounter(CacheCounters.CACHE_SIZE_ENTRIES)
                   .increment(driverLookup.size());

            // Log for debugging (visible in task attempt logs)
            System.err.println("Distributed Cache loaded: "
                + driverLookup.size() + " drivers in "
                + setupTimeMs + "ms");
        }


        @Override
        protected void map(LongWritable offset, Text line, Context context)
                throws IOException, InterruptedException {

            context.getCounter(CacheCounters.TOTAL_RECORDS).increment(1);

            String record = line.toString().trim();
            if (record.isEmpty()) return;

            // Skip header
            if (record.startsWith("TRIP_ID")) {
                context.getCounter(CacheCounters.HEADER_SKIPPED).increment(1);
                return;
            }

            // Parse trip record
            // TRIP_ID|DRIVER_ID|PICKUP_ZONE|DROPOFF_ZONE|DISTANCE|FARE|TIP|TIMESTAMP
            String[] fields = record.split("\\|");
            if (fields.length != 8) {
                context.getCounter(CacheCounters.INVALID_RECORDS).increment(1);
                return;
            }

            String tripId     = fields[0].trim();
            String driverId   = fields[1].trim();
            String pickupZone = fields[2].trim();
            String dropoffZone= fields[3].trim();
            String distance   = fields[4].trim();
            String fare       = fields[5].trim();
            String tip        = fields[6].trim();
            String timestamp  = fields[7].trim();

            // ══════════════════════════════════════════════
            // CORE OPERATION: HashMap lookup — O(1)
            // ══════════════════════════════════════════════
            // This is why distributed cache exists:
            // → No HDFS read (local memory lookup)
            // → No network call
            // → ~50 nanoseconds per lookup
            // → vs ~50 milliseconds for HDFS read (1,000,000× faster)

            String driverInfo = driverLookup.get(driverId);

            String driverName;
            String rating;
            String vehicle;

            if (driverInfo != null) {
                // ── CACHE HIT: Driver found ──
                context.getCounter(CacheCounters.CACHE_HIT).increment(1);
                String[] driverParts = driverInfo.split("\\|");
                driverName = driverParts[0];
                rating = driverParts[1];
                vehicle = driverParts[2];
            } else {
                // ── CACHE MISS: Driver not in lookup ──
                context.getCounter(CacheCounters.CACHE_MISS).increment(1);
                driverName = "UNKNOWN";
                rating = "0.0";
                vehicle = "UNKNOWN";
            }

            // ── Build enriched output record ──
            String enriched = String.join("|",
                tripId, driverId, driverName, rating, vehicle,
                pickupZone, dropoffZone, distance, fare, tip, timestamp
            );

            enrichedRecord.set(enriched);

            // NullWritable key — clean output with no key prefix (Project 11)
            context.write(NullWritable.get(), enrichedRecord);
            context.getCounter(CacheCounters.ENRICHED_RECORDS).increment(1);
        }


        @Override
        protected void cleanup(Context context) throws IOException {
            // ── Release HashMap memory ──
            // Not strictly necessary (JVM exits after task)
            // But good practice for JVM reuse scenarios
            if (driverLookup != null) {
                System.err.println("Cache stats: "
                    + driverLookup.size() + " entries, loaded in "
                    + setupTimeMs + "ms");
                driverLookup.clear();
                driverLookup = null;
            }
        }
    }


    // ════════════════════════════════════════════════════════
    // DRIVER: Configure distributed cache and launch job
    // ════════════════════════════════════════════════════════

    @Override
    public int run(String[] args) throws Exception {

        if (args.length != 3) {
            System.err.println("Usage: TripEnrichmentJob <trip_input> <output> <driver_cache_file>");
            System.err.println();
            System.err.println("  trip_input:        HDFS path to trip records");
            System.err.println("  output:            HDFS path for enriched output");
            System.err.println("  driver_cache_file: HDFS path to driver lookup file");
            System.err.println();
            System.err.println("Example:");
            System.err.println("  TripEnrichmentJob /data/trips /output/enriched /data/lookups/drivers.txt");
            return 1;
        }

        String tripInput = args[0];
        String output = args[1];
        String driverCachePath = args[2];

        Configuration conf = getConf();
        Job job = Job.getInstance(conf, "Trip Enrichment via Distributed Cache");
        job.setJarByClass(TripEnrichmentJob.class);

        // ── Mapper (no reducer — map-only job) ──
        job.setMapperClass(EnrichmentMapper.class);

        // ── Map-only: zero reducers (Project 11 pattern) ──
        job.setNumReduceTasks(0);

        // ── Output types ──
        job.setOutputKeyClass(NullWritable.class);
        job.setOutputValueClass(Text.class);

        // ══════════════════════════════════════════════════════
        // THE KEY LINE: ADD FILE TO DISTRIBUTED CACHE
        // ══════════════════════════════════════════════════════
        //
        // URI fragment "#drivers" creates a symlink named "drivers"
        // in each mapper's working directory.
        //
        // Mapper accesses it via: new FileReader("drivers")
        //
        // Hadoop distributes this file to EVERY node that runs
        // a mapper task for this job — ONCE per node, before
        // any mapper starts.

        job.addCacheFile(new URI(driverCachePath + "#drivers"));

        System.out.println("═══ DISTRIBUTED CACHE CONFIGURATION ═══");
        System.out.println("  Cache file: " + driverCachePath);
        System.out.println("  Symlink:    drivers");
        System.out.println("  Access:     new FileReader(\"drivers\")");
        System.out.println("════════════════════════════════════════");

        // ── Input/Output paths ──
        FileInputFormat.addInputPath(job, new Path(tripInput));
        FileOutputFormat.setOutputPath(job, new Path(output));

        // ── Run ──
        boolean success = job.waitForCompletion(true);

        if (success) {
            printCacheReport(job);
        }

        return success ? 0 : 1;
    }


    private void printCacheReport(Job job) throws IOException {
        System.out.println("\n╔══════════════════════════════════════════════════╗");
        System.out.println("║        DISTRIBUTED CACHE ENRICHMENT REPORT       ║");
        System.out.println("╠══════════════════════════════════════════════════╣");

        long total = job.getCounters()
            .findCounter(EnrichmentMapper.CacheCounters.TOTAL_RECORDS).getValue();
        long hits = job.getCounters()
            .findCounter(EnrichmentMapper.CacheCounters.CACHE_HIT).getValue();
        long misses = job.getCounters()
            .findCounter(EnrichmentMapper.CacheCounters.CACHE_MISS).getValue();
        long enriched = job.getCounters()
            .findCounter(EnrichmentMapper.CacheCounters.ENRICHED_RECORDS).getValue();
        long cacheSize = job.getCounters()
            .findCounter(EnrichmentMapper.CacheCounters.CACHE_SIZE_ENTRIES).getValue();
        long loadTime = job.getCounters()
            .findCounter(EnrichmentMapper.CacheCounters.CACHE_LOAD_TIME_MS).getValue();
        long invalid = job.getCounters()
            .findCounter(EnrichmentMapper.CacheCounters.INVALID_RECORDS).getValue();

        double hitRate = (hits + misses) > 0 ? hits * 100.0 / (hits + misses) : 0;

        System.out.printf("║  Cache entries loaded:    %,d%n", cacheSize);
        System.out.printf("║  Cache load time (total): %,d ms%n", loadTime);
        System.out.println("║──────────────────────────────────────────────────║");
        System.out.printf("║  Total input records:     %,d%n", total);
        System.out.printf("║  Invalid records:         %,d%n", invalid);
        System.out.printf("║  Enriched records:        %,d%n", enriched);
        System.out.println("║──────────────────────────────────────────────────║");
        System.out.printf("║  Cache HITS:              %,d%n", hits);
        System.out.printf("║  Cache MISSES:            %,d%n", misses);
        System.out.printf("║  Hit rate:                %.1f%%%n", hitRate);
        System.out.println("║──────────────────────────────────────────────────║");

        if (hitRate < 90) {
            System.out.println("║  ⚠️  LOW HIT RATE — Check cache file completeness");
        } else {
            System.out.println("║  ✅ Cache hit rate is healthy");
        }

        System.out.println("╚══════════════════════════════════════════════════╝\n");
    }


    public static void main(String[] args) throws Exception {
        int exitCode = ToolRunner.run(new Configuration(),
                                      new TripEnrichmentJob(), args);
        System.exit(exitCode);
    }

def analyze_cache_feasibility(lookup_filepath, mapper_heap_mb=1024):
    """
    Determine if a lookup file is suitable for distributed cache.
    Estimates memory usage and provides recommendation.
    """
    
    print("═" * 70)
    print("DISTRIBUTED CACHE FEASIBILITY ANALYSIS")
    print("═" * 70)
    
    if not os.path.exists(lookup_filepath):
        print(f"File not found: {lookup_filepath}")
        return
    
    file_size_bytes = os.path.getsize(lookup_filepath)
    file_size_mb = file_size_bytes / (1024 * 1024)
    
    # Count lines and measure key/value sizes
    line_count = 0
    total_key_bytes = 0
    total_value_bytes = 0
    
    with open(lookup_filepath, 'r') as f:
        for line in f:
            line = line.strip()
            if not line or line.startswith("DRIVER_ID"):
                continue
            
            parts = line.split("|", 1)
            if len(parts) >= 2:
                key = parts[0].strip()
                value = parts[1].strip()
                total_key_bytes += len(key.encode('utf-8'))
                total_value_bytes += len(value.encode('utf-8'))
                line_count += 1
    
    if line_count == 0:
        print("No valid entries found in file.")
        return
    
    avg_key_bytes = total_key_bytes / line_count
    avg_value_bytes = total_value_bytes / line_count
    
    # ── Estimate HashMap memory ──
    # Java HashMap memory model:
    # Entry: 32 bytes (object header + hash + key ref + value ref + next ref)
    # String: 40 bytes (object header + char[] ref + hash) + char data
    # Backing array: entries × 1.33 (load factor 0.75) × 8 bytes per pointer
    
    entry_overhead = 32
    string_overhead = 40  # Object header + metadata
    key_memory = string_overhead + avg_key_bytes * 2  # Java chars are 2 bytes
    value_memory = string_overhead + avg_value_bytes * 2
    per_entry_memory = entry_overhead + key_memory + value_memory
    
    # Backing array (load factor 0.75 → array size = entries / 0.75)
    array_size = int(line_count / 0.75) + 1
    # Round up to power of 2 (HashMap requirement)
    array_size_pow2 = 1
    while array_size_pow2 < array_size:
        array_size_pow2 *= 2
    array_memory = array_size_pow2 * 8  # 8 bytes per reference
    
    total_hashmap_memory = (per_entry_memory * line_count) + array_memory
    total_hashmap_mb = total_hashmap_memory / (1024 * 1024)
    
    # Memory overhead ratio
    overhead_ratio = total_hashmap_mb / max(file_size_mb, 0.001)
    
    # Available memory after JVM overhead
    jvm_overhead_mb = 100  # GC, class metadata, etc.
    processing_memory_mb = 200  # Buffer for actual MapReduce processing
    available_for_cache = mapper_heap_mb - jvm_overhead_mb - processing_memory_mb
    
    print(f"\n── FILE ANALYSIS ──")
    print(f"  File size:           {file_size_mb:.2f} MB")
    print(f"  Entries:             {line_count:,}")
    print(f"  Avg key size:        {avg_key_bytes:.0f} bytes")
    print(f"  Avg value size:      {avg_value_bytes:.0f} bytes")
    
    print(f"\n── MEMORY ESTIMATION ──")
    print(f"  Per-entry memory:    {per_entry_memory:.0f} bytes")
    print(f"  HashMap total:       {total_hashmap_mb:.1f} MB")
    print(f"  Overhead ratio:      {overhead_ratio:.1f}× file size")
    print(f"  (HashMap is ~{overhead_ratio:.0f}× larger than raw file)")
    
    print(f"\n── MAPPER MEMORY BUDGET ──")
    print(f"  Mapper heap:         {mapper_heap_mb} MB")
    print(f"  JVM overhead:        ~{jvm_overhead_mb} MB")
    print(f"  Processing reserve:  ~{processing_memory_mb} MB")
    print(f"  Available for cache: ~{available_for_cache} MB")
    print(f"  Cache needs:         {total_hashmap_mb:.1f} MB")
    print(f"  Remaining:           {available_for_cache - total_hashmap_mb:.1f} MB")
    
    print(f"\n── RECOMMENDATION ──")
    
    cache_pct = total_hashmap_mb * 100 / available_for_cache
    
    if total_hashmap_mb < available_for_cache * 0.3:
        print(f"  ✅ DISTRIBUTED CACHE: IDEAL")
        print(f"     Cache uses {cache_pct:.0f}% of available memory")
        print(f"     Plenty of room for processing")
    elif total_hashmap_mb < available_for_cache * 0.6:

        print(f"     Cache uses {cache_pct:.0f}% of available memory")
        print(f"     Consider increasing mapper heap if needed:")
        recommended_heap = int(total_hashmap_mb / 0.3) + jvm_overhead_mb + processing_memory_mb

    elif total_hashmap_mb < available_for_cache:

        print(f"     Cache uses {cache_pct:.0f}% of available memory")
        print(f"     Increase mapper heap:")

✅ DISTRIBUTED CACHE MECHANISM:
   → Ships small files from HDFS to LOCAL disk of every worker node
   → Happens ONCE per node before any mapper starts

   → Cache file < 200MB raw → HashMap < 600MB → safe
