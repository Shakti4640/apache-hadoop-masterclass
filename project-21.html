PROJECT 21: CHAINING THREE MAPREDUCE JOBS INTO A DATA PIPELINE

🔴 HYPOTHETICAL PROBLEM

SCENARIO:
→ You are a data platform engineer at a telecom company
→ 300 million Call Detail Records (CDRs) arrive daily in HDFS
→ Business intelligence team needs a DAILY REPORT containing:

  STEP 1: CLEAN raw CDRs (remove malformed, normalize phone formats)
  STEP 2: ENRICH cleaned CDRs with customer profile data (join)
  STEP 3: AGGREGATE enriched data into per-customer daily summaries
          AND write suspicious patterns to a separate flagged directory

CURRENT STATE (BAD):
→ An operations engineer manually runs three commands every morning:

  6:00 AM → hadoop jar pipeline.jar CleanJob /raw/2024-06-15 /tmp/cleaned
  (waits 45 minutes, checks if it succeeded)
  
  6:50 AM → hadoop jar pipeline.jar EnrichJob /tmp/cleaned /tmp/enriched
  (waits 30 minutes, checks if it succeeded)
  
  7:25 AM → hadoop jar pipeline.jar AggregateJob /tmp/enriched /output/report
  (waits 25 minutes)
  
  7:50 AM → hdfs dfs -rm -r /tmp/cleaned /tmp/enriched
  (cleanup intermediate data)

PROBLEMS WITH MANUAL APPROACH:
→ Human must babysit three jobs for 2 hours every morning
→ If CleanJob fails at 6:30 AM, EnrichJob runs on stale data
→ If engineer forgets to clean /tmp, disk fills up in 3 days
→ If EnrichJob fails, AggregateJob still runs → garbage output
→ Weekend/holiday: nobody runs the pipeline → reports missing
→ No dependency tracking: jobs don't know about each other

WHAT WE NEED:
→ Define: Job B depends on Job A, Job C depends on Job B
→ Automatic: If A fails → don't start B or C
→ Automatic: Clean up intermediate data after success
→ Automatic: Run without human intervention
→ Bonus: If Job B and Job C were independent, run them IN PARALLEL

THIS IS JOB CHAINING — the foundation of data pipeline engineering

✅ WHAT SHOULD DO

1. ALWAYS USE UNIQUE INTERMEDIATE PATHS PER PIPELINE RUN
   → BAD:  /tmp/cleaned/ (shared across runs — race condition)
   → GOOD: /tmp/pipeline/2024-06-15-run001/cleaned/
   → Include date, run ID, or timestamp in path
   → Prevents two simultaneous pipeline runs from colliding

2. CHECK JOB SUCCESS BEFORE PROCEEDING
   → job.waitForCompletion(true) returns boolean
   → true = success, false = failure
   → ALWAYS check return value before starting next job
   → With JobControl: check getFailedJobList() after completion

3. CLEAN UP INTERMEDIATE DATA AFTER SUCCESS
   → Intermediate HDFS paths consume 3x storage (replication)
   → 10GB intermediate × 3 replicas × 2 stages = 60GB wasted
   → Delete /tmp/pipeline/ ONLY after final job succeeds
   → Keep intermediate data on FAILURE for debugging

4. LOG PIPELINE PROGRESS CLEARLY
   → Print: "=== STAGE 1/3: CleanJob STARTING ==="
   → Print: "=== STAGE 1/3: CleanJob COMPLETED (45 min) ==="
   → Print: "=== STAGE 2/3: EnrichJob STARTING ==="
   → Operators need to know where the pipeline is at a glance

5. PRESERVE COUNTERS FROM ALL JOBS
   → Each job has its own Counters (Project 7)
   → Collect counters from ALL jobs for end-of-pipeline summary
   → "Records in: 300M → Cleaned: 285M → Enriched: 280M → Reported: 280M"
   → Drop-off at each stage reveals data quality issues

6. MAKE INTERMEDIATE PATHS CONFIGURABLE
   → Don't hardcode /tmp/pipeline/
   → Pass base path as argument or configuration property
   → Different environments (dev/staging/prod) use different paths

🚫 WHAT SHOULD NOT DO

1. DO NOT start Job 2 without verifying Job 1 succeeded
   → job1.waitForCompletion(true) can return false
   → If you ignore the return value, Job 2 reads empty/partial input
   → Output is silently wrong — no error, just garbage

2. DO NOT share the same Configuration object across jobs
   → Job.getInstance(conf) creates a COPY of conf
   → BUT: if you modify conf between job creations,
     earlier jobs may pick up later modifications
   → SAFEST: Create fresh Configuration per job
   → OR: Create all jobs FIRST, then modify nothing

3. DO NOT delete intermediate data before confirming final job success
   → Pipeline: Job1 → Job2 → Job3
   → If you delete Job1 output after Job2 starts
     AND Job2 fails midway
   → You must rerun from Job1 (expensive!)
   → RULE: Delete ALL intermediate data ONLY after Job3 succeeds

4. DO NOT ignore the JobControl thread lifecycle
   → JobControl runs in a SEPARATE THREAD
   → You must call jobControl.run() in a Thread
   → Then poll jobControl.allFinished() in your main thread
   → Forgetting to start the thread → pipeline hangs forever

5. DO NOT create circular dependencies
   → job1.addDependingJob(job2) + job2.addDependingJob(job1)
   → DEADLOCK — neither job can start
   → JobControl does NOT detect cycles
   → You must ensure DAG is acyclic manually

6. DO NOT use job chaining for more than 5-7 jobs
   → Beyond 5 jobs: use Apache Oozie (Project 31) or Airflow
   → JobControl lacks: retries, alerts, scheduling, UI
   → It is a PROGRAMMATIC tool, not a WORKFLOW MANAGER
   → Fine for development and simple pipelines
   → Insufficient for production SLA-driven workflows

🚫 WHEN NOT TO USE

1. SINGLE JOB SUFFICES
   → If your logic fits in one MapReduce job
   → Adding a second job "for cleanliness" wastes I/O
   → Each job boundary = full write to HDFS + full read from HDFS
   → 10GB intermediate × 2 boundaries = 60GB extra I/O (with replication)

2. HIGHER-LEVEL TOOLS HANDLE IT
   → Hive query with multiple stages → Hive compiles to chained MR internally
   → Pig script with multiple operations → Pig optimizes the DAG
   → Spark → keeps intermediate data in MEMORY (no HDFS write between stages)
   → If using these tools: don't manually chain MapReduce jobs

3. REAL-TIME / LOW-LATENCY REQUIREMENTS
   → Each MapReduce job has ~30 second startup overhead minimum
   → 3-job chain: ~90 seconds just in startup overhead
   → Plus actual processing time
   → For sub-minute pipelines: use Spark Streaming or Flink

4. MORE THAN 7-8 JOBS IN THE CHAIN
   → JobControl becomes hard to manage
   → No built-in retry, alerting, or scheduling
   → Switch to Oozie (Project 31) or Airflow
   → These provide UI, retries, SLA tracking, email alerts

5. CROSS-CLUSTER PIPELINES
   → JobControl works within ONE Hadoop cluster
   → If pipeline spans multiple clusters or cloud services
   → Use Airflow or cloud-native orchestrators (Step Functions, etc.)

⚙️ HOW IT ONLY WORKS — INTERNAL MECHANICS

═══════════════════════════════════════════════════════════════
INTERNAL MECHANIC 1: HOW SEQUENTIAL CHAINING WORKS
═══════════════════════════════════════════════════════════════

  When you call:
    boolean success = job.waitForCompletion(true);

  INTERNALLY:
  
  Step 1 — job.submit()
    → Serializes Configuration to XML
    → Uploads JAR and config to HDFS staging directory
    → Contacts ResourceManager to create ApplicationMaster
    → Returns immediately (asynchronous submission)

  Step 2 — job.waitForCompletion() blocks
    → Polls ResourceManager every 5 seconds for status
    → Parameter 'true' enables progress printing to stdout
    → Thread is BLOCKED until job completes or fails

  Step 3 — Return value
    → true  = JobStatus.SUCCEEDED
    → false = JobStatus.FAILED or JobStatus.KILLED

  SEQUENTIAL CHAIN EXECUTION:

  Thread: ──[submit job1]──[BLOCKED waiting]──[job1 done]──
            ──[submit job2]──[BLOCKED waiting]──[job2 done]──
            ──[submit job3]──[BLOCKED waiting]──[job3 done]──

  TOTAL TIME = job1_time + job2_time + job3_time
  NO PARALLELISM POSSIBLE

═══════════════════════════════════════════════════════════════
INTERNAL MECHANIC 2: HOW JobControl + ControlledJob WORKS
═══════════════════════════════════════════════════════════════

  ControlledJob: Wrapper around a Job with dependency list
  JobControl: Execution engine that manages multiple ControlledJobs

  SETUP PHASE:

    ControlledJob cJob1 = new ControlledJob(job1.getConfiguration());
    cJob1.setJob(job1);

    ControlledJob cJob2 = new ControlledJob(job2.getConfiguration());
    cJob2.setJob(job2);
    cJob2.addDependingJob(cJob1);  // job2 depends on job1

    ControlledJob cJob3 = new ControlledJob(job3.getConfiguration());
    cJob3.setJob(job3);
    cJob3.addDependingJob(cJob2);  // job3 depends on job2

    JobControl control = new JobControl("CDR Pipeline");
    control.addJob(cJob1);
    control.addJob(cJob2);
    control.addJob(cJob3);

  INTERNAL STATE MACHINE PER JOB:

    ┌─────────┐  deps met   ┌─────────┐  submitted  ┌─────────┐
    │ WAITING │────────────→│  READY  │────────────→│ RUNNING │
    └─────────┘             └─────────┘             └────┬────┘
                                                         │
                                          ┌──────────────┼──────────────┐
                                          ▼              │              ▼
                                    ┌──────────┐         │       ┌──────────┐
                                    │ SUCCESS  │         │       │ FAILED   │
                                    └──────────┘         │       └──────────┘
                                                         ▼
                                                  ┌──────────────┐
                                                  │ DEPENDENT_   │
                                                  │ FAILED       │
                                                  └──────────────┘

  EXECUTION LOOP (runs in separate thread):

    Thread controlThread = new Thread(control);
    controlThread.start();

    JobControl.run() internally does:
    ┌──────────────────────────────────────────────────────┐
    │  while (!allFinished()) {                            │
    │                                                      │
    │    // Check WAITING jobs                             │
    │    for each WAITING job:                             │
    │      if all dependencies are SUCCESS:                │
    │        move to READY                                 │
    │      if any dependency is FAILED:                    │
    │        move to DEPENDENT_FAILED (skip this job)      │
    │                                                      │
    │    // Submit READY jobs                              │
    │    for each READY job:                               │
    │      job.submit()  // asynchronous                   │
    │      move to RUNNING                                 │
    │                                                      │
    │    // Check RUNNING jobs                             │
    │    for each RUNNING job:                             │
    │      if job.isComplete():                            │
    │        if job.isSuccessful(): move to SUCCESS        │
    │        else: move to FAILED                          │
    │                                                      │
    │    Thread.sleep(5000)  // poll every 5 seconds       │
    │  }                                                   │
    └──────────────────────────────────────────────────────┘

  PARALLELISM EXAMPLE:

    Given DAG:
      Job1 ──→ Job3
      Job2 ──→ Job3  (Job3 depends on BOTH Job1 and Job2)

    Execution:
      Time 0:   Job1=READY, Job2=READY, Job3=WAITING
      Time 0+:  Job1=RUNNING, Job2=RUNNING, Job3=WAITING (parallel!)
      Time 15m: Job1=SUCCESS, Job2=RUNNING, Job3=WAITING
      Time 20m: Job1=SUCCESS, Job2=SUCCESS, Job3=READY (both deps met)
      Time 20m+:Job3=RUNNING
      Time 35m: Job3=SUCCESS → allFinished()

    Total time: 35 minutes (not 15+20+15 = 50 minutes sequential)

═══════════════════════════════════════════════════════════════
INTERNAL MECHANIC 3: INTERMEDIATE DATA ON HDFS
═══════════════════════════════════════════════════════════════

  Between jobs, ALL data goes through HDFS:

    Job 1 Reducers
        │
        ▼
    ┌────────────────────────────────────────┐
    │  HDFS: /tmp/pipeline/cleaned/          │
    │  → part-r-00000 (Block A, B, C)       │
    │  → part-r-00001 (Block D, E, F)       │
    │  → Each block replicated 3 times       │
    │  → Written to DataNode local disks     │
    └──────────────────┬─────────────────────┘
                       │
                       ▼  (full HDFS read)
    Job 2 Mappers read from DataNodes

  COST OF EACH JOB BOUNDARY:
  → Job 1 writes 10GB → HDFS stores 30GB (3x replication)
  → Job 2 reads 10GB back from HDFS
  → Total I/O per boundary: 10GB write + 10GB read = 20GB
  → With replication overhead: 40GB actual disk I/O

  THIS IS WHY SPARK IS FASTER:
  → Spark keeps intermediate data in MEMORY between stages
  → No HDFS write/read between stages
  → Same 3-stage pipeline: Spark may be 5-10x faster
  → (Covered in Project 48: Spark Migration)

  BUT HADOOP CHAINING IS MORE RELIABLE:
  → Intermediate data on HDFS is DURABLE (replicated)
  → If Job 2 fails, intermediate data survives
  → Restart Job 2 without rerunning Job 1
  → Spark in-memory data is LOST on failure → must recompute

═══════════════════════════════════════════════════════════════
INTERNAL MECHANIC 4: HOW OUTPUT PATH BECOMES INPUT PATH
═══════════════════════════════════════════════════════════════

  Job 1 output: /tmp/pipeline/cleaned/
  Contains:
    /tmp/pipeline/cleaned/part-r-00000
    /tmp/pipeline/cleaned/part-r-00001
    /tmp/pipeline/cleaned/_SUCCESS    ← marker file

  Job 2 input: /tmp/pipeline/cleaned/
  
  When FileInputFormat.addInputPath(job2, new Path("/tmp/pipeline/cleaned/")):
  → Scans directory for files
  → SKIPS files starting with _ or . (like _SUCCESS, .crc)
  → Reads part-r-00000, part-r-00001 as input splits
  → Each split assigned to a mapper

  THE _SUCCESS FILE:
  → Created by OutputCommitter ONLY when job succeeds
  → Absence means: job failed or is still running
  → You can check for _SUCCESS before starting next job:
    
    FileSystem fs = FileSystem.get(conf);
    Path successFlag = new Path("/tmp/pipeline/cleaned/_SUCCESS");
    if (!fs.exists(successFlag)) {
        throw new RuntimeException("Previous job did not complete!");
    }

⚠️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "Job 2 starts but finds zero input records"
→ CAUSE: Job 1 output path differs from Job 2 input path
→ Job 1 writes to: /tmp/pipeline/cleaned
→ Job 2 reads from: /tmp/pipeline/clean  (typo — missing 'd')
→ No error — just zero mappers, zero output
→ FIX: Define paths as constants, reuse across jobs
→ PATTERN:
    static final Path CLEANED_PATH = new Path("/tmp/pipeline/cleaned");
    // Job 1: FileOutputFormat.setOutputPath(job1, CLEANED_PATH);
    // Job 2: FileInputFormat.addInputPath(job2, CLEANED_PATH);

PITFALL 2: "Output path already exists" error on rerun
→ CAUSE: Previous run created /tmp/pipeline/cleaned/ 
→ MapReduce refuses to overwrite existing output (safety)
→ FIX: Delete output paths before starting pipeline
→ PATTERN:
    FileSystem fs = FileSystem.get(conf);
    fs.delete(CLEANED_PATH, true);  // recursive delete

PITFALL 3: "JobControl thread never finishes"
→ CAUSE: Forgot to call control.stop() or circular dependency
→ JobControl.run() loops forever if allFinished() never returns true
→ FIX: Add timeout in monitoring loop
→ FIX: Verify no circular dependencies

PITFALL 4: "Job 3 reads partial output from Job 2"
→ CAUSE: Started Job 3 before Job 2 fully committed output
→ MapReduce output uses two-phase commit:
   1. Tasks write to temporary _temporary/ directory
   2. On job success, OutputCommitter moves files to final path
→ If you check for files BEFORE _SUCCESS marker → partial data
→ FIX: Use waitForCompletion() or check _SUCCESS file

PITFALL 5: "Intermediate paths fill up HDFS"
→ CAUSE: Pipeline fails midway, intermediate data never cleaned
→ Over days: /tmp/pipeline/2024-06-01/, /2024-06-02/, ...
→ Each run: 30GB intermediate × 2 stages = 60GB
→ 30 failed runs: 1.8TB wasted
→ FIX: Cleanup script that deletes intermediate data older than N days
→ FIX: Always attempt cleanup in finally{} block

PITFALL 6: "Configuration from Job 1 leaks into Job 2"
→ CAUSE: Reusing same Configuration object
→ Job 1 sets mapreduce.job.reduces=10
→ Job 2 should use 20 reducers but inherits 10
→ FIX: Create independent Configuration per job
→ OR: Explicitly set ALL configuration for each job

PITFALL 7: "ControlledJob state shows DEPENDENT_FAILED"
→ CAUSE: A dependency job failed → all downstream jobs skip
→ This is CORRECT behavior — not a bug
→ MEANING: "I didn't run because my prerequisite failed"
→ ACTION: Fix the failed job, rerun entire pipeline
→ Partial rerun from midpoint requires manual path management

SCALING DIMENSION 2: INTERMEDIATE DATA VOLUME

  → Each job boundary writes full output to HDFS
  → N-job chain with data volume D:
    Total HDFS writes: (N-1) × D × replication_factor
    Total HDFS reads:  (N-1) × D

  EXAMPLE: 3-job chain, 50GB data, replication=3
    HDFS writes: 2 × 50GB × 3 = 300GB disk writes
    HDFS reads:  2 × 50GB = 100GB disk reads
    Total I/O:   400GB for 50GB of actual data

  → THIS is the fundamental cost of MapReduce job chaining
  → Spark avoids this with in-memory persistence
  → But HDFS persistence enables restart from midpoint on failure

SCALING DIMENSION 3: CLUSTER RESOURCE UTILIZATION

  Sequential chain (3 jobs):
  ┌───────┬───────────────────────────────────────────────┐
  │ Time  │ Cluster utilization                           │
  ├───────┼───────────────────────────────────────────────┤
  │ 0-45m │ Job1 running — 100% cluster (map+reduce)     │

  │ 75m   │ Gap: teardown + startup                       │
  │ 75-100│ Job3 running — 100% cluster                  │
  └───────┴───────────────────────────────────────────────┘

  Parallel DAG (Job1+Job2 parallel, then Job3):
  ┌───────┬───────────────────────────────────────────────┐

import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.io.NullWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Mapper;

import java.io.IOException;
import java.util.regex.Pattern;

public class CdrCleanMapper
        extends Mapper<LongWritable, Text, NullWritable, Text> {

    // Phone number validation: 10-15 digits, optional + prefix
    private static final Pattern PHONE_PATTERN =
        Pattern.compile("^\\+?\\d{10,15}$");

    private Text cleanedRecord = new Text();

    @Override
    protected void map(LongWritable offset, Text line, Context context)
            throws IOException, InterruptedException {

        String record = line.toString().trim();

        // Skip header and empty lines
        if (record.isEmpty() || record.startsWith("CALLER")) {
            return;
        }

        // Input format:
        // CALLER | CALLEE | START_TIME | DURATION_SEC | CALL_TYPE | STATUS
        String[] fields = record.split("\\|");

        if (fields.length < 6) {
            context.getCounter("CLEAN", "MALFORMED_RECORD").increment(1);
            return;
        }

        String caller = fields[0].trim();
        String callee = fields[1].trim();
        String startTime = fields[2].trim();
        String durationStr = fields[3].trim();
        String callType = fields[4].trim();
        String status = fields[5].trim();

        // ── Validation rules ──

        // Rule 1: Phone numbers must be valid
        if (!PHONE_PATTERN.matcher(caller).matches()) {
            context.getCounter("CLEAN", "BAD_CALLER_PHONE").increment(1);
            return;
        }
        if (!PHONE_PATTERN.matcher(callee).matches()) {
            context.getCounter("CLEAN", "BAD_CALLEE_PHONE").increment(1);
            return;
        }

        // Rule 2: Duration must be positive number
        int duration;
        try {
            duration = Integer.parseInt(durationStr);
            if (duration < 0) {
                context.getCounter("CLEAN", "NEGATIVE_DURATION").increment(1);
                return;
            }
        } catch (NumberFormatException e) {
            context.getCounter("CLEAN", "BAD_DURATION").increment(1);
            return;
        }

        // Rule 3: Normalize phone numbers (remove + prefix)
        String normalizedCaller = caller.startsWith("+") ? caller.substring(1) : caller;
        String normalizedCallee = callee.startsWith("+") ? callee.substring(1) : callee;

        // Rule 4: Normalize call type to uppercase
        callType = callType.toUpperCase();

        // Build cleaned record
        String cleaned = String.join(" | ",
            normalizedCaller, normalizedCallee, startTime,
            String.valueOf(duration), callType, status);

        cleanedRecord.set(cleaned);
        context.write(NullWritable.get(), cleanedRecord);
        context.getCounter("CLEAN", "VALID_RECORDS").increment(1);
    }
}

import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.io.NullWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Mapper;

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;
import java.net.URI;
import java.util.HashMap;
import java.util.Map;

public class CdrEnrichMapper
        extends Mapper<LongWritable, Text, NullWritable, Text> {

    // Customer profile lookup: phone → "name|plan|region"
    private Map<String, String> customerProfiles = new HashMap<>();
    private Text enrichedRecord = new Text();

    @Override
    protected void setup(Context context)
            throws IOException, InterruptedException {

        // Load customer profiles from Distributed Cache (Project 15)
        URI[] cacheFiles = context.getCacheFiles();

        if (cacheFiles != null && cacheFiles.length > 0) {
            for (URI cacheFile : cacheFiles) {
                String filename = new java.io.File(cacheFile.getPath()).getName();
                BufferedReader reader = new BufferedReader(new FileReader(filename));

                String line;
                while ((line = reader.readLine()) != null) {
                    if (line.startsWith("PHONE") || line.trim().isEmpty()) continue;

                    // Format: PHONE | NAME | PLAN_TYPE | REGION
                    String[] parts = line.split("\\|");
                    if (parts.length >= 4) {
                        String phone = parts[0].trim();
                        String profile = parts[1].trim() + " | "
                                       + parts[2].trim() + " | "
                                       + parts[3].trim();
                        customerProfiles.put(phone, profile);
                    }
                }
                reader.close();
            }
        }

        context.getCounter("ENRICH", "PROFILES_LOADED")
            .increment(customerProfiles.size());
    }

    @Override
    protected void map(LongWritable offset, Text line, Context context)
            throws IOException, InterruptedException {

        String record = line.toString().trim();
        if (record.isEmpty()) return;

        // Input (from CleanJob output):
        // CALLER | CALLEE | START_TIME | DURATION | CALL_TYPE | STATUS
        String[] fields = record.split("\\|");
        if (fields.length < 6) return;

        String caller = fields[0].trim();

        // Look up customer profile for caller
        String profile = customerProfiles.get(caller);

        if (profile != null) {
            // Enriched format:
            // CALLER | CALLEE | START_TIME | DURATION | CALL_TYPE | STATUS | NAME | PLAN | REGION
            String enriched = record + " | " + profile;
            enrichedRecord.set(enriched);
            context.write(NullWritable.get(), enrichedRecord);
            context.getCounter("ENRICH", "ENRICHED_RECORDS").increment(1);
        } else {
            // No profile found — still emit with "UNKNOWN" placeholders
            String enriched = record + " | UNKNOWN | UNKNOWN | UNKNOWN";
            enrichedRecord.set(enriched);
            context.write(NullWritable.get(), enrichedRecord);
            context.getCounter("ENRICH", "UNKNOWN_CALLER").increment(1);
        }
    }
}

import org.apache.hadoop.io.LongWritable;
import org.apache.hadoop.io.NullWritable;
import org.apache.hadoop.io.Text;
import org.apache.hadoop.mapreduce.Mapper;
import org.apache.hadoop.mapreduce.Reducer;
import org.apache.hadoop.mapreduce.lib.output.MultipleOutputs;

import java.io.IOException;

// ── MAPPER: Key by caller phone number ──

class CdrAggregateMapper
        extends Mapper<LongWritable, Text, Text, Text> {

    private Text callerKey = new Text();

    @Override
    protected void map(LongWritable offset, Text line, Context context)
            throws IOException, InterruptedException {

        String record = line.toString().trim();
        if (record.isEmpty()) return;

        // Enriched format:
        // CALLER | CALLEE | START_TIME | DURATION | CALL_TYPE | STATUS | NAME | PLAN | REGION
        String[] fields = record.split("\\|");
        if (fields.length < 9) return;

        String caller = fields[0].trim();
        callerKey.set(caller);

        context.write(callerKey, line);
    }
}

// ── REDUCER: Aggregate per customer + write to multiple outputs ──

class CdrAggregateReducer
        extends Reducer<Text, Text, NullWritable, Text> {

    private MultipleOutputs<NullWritable, Text> multipleOutputs;
    private Text outputValue = new Text();

    private static final int SUSPICIOUS_CALL_COUNT = 50;
    private static final int SUSPICIOUS_DURATION = 36000; // 10 hours total

    @Override
    protected void setup(Context context)
            throws IOException, InterruptedException {
        multipleOutputs = new MultipleOutputs<>(context);
    }

    @Override
    protected void reduce(Text callerPhone, Iterable<Text> records,
                          Context context)
            throws IOException, InterruptedException {

        int totalCalls = 0;
        int totalDuration = 0;
        int voiceCalls = 0;
        int smsCalls = 0;
        String customerName = "UNKNOWN";
        String planType = "UNKNOWN";
        String region = "UNKNOWN";

        for (Text record : records) {
            String[] fields = record.toString().split("\\|");
            if (fields.length < 9) continue;

            totalCalls++;

            try {
                totalDuration += Integer.parseInt(fields[3].trim());
            } catch (NumberFormatException e) { /* skip */ }

            String callType = fields[4].trim();
            if ("VOICE".equals(callType)) voiceCalls++;
            else if ("SMS".equals(callType)) smsCalls++;

            // Take customer info from any record (same for all)
            if ("UNKNOWN".equals(customerName)) {
                customerName = fields[6].trim();
                planType = fields[7].trim();
                region = fields[8].trim();
            }
        }

        // ── Write summary to "summary" channel ──
        String summary = String.format(
            "%s | %s | %s | %s | %d | %d | %d | %d",
            callerPhone.toString(), customerName, planType, region,
            totalCalls, totalDuration, voiceCalls, smsCalls
        );
        outputValue.set(summary);

        // Dynamic path by region (Project 20 pattern)
        String regionClean = region.replaceAll("[^a-zA-Z0-9]", "_").toLowerCase();
        multipleOutputs.write(
            "summary", NullWritable.get(), outputValue,
            "by_region/" + regionClean + "/"
        );

        context.getCounter("AGGREGATE", "CUSTOMER_SUMMARIES").increment(1);

        // ── Check for suspicious patterns → write to "flagged" channel ──
        if (totalCalls > SUSPICIOUS_CALL_COUNT
                || totalDuration > SUSPICIOUS_DURATION) {

            String flagged = String.format(
                "ALERT | %s | %s | %s | calls=%d | duration=%ds | reason=%s",
                callerPhone.toString(), customerName, region,
                totalCalls, totalDuration,
                totalCalls > SUSPICIOUS_CALL_COUNT ? "HIGH_CALL_COUNT" : "HIGH_DURATION"
            );
            outputValue.set(flagged);

            multipleOutputs.write("flagged", NullWritable.get(), outputValue);
            context.getCounter("AGGREGATE", "FLAGGED_CUSTOMERS").increment(1);
        }
    }

    @Override
    protected void cleanup(Context context)
            throws IOException, InterruptedException {
        multipleOutputs.close();  // MANDATORY (Project 20 lesson)
    }
}

import org.apache.hadoop.conf.Configuration;
import org.apache.hadoop.conf.Configured;
import org.apache.hadoop.fs.FileSystem;

public class CdrPipelineDriver extends Configured implements Tool {

    // ══════════════════════════════════════════════════════
    // PATH MANAGEMENT: All paths defined centrally
    // Prevents typo mismatches between jobs (Pitfall 1)
    // ══════════════════════════════════════════════════════

    private Path rawInputPath;
    private Path customerProfilePath;
    private Path cleanedPath;      // intermediate — deleted after success
    private Path enrichedPath;     // intermediate — deleted after success
    private Path finalOutputPath;  // permanent output

    private void initPaths(String rawInput, String outputBase, String runId) {
        this.rawInputPath = new Path(rawInput);
        this.customerProfilePath = new Path("/data/customer_profiles/profiles.txt");

        // Unique intermediate paths per run (prevents collision)
        String intermBase = "/tmp/cdr_pipeline/" + runId;
        this.cleanedPath = new Path(intermBase + "/cleaned");
        this.enrichedPath = new Path(intermBase + "/enriched");
        this.finalOutputPath = new Path(outputBase + "/" + runId);
    }

    // ══════════════════════════════════════════════════════
    // JOB FACTORY METHODS: Create each job independently
    // Each job gets its OWN Configuration (prevents leakage)
    // ══════════════════════════════════════════════════════

    private Job createCleanJob() throws IOException {
        Configuration conf = new Configuration(getConf());
        Job job = Job.getInstance(conf, "CDR Pipeline — Stage 1: Clean");
        job.setJarByClass(CdrPipelineDriver.class);

        // Map-Only job (Project 11): zero reducers
        job.setMapperClass(CdrCleanMapper.class);
        job.setNumReduceTasks(0);

        job.setOutputKeyClass(NullWritable.class);
        job.setOutputValueClass(Text.class);

        job.setInputFormatClass(TextInputFormat.class);
        job.setOutputFormatClass(TextOutputFormat.class);

        FileInputFormat.addInputPath(job, rawInputPath);
        FileOutputFormat.setOutputPath(job, cleanedPath);

        return job;
    }

    private Job createEnrichJob() throws IOException {
        Configuration conf = new Configuration(getConf());
        Job job = Job.getInstance(conf, "CDR Pipeline — Stage 2: Enrich");
        job.setJarByClass(CdrPipelineDriver.class);

        // Map-Only job with Distributed Cache (Project 15, 16)
        job.setMapperClass(CdrEnrichMapper.class);
        job.setNumReduceTasks(0);

        job.setOutputKeyClass(NullWritable.class);
        job.setOutputValueClass(Text.class);

        job.setInputFormatClass(TextInputFormat.class);
        job.setOutputFormatClass(TextOutputFormat.class);

        // Input = output from CleanJob (SAME PATH object)
        FileInputFormat.addInputPath(job, cleanedPath);
        FileOutputFormat.setOutputPath(job, enrichedPath);

        // Add customer profiles to Distributed Cache
        job.addCacheFile(customerProfilePath.toUri());

        return job;
    }

    private Job createAggregateJob() throws IOException {
        Configuration conf = new Configuration(getConf());
        Job job = Job.getInstance(conf, "CDR Pipeline — Stage 3: Aggregate");
        job.setJarByClass(CdrPipelineDriver.class);

        job.setMapperClass(CdrAggregateMapper.class);
        job.setReducerClass(CdrAggregateReducer.class);

        job.setMapOutputKeyClass(Text.class);
        job.setMapOutputValueClass(Text.class);

        job.setOutputKeyClass(NullWritable.class);
        job.setOutputValueClass(Text.class);

        job.setNumReduceTasks(4);

        job.setInputFormatClass(TextInputFormat.class);

        // Input = output from EnrichJob
        FileInputFormat.addInputPath(job, enrichedPath);
        FileOutputFormat.setOutputPath(job, finalOutputPath);

        // MultipleOutputs configuration (Project 20)
        MultipleOutputs.addNamedOutput(job, "summary",
            TextOutputFormat.class, NullWritable.class, Text.class);
        MultipleOutputs.addNamedOutput(job, "flagged",
            TextOutputFormat.class, NullWritable.class, Text.class);

        LazyOutputFormat.setOutputFormatClass(job, TextOutputFormat.class);

        return job;
    }

    // ══════════════════════════════════════════════════════════════
    // APPROACH A: Sequential waitForCompletion
    // Simplest approach — linear chain, no parallelism
    // Good for: 2-3 jobs in strict sequence
    // ══════════════════════════════════════════════════════════════

    private int runSequential() throws Exception {
        FileSystem fs = FileSystem.get(getConf());
        Instant pipelineStart = Instant.now();

        // ── Pre-cleanup: Remove stale intermediate data ──
        System.out.println("[CLEANUP] Removing stale intermediate paths...");
        fs.delete(cleanedPath, true);
        fs.delete(enrichedPath, true);
        fs.delete(finalOutputPath, true);

        try {
            // ════════════════════════════════════════
            // STAGE 1: CLEAN
            // ════════════════════════════════════════
            System.out.println("\n" + "=".repeat(60));
            System.out.println("[STAGE 1/3] CleanJob — STARTING");
            System.out.println("  Input:  " + rawInputPath);
            System.out.println("  Output: " + cleanedPath);
            Instant stage1Start = Instant.now();

            Job cleanJob = createCleanJob();
            boolean stage1Success = cleanJob.waitForCompletion(true);

            Duration stage1Duration = Duration.between(stage1Start, Instant.now());
            System.out.println("[STAGE 1/3] CleanJob — "
                + (stage1Success ? "SUCCEEDED" : "FAILED")
                + " (" + formatDuration(stage1Duration) + ")");

            if (!stage1Success) {
                System.err.println("❌ PIPELINE ABORTED: CleanJob failed");
                System.err.println("   Check YARN UI for job details");
                return 1;
            }

            // Log counters from Stage 1
            long validRecords = cleanJob.getCounters()
                .findCounter("CLEAN", "VALID_RECORDS").getValue();
            long malformed = cleanJob.getCounters()
                .findCounter("CLEAN", "MALFORMED_RECORD").getValue();
            System.out.println("  Records: " + validRecords + " valid, "
                + malformed + " dropped");

            // ════════════════════════════════════════
            // STAGE 2: ENRICH
            // ════════════════════════════════════════
            System.out.println("\n" + "=".repeat(60));
            System.out.println("[STAGE 2/3] EnrichJob — STARTING");
            System.out.println("  Input:  " + cleanedPath);
            System.out.println("  Output: " + enrichedPath);
            Instant stage2Start = Instant.now();

            Job enrichJob = createEnrichJob();
            boolean stage2Success = enrichJob.waitForCompletion(true);

            Duration stage2Duration = Duration.between(stage2Start, Instant.now());
            System.out.println("[STAGE 2/3] EnrichJob — "
                + (stage2Success ? "SUCCEEDED" : "FAILED")
                + " (" + formatDuration(stage2Duration) + ")");

            if (!stage2Success) {
                System.err.println("❌ PIPELINE ABORTED: EnrichJob failed");
                System.err.println("   CleanJob output preserved at: " + cleanedPath);
                System.err.println("   Fix and rerun from Stage 2 manually");
                return 2;
            }

            long enriched = enrichJob.getCounters()
                .findCounter("ENRICH", "ENRICHED_RECORDS").getValue();
            long unknown = enrichJob.getCounters()
                .findCounter("ENRICH", "UNKNOWN_CALLER").getValue();
            System.out.println("  Records: " + enriched + " enriched, "
                + unknown + " unknown caller");

            // ════════════════════════════════════════
            // STAGE 3: AGGREGATE
            // ════════════════════════════════════════
            System.out.println("\n" + "=".repeat(60));
            System.out.println("[STAGE 3/3] AggregateJob — STARTING");
            System.out.println("  Input:  " + enrichedPath);
            System.out.println("  Output: " + finalOutputPath);
            Instant stage3Start = Instant.now();

            Job aggregateJob = createAggregateJob();
            boolean stage3Success = aggregateJob.waitForCompletion(true);

            Duration stage3Duration = Duration.between(stage3Start, Instant.now());
            System.out.println("[STAGE 3/3] AggregateJob — "
                + (stage3Success ? "SUCCEEDED" : "FAILED")
                + " (" + formatDuration(stage3Duration) + ")");

            if (!stage3Success) {
                System.err.println("❌ PIPELINE ABORTED: AggregateJob failed");
                System.err.println("   EnrichJob output preserved at: " + enrichedPath);
                return 3;
            }

            long summaries = aggregateJob.getCounters()
                .findCounter("AGGREGATE", "CUSTOMER_SUMMARIES").getValue();
            long flagged = aggregateJob.getCounters()
                .findCounter("AGGREGATE", "FLAGGED_CUSTOMERS").getValue();
            System.out.println("  Summaries: " + summaries
                + ", Flagged: " + flagged);

            // ════════════════════════════════════════
            // SUCCESS — Clean up intermediate data
            // ════════════════════════════════════════
            Duration totalDuration = Duration.between(pipelineStart, Instant.now());
            System.out.println("\n" + "=".repeat(60));
            System.out.println("✅ PIPELINE COMPLETED SUCCESSFULLY");
            System.out.println("   Total time: " + formatDuration(totalDuration));
            System.out.println("   Final output: " + finalOutputPath);

            System.out.println("\n[CLEANUP] Deleting intermediate data...");
            fs.delete(cleanedPath, true);
            fs.delete(enrichedPath, true);
            System.out.println("[CLEANUP] Done — intermediate paths removed");

            // Pipeline summary
            System.out.println("\n" + "=".repeat(60));
            System.out.println("PIPELINE SUMMARY:");
            System.out.println("  Raw input records:      ~" + (validRecords + malformed));
            System.out.println("  After cleaning:         " + validRecords);
            System.out.println("  After enrichment:       " + (enriched + unknown));
            System.out.println("  Customer summaries:     " + summaries);
            System.out.println("  Flagged customers:      " + flagged);
            System.out.println("  Drop rate:              "
                + String.format("%.1f%%",
                    (1.0 - (double) summaries / (validRecords + malformed)) * 100));
            System.out.println("=".repeat(60));

            return 0;

        } catch (Exception e) {
            System.err.println("❌ PIPELINE EXCEPTION: " + e.getMessage());
            System.err.println("   Intermediate data preserved for debugging:");
            System.err.println("   Cleaned:  " + cleanedPath);
            System.err.println("   Enriched: " + enrichedPath);
            throw e;
        }

    // Supports parallelism for independent branches
    // More robust for complex pipelines
    // ══════════════════════════════════════════════════════════════

    private int runWithJobControl() throws Exception {
        FileSystem fs = FileSystem.get(getConf());

        // Pre-cleanup
        fs.delete(cleanedPath, true);
        fs.delete(enrichedPath, true);
        fs.delete(finalOutputPath, true);

        Instant pipelineStart = Instant.now();

        // ── Create all three jobs ──
        Job cleanJob = createCleanJob();
        Job enrichJob = createEnrichJob();
        Job aggregateJob = createAggregateJob();

        // ── Wrap in ControlledJob ──
        ControlledJob cClean = new ControlledJob(cleanJob.getConfiguration());
        cClean.setJob(cleanJob);

        ControlledJob cEnrich = new ControlledJob(enrichJob.getConfiguration());
        cEnrich.setJob(enrichJob);

        ControlledJob cAggregate = new ControlledJob(aggregateJob.getConfiguration());
        cAggregate.setJob(aggregateJob);

        // ── Define dependencies ──
        // EnrichJob depends on CleanJob
        cEnrich.addDependingJob(cClean);
        // AggregateJob depends on EnrichJob
        cAggregate.addDependingJob(cEnrich);

        // ── Create JobControl and add all jobs ──
        JobControl jobControl = new JobControl("CDR-Pipeline");
        jobControl.addJob(cClean);
        jobControl.addJob(cEnrich);
        jobControl.addJob(cAggregate);

        // ── Start JobControl in a separate thread ──
        // JobControl.run() is a blocking loop — must run in its own thread
        Thread controlThread = new Thread(jobControl);
        controlThread.setDaemon(true);
        controlThread.start();

        System.out.println("=".repeat(60));
        System.out.println("PIPELINE STARTED via JobControl");
        System.out.println("  Dependencies: Clean → Enrich → Aggregate");
        System.out.println("=".repeat(60));

        // ── Monitor loop: poll until all jobs finish or timeout ──
        long timeoutMs = 4 * 60 * 60 * 1000; // 4 hour timeout
        long startMs = System.currentTimeMillis();
        int previousRunning = -1;
        int previousSuccess = -1;

        while (!jobControl.allFinished()) {

            // Timeout check
            if (System.currentTimeMillis() - startMs > timeoutMs) {
                System.err.println("❌ PIPELINE TIMEOUT after 4 hours");
                jobControl.stop();
                return 99;
            }

            // Progress reporting (only when state changes)
            List<ControlledJob> running = jobControl.getRunningJobList();
            List<ControlledJob> success = jobControl.getSuccessfulJobList();
            List<ControlledJob> failed = jobControl.getFailedJobList();

            if (running.size() != previousRunning
                    || success.size() != previousSuccess) {

                System.out.println(String.format(
                    "[PIPELINE STATUS] Running: %d | Succeeded: %d | Failed: %d | Waiting: %d",
                    running.size(),
                    success.size(),
                    failed.size(),
                    3 - running.size() - success.size() - failed.size()
                ));

                // Log which jobs are running
                for (ControlledJob cj : running) {
                    System.out.println("  ▶ RUNNING: " + cj.getJob().getJobName());
                }

                previousRunning = running.size();
                previousSuccess = success.size();
            }

            // Early termination on failure
            if (!failed.isEmpty()) {
                System.err.println("❌ JOB FAILURE DETECTED:");
                for (ControlledJob cj : failed) {
                    System.err.println("   FAILED: " + cj.getJob().getJobName());
                }
                // JobControl automatically marks dependent jobs as DEPENDENT_FAILED
                // Wait for all to reach terminal state
            }

            Thread.sleep(5000); // Poll every 5 seconds
        }

        // ── JobControl finished — analyze results ──
        jobControl.stop();
        Duration totalDuration = Duration.between(pipelineStart, Instant.now());

        List<ControlledJob> successJobs = jobControl.getSuccessfulJobList();
        List<ControlledJob> failedJobs = jobControl.getFailedJobList();

        System.out.println("\n" + "=".repeat(60));

            .findCounter("AGGREGATE", "CUSTOMER_SUMMARIES").getValue();
        long flagged = aggregateJob.getCounters()
            .findCounter("AGGREGATE", "FLAGGED_CUSTOMERS").getValue();
        System.out.println("    Summaries: " + summaries + ", Flagged: " + flagged);

        // Cleanup intermediate data
        System.out.println("\n[CLEANUP] Deleting intermediate data...");
        fs.delete(cleanedPath, true);
        fs.delete(enrichedPath, true);

        // Also clean parent temp directory if empty
        Path tempParent = cleanedPath.getParent();
        if (fs.exists(tempParent) && fs.listStatus(tempParent).length == 0) {
            fs.delete(tempParent, true);
        }
        System.out.println("[CLEANUP] Done");

        System.out.println("\n  Final output: " + finalOutputPath);
        System.out.println("=".repeat(60));

    private String formatDuration(Duration d) {
        long hours = d.toHours();
        long minutes = d.toMinutesPart();
        long seconds = d.toSecondsPart();

        if (hours > 0) {
            return String.format("%dh %dm %ds", hours, minutes, seconds);
        } else if (minutes > 0) {
            return String.format("%dm %ds", minutes, seconds);
        } else {
            return String.format("%ds", seconds);
        }
    }

#!/bin/bash

while read mod_date dir_path; do
    # Calculate age in days
    mod_epoch=$(date -d "${mod_date}" +%s 2>/dev/null)
    now_epoch=$(date +%s)
    
    if [ -n "${mod_epoch}" ]; then
        age_days=$(( (now_epoch - mod_epoch) / 86400 ))

            echo "  DELETING: ${dir_path} (${age_days} days old)"
            hdfs dfs -rm -r "${dir_path}"
        else
            echo "  KEEPING:  ${dir_path} (${age_days} days old)"
        fi
    fi
done

WHAT PROJECT 21 ESTABLISHED:

✅ PROBLEM: Production data pipelines require multiple MapReduce
   jobs executed in sequence with dependency management
   → Manual execution is error-prone and unscalable
   → Without dependency tracking, downstream jobs run on bad data

✅ TWO CHAINING APPROACHES:
   APPROACH A — Sequential waitForCompletion()
     → Simplest: job1.wait() → job2.wait() → job3.wait()
     → Linear only, no parallelism
     → Good for 2-3 jobs in strict sequence

   APPROACH B — JobControl + ControlledJob
     → DAG-aware: define dependencies between jobs
     → Enables parallelism for independent branches
     → Good for 3-7 jobs with branching

✅ INTERMEDIATE DATA MANAGEMENT:
   → Each job boundary writes to HDFS (expensive but durable)
   → Unique paths per run prevent collisions
   → Delete intermediate data ONLY after final job succeeds
   → Preserved intermediate data enables midpoint restart on failure

✅ CRITICAL PATTERNS:
   → Define ALL paths as constants (prevent typo mismatches)
   → Fresh Configuration per job (prevent config leakage)
   → Check return values before proceeding
   → Log stage transitions with timestamps
   → Collect counters across ALL jobs for pipeline accounting
